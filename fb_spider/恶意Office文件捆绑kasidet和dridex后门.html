恶意Office文件捆绑kasidet和dridex后门

老王隔壁的白帽子 2016-02-16 +5 共155346人围观 ，发现 2 个不明物体 网络安全

0×01  介绍

在这之前，我们已经介绍过了Direx银行木马通过Office恶意的宏病毒等不同的方法来进行传播的案列。然而，在过去的两周我们发现这些宏病毒不仅仅可以释放Direx后门，而且还在操作系统上释放Kasidet后门。这些恶意的Office文档攻击方法通过邮件进行鱼叉式网络钓鱼正在不断的扩散开来。并且在Office文档中的恶意宏病毒具有一定的迷惑性，代码如下所示：

宏指令从硬编码的URL下载payload。我们捕获到的不同文档中的payload使用如下URL:

l armandosofsalem[.]com/l9k7hg4/b4387kfd[.]exe
l trinity.ad-ventures[.]es/l9k7hg4/b4387kfd[.]exe
l 188.226.152[.]172/l9k7hg4/b4387kfd[.]exe

本文我们将详细分析此次出现的Kasidet木马。

0×02 Kasidet 分析

安装：

Kasidet安装目录为%APPDATA%文件夹。木马创建一个名为“Y1FEZFVYXllb”的文件夹，这个硬编码的字符串在恶意样本中。同样的字符串作为互斥体名称，并创建注册表项以确保系统重启的持久性。

AntiVM检查：

再执行下面的一系列检查，Kasidet尝试检测分析系统：

通过Windows的API函数”IsDebuggerPresent”和”CheckRemoteDebuggerPresent”检查调试信息。同样检查下面的比较流行的沙箱相关的字符串：

User Name: "MALTEST", "TEQUILABOOMBOOM", "SANDBOX", "VIRUS", "MALWARE"
File Name: "SAMPLE", "VIRUS", "SANDBOX"

同样通过检查kernel32.dll是否导出函数”wine_get_unix_file_name”来检测wine软件。通过注册表项检查Vmware、VirtualBox、QEMU以及Bochs。

信息窃取能力：

Kasidet通过下面的两种方法窃取用户信息：

1. 内存刮擦—这种方法可以在POS机系统的内存中获得用户的信用卡数据。通过扫描除操作系统以外所有的运行进程，如下：

System
smss.exe
csrss.exe
winlogon.exe
lsass.exe
spoolsv.exe
devenv.exe

使用如下格式的URI将这些信息传回攻击者：

d=1&id=&name=&type=&data=&p=< Process elevation status >

2.浏览器钩子—从Web浏览器中窃取数据。可以在FireFox，谷歌以及IE浏览器中注入代码,浏览器的名字使用相同的hash函数被Carberp恶意软件加密，而不是简单的以文本形式存在。下面的API是浏览器中用来窃取敏感数据的钩子函数。

使用如下格式的URI将这些信息传回攻击者：

ff=1&id=&name=&host=&form=< Base64 encoded HTTP header data>&browser=

如果系统区域设置或GeoUserID是俄罗斯，那么Kasidet变种的信息窃取功能将被禁用。
