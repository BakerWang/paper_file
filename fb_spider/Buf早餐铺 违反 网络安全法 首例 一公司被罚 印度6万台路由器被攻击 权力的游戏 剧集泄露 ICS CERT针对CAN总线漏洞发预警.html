欧阳洋葱 2017-08-02 共175481人围观 ，发现 1 个不明物体 资讯

Black Hat 和 DEF CON 两大盛会刚刚结束，实际上黑客界的著名两会还有许多值得探讨和深入的议题等待各位挖掘。FreeBuf 未来一段时间内也会进一步为各位呈现这两会中的有趣内容，可能在早报中，也可能单独成文，感兴趣的同学请多多留意了。

今天 BUF 早餐铺的主要内容有——印度超 60000 台调制解调器和路由器遭遇攻击下线；《权力的游戏》未公开剧集泄露；《网络安全法》首个案例：一公司未留用户登录日志被罚；美国 ICS-CERT 针对 CAN 总线漏洞发出预警； Chrome 插件“Copyfish”被攻击者劫持。


【企业安全】

BrickerBot 恶意程序在印度多地爆发，超 60000 台调制解调器和路由器下线

BrickBot 恶意程序作者近日宣称针对印度多地发动网络攻击。本次攻击导致超过 60000 台调制解调器和路由器断开网络。本次事件影响到印度两大通讯服务提供商 BSNL（Bharat Sanchar Nigam Limited）和 MTNL（Mahanagar Telephone Nigam Limited） 。实际上从上周二到上周六期间，就有用户上报称其互联网连接断开，路由器及调制解调器的红色 LED 灯常亮。BSNL 表示是恶意程序所致，还说本次事件导致印度东北、北部和南部互联网断开连接。BSNL 预计受影响的有 60000 台设备，覆盖其宽带连接服务的 45% ；MTNL方面则没有提供任何数据。

BSNL 还说，恶意程序主要攻击那些没有修改默认密码的调制解调器，在此次事件过后已要求用户进行修改。BSNL 技术团队上周一直忙于帮助用户进行调制解调器和路由器的密码重置。BrickerBot 恶意程序主要感染基于 Linux 的 IoT 和网络设备。其它同类恶意程序都是让感染设备加入僵尸网络，而 BrickerBot 则是覆盖设备闪存，令其变砖。

BrickerBot 作者表示他写这款恶意程序是为了提升如 ISP 运营商等的安全意识。他还说，由于 BSNL 和 MTNL 允许通过 7547 端口从外部连接到其网络中，所以才遭遇了本次攻击，这个端口是管理协议 TR069 所用，ISP 用其发送命令并管理用户的路由器。两家运营商随后过滤了针对该端口的访问，因此被攻击设备显著减少。[来源：BleepingComputer]

HBO 被黑，美剧《权力的游戏》剧本和其它剧集或泄露

本周一，HBO表示其网络遭遇网络攻击。国外媒体报道称本次攻击导致 HBO 尚未公开的 1.5 TB 电视节目和资料泄露，其中包括这一季《权力的游戏》相关信息，和尚未公开的剧集剧本，还有其它一些内容。攻击者周末通过邮件将此事散布给了多家媒体。

当前 HBO 已经联系了执法机构和安全公司。HBO声明中并未提到究竟哪些数据泄露，不过多家外媒还提到《球手们（Ballers）》和《 104 号房间（Room 104）》尚未播出剧集也遭遇泄露。这已经是 HBO 第二次遭遇网络攻击了，早在 2015 年《权力的游戏》第五季前 4 集也曾遭遇了泄露。[来源：SecurityWeek]

北美多家航空公司遭遇数据泄露，并向用户和雇员发出示警

过去几天多家北美航空公司遭遇网络攻击和数据泄露，他们近期都已经向客户及员工发出示警：

维珍美国航空表示检测到针对信息系统的未经授权访问，3100 名员工和承包人的身份凭证被窃取，另有 110 人的社保号、驾驶证或政府发放ID、住址和医疗相关信息被窃；

加拿大的西捷航空上周五通知用户，未经授权第三方获取了 WestJet Rewards 成员资料数据，不过泄露数据不包含信用卡等金融信息。西捷航空也已经报警，当前正在通知受影响用户，并建议其按照常规流程修改密码；

美国佛罗里达廉价航空公司 Spirit Airlines 近期通过邮件通知用户，FREE SPIRIT 账户数据遭遇泄露，这家公司表示有人在第三方网站公布了客户信息，总共涉及到 1170 万 Spirit 帐号。据说攻击者先前对系统中的漏洞向 Spirit 航空发出了警告，但在公司忽略其警告后，攻击者打算将这批数据放到暗网上出售——这批泄露的数据包括账户号、密码、出生日期、电话号码、住址、邮箱地址。不过 Spirit 坚持表示，这批数据是从其它来源获取的——安全专家也确认这批数据的确来自多个不同的来源。[来源： SecurityWeek]

《网络安全法》实施后首个案例：一公司未留用户登录日志被罚

重庆市公安局网安总队近日成功查处首例违反《网络安全法》行政案件，当地一网络运营者在提供网络服务过程中，未依法留存用户登录网络日志，因此受到警告处罚，并被责令限期十五日内整改。

重庆市公安局网安总队在日常检查中发现，重庆市某科技发展有限公司自今年6月1日《网络安全法》正式实施以来，在提供互联网数据中心服务时，存在未依法留存用户登录相关网络日志的违法行为。警方称，根据《网络安全法》相关规定，决定给予该公司警告处罚，并责令限期十五日内进行整改。该公司收到《行政处罚通知书》后，立即编制了《整改方案》并着手实施整改，待整改完成后，公安机关将对其整改情况进行验收。[来源：网易科技]

【工控安全】

美国 ICS-CERT 针对 CAN 总线漏洞发出预警

美国工业控制系统网络紧急响应中心 ICS-CERT 上周五发出漏洞预警，这次的漏洞影响到 Controller Area Network（CAN） 总线标准。 CAN 是个高可靠性的串行总线通讯标准，许多汽车都采用该标准——它支持汽车中的各部分组件相互通讯，CAN 也应用在医疗和其它领域。

意大利研究人员先前就公布了一份 paper ，提到多个 CAN 缺陷，其中还列举了一种攻击方式可发动 DoS 攻击。研究人员也公布了 PoC exploit ，以及视频演示在 2012 阿尔法罗密欧 Giulietta 车型中禁用泊车所用的传感器。ICS-CERT 表示虽然攻击需要攻击者物理访问汽车，但相较移往的攻击技术，这次的攻击更难被检测到。

而且因为CAN是多个行业及大量产品使用的标准，要进行漏洞修复不是件容易的事情。ICS-CERT表示当前正与供应商和研究人员合作，找到此类攻击缓解方案。[来源：SecurityWeek]

【DEF CON 黑客大会】

能打造恶意程序的机器学习系统OpenAI Gym，全面躲避检测

我们在先前 Black Hat USA 2017 的预热文章中提到过攻防领域未来机器学习的对抗。在刚刚结束的 DEF CON 大会上，专家演示如何利用名为 OpenAI Gym 的机器学习系统来构建恶意程序，而且恶意程序还能躲避检测。我们以往讨论机器学习的时候，都是谈机器学习在“防御”领域的发展，而这次是机器学习在“攻击”领域的应用。

来自 Endgame 的技术专家 Hyrum Anderson 采用 Elon Musk 的 OpenAI 框架构建恶意程序，这套系统实际上只是对看起来合法的代码进行了略微修改，成为恶意代码。这可以起到欺骗反病毒引擎的作用。就像 Anderson 说的那样，所有机器学习模型都有盲点，如果攻击者掌握足够多机器学习的知识，就可以找到这些盲点。

据说 Anderson 及其团队构建这套系统，对合法代码进行较小幅度的调整，然后提交作安全检查，得到反馈之后再作微调以加强恶意程序躲避检测的能力。超过 10 万样本与安全引擎经过这样 15 小时的培训后，16% 的恶意程序样本能够躲过安全系统。当前Anderson及其团队已经在GitHub上公布了OpenAI Gym的代码。[来源： SecurityAffairs]

90 分钟就能攻下美国大选所用的各种投票设备

这次的 DEF CON 有个 “Voting Machine Village” 活动，就是让参与者尝试入侵投票设备，帮助发现其漏洞。这个活动的发起应该是源自去年美国总统大选期间，大选所用的票选系统被攻击。

Voting Machine Village 活动准备了 30 台不同的投票设备，包括 Sequoia AVC Edge、ES&S iVotronic、AccuVote TSX、WinVote、Diebold Expresspoll 4000 等。结果这群参与者不到90分钟就完成了这些投票机的入侵。

比如说在美国 19 个州使用的 AccuVote TSX 和在 13 个州使用的 Sequoia AVC Edge 都存在严重安全漏洞；有安全研究人员入侵了 Diebold TSX 投票机的硬件和固件；Express-Poolbook 系统中则存在已知的 OpenSSL 漏洞 CVE-2011-4019 ，攻击者可利用漏洞发动远程攻击；还有攻击者在 WinVote 的操作系统中发现远程访问漏洞，可导致存储在设备中的选票数据暴露…[来源：TheHackerNews]

使用 Chrome 插件“Copyfish”要小心，它已经被攻击者劫持

最近有攻击者黑了德国开发团队 a9t9 的 Chrome Web Store 商店帐号，并通过劫持该团队开发的 Chrome 插件，来部署垃圾信息。这款插件名为 Copyfish ，用户用这款插件可以从图片、PDF文档和视频中获取文字信息，该插件用户数量已经超过 37500 名。最近黑客劫持了这款插件，在其中加入垃圾信息和广告。Firefox 版 Copyfish 不受影响。

a9t9 开发团队调查发现，先前有一名成员收到过伪装成来自 Chrome Web Store 官方的钓鱼邮件，邮件宣称需要更新其 Copyfish 插件，否则插件将从商店移除。邮件中携带有恶意链接，这名成员点击链接后，页面要求输入密码，据说页面和谷歌原版几乎一模一样。随后攻击者还将带垃圾信息的 Copyfish 插件移到了自己的开发者账户中。a9t9 团队对用户发出示警：不要再使用当前的 Copyfish 插件。[来源：TheHackerNews]
