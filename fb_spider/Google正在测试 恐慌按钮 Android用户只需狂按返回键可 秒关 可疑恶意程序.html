Google正在测试“恐慌按钮”：Android用户只需狂按返回键可“秒关”可疑恶意程序

Alpha_h4ck 2017-07-12 共257178人围观 ，发现 9 个不明物体 资讯

Android系统一直以来都因运行速度较慢和恶意软件较多等原因而饱受用户诟病，关于运行速度的问题，Android 8将会让大家耳目一新。而对于恶意软件较多这个问题来说，Google在Android 7.1中增加了一个名叫“恐慌按钮”的行为，该功能将允许用户通过狂按返回键来迅速关闭任何可疑的恶意程序。

Google将该功能作为一个可选项添加进了Android操作系统的config.xml文件中【第845行】:

< !-- Control the behavior when the userpanic presses the back button.
0 - Nothing
1 - Go to home
-- >
< integername="config_backPanicBehavior" >0< /integer >

backPanicBehavior这个选项当前还处于禁用状态，貌似Google还在对这个功能进行测试，我们还可以在PhoneWindowManager.java文件中找到与backPanicBehavior有关的设置。

static final int PANIC_PRESS_BACK_COUNT =4;
static final int PANIC_PRESS_BACK_NOTHING =0;
static final int PANIC_PRESS_BACK_HOME = 1;

//按下多少次返回键后才可触发恐慌模式

连按4次返回键之后即可退出任意App

根据这两份文件中的设置信息，如果Google在将来的Android操作系统中启用这项功能的话，用户将可以通过连按4次返回键来关闭系统当前所有运行的App，并返回手机的home界面。为了区分常规操作和“恐慌”操作，用户需要连续按下4次返回键，并且每次的间隔不能够超过0.3秒。在用户对手机返回键进行连续操作时，系统便会启动对用户按下返回键的监测，以此来判断是否需要触发该功能。

XDA论坛上的Android大神(也是第一个发现该功能的人)透露，如果Android用户怀疑自己的设备感染了恶意软件的话，用户就可以通过连按4下返回键关闭所有正在运行的App并回到Home界面。回到主屏幕之后，用户就可以将可疑App直接删除了。

目前我们还不清楚“恐慌按钮”这个功能是否已经开发完毕，而且我们也没有从Google那里得到正面评论。除此之外，我们也不知道该功能是否能够关掉那些会锁定手机屏幕的勒索软件。虽然该功能还需要进行大量测试，但至少听起来还是很有吸引力的。

用户现在可开启“恐慌按钮”，但建议别着急

用户如果愿意的话，可以自行开启该功能，不过由于Google目前还没有正式推出该功能，因此开启之后的风险需用户自行承担。

如果想要启用backPanicBehavior选项，用户必须修改SystemUI APK（存有上述两份文件）。因此，用户需要使用特殊的软件以及root权限来修改手机APK的内部文件，或者用户也可以将App下载到本地，然后在本地修改完成之后再安装到手机中。

这两种操作都非常的复杂，我们不建议普通用户进行这种尝试，更何况这种修改将有可能影响Android系统的正常运行。
