如何从ios8内存中获取敏感数据？

xiaoqin00 2016-04-22 共156264人围观 ，发现 3 个不明物体 系统安全终端安全

去年的一月，NetSPI在博客中发表了一篇博文——《从IOS设备中提取数据》。尽管NetSPI提供了一篇能让这个过程变得十分简单的脚本，但是它只能在iOS7或者更低的版本中使用并且要求要有GDB。

（GDB是GNU开源组织发布的一个强大的UNIX下的程序调试工具）。现在，GDB在iOS8中已经没有了。。。幸运的是，这里还有其他方法来实现GDB的功能并且从运行iOS8系列系统的苹果手机中提取数据。但是它有如下要求：

  • LLDB (http://lldb.llvm.org/)
  • Debugserver (part of Xcode)
  • Tcprelay.py (https://code.google.com/p/iphonetunnel-mac/source/browse/trunk/gui/tcprelay.py?r=5)

当然，你还需要一个已经越狱了的iPhone或者iPad。关于这方面的操作我在这就不再缀续了。

首先，开启tcprelay，这样我们就可以通过USB来连接苹果设备。

    $ ./tcprelay.py -t 22:2222 1234:1234
    Forwarding local port 2222 to remote port 22
    Forwarding local port 1234 to remote port 1234
    Incoming connection to 2222
    Waiting for devices...
    Connecting to device <MuxDevice: ID 17 ProdID 0x12a8 Serial '0ea150b00ba3deeacb42f399492b7990416a0c87' Location 0x14120000>
    Connection established, relaying data
    Incoming connection to 1234
    Waiting for devices...
    Connecting to device <MuxDevice: ID 17 ProdID 0x12a8 Serial '0ea150b00ba3deeacb42f399492b7990416a0c87' Location 0x14120000>
    Connection established, relaying data

“tcprelay.py -t 22:2222 1234:1234”这条命令是将两个本地端口重定向。前一个端口指令是将22号端口的ssh服务重定向到2222端口实现。后一个是开启debugserver服务端口。

然后你要连接上iOS设备并且开启debug服务（这里我默认已经将软件拷到电脑上了）。如果你没有拷到电脑上，你可以用scp来拷贝。

    $ ssh root@127.0.0.1 -p 2222
    root@127.0.0.1's password:

如果app已经在运行了，可以通过‘ps aux | grep <appname>’指令来查看app的名称并且将其连接到debugserver服务（注意是app的名称而不是ＰＩＤ）：

    root# ./debugserver *:1234 -a appname
    debugserver-@(#)PROGRAM:debugserver  PROJECT:debugserver-320.2.89
    for arm64.
    Attaching to process appname...
    Listening to port 1234 for a connection from *...
    Waiting for debugger instructions for process 0.

‘./debugserver *:1234 -a appname’ 命令是告诉软件开启1234端口并且连接到名为‘appname’的app上。这个过程也许要花费一些时间，请耐心等待。在Mac电脑中开启LLDB并且连接到运行在ios设备中的debugserver软件中。

现在，我们有中继端口1324并且debugserver在监听本地的1234端口。

接下来要做的就是将刚刚输出的数据转化为LLDB指令来从内存单元中获取我们能用的数据。在转化时，你可以跳过零填充区。

以下面的输出数据举例：

0x00000003 data-cstr        [0x000000000016e5e0-0x0000000000189067)  0x001275e0 0x0001aa87 0x00000002 appname.__TEXT.__cstring

转化为LLDB指令后就是：

Memory read --outfile ~/0x00000003data-cstr 0x000000000016e5e0 0x0000000000189067 –force

这指令是告诉LLDB程序去 0x000000000016e5e0到0x0000000000189067之间的内存中取出数据并且存放到 0x00000003data-cstr中。

(lldb) memory read --outfile ~/0x00000003data-cstr 0x000000000016e5e0 0x0000000000189067 –force

这条命令在执行完时，你不会看到所有创建的文件。当所有的文件创建好后，你就可查找你所需要的数据了（也许是数字，也许是密码之类的。）额，文件大概是这个样子的：

这种技术可以用来确定应用程序没有删除敏感信息。

还是那句老话，所有应用程序应该释放处理敏感信息的类和方法的内存，否则敏感信息可能会泄露。
