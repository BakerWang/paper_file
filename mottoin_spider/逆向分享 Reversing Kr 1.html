逆向分享 – Reversing.Kr (1)

Moto • 2016年8月30日 • 专题, 逆向破解 • 阅读 2342

学逆向有一段时间了，刚好拿reversing.kr的题练练手，这网站是棒子国的一个很不错的逆向学习网站，会提供一些专门的逆向程序供各位摩擦，大家有兴趣也可以去试试，在这里我分享一些做题的思路。

Windows

0x00 Easy Crack

关键点在check函数

判断输入的首字母是否等于E,第二个字母是否等于a,第三四个字母是否等于5y，剩余的是否等于R3versing。

flag:Ea5yR3versing

0x01 Easy Keygen

关键点在main函数

name以3个字节为一组，每组分别与16,32,48异或并转换为16进制数，最后与serial对比，脚本如下

serial = '5B134977135E7D13'
name = ''

for i in xrange(0, len(serial), 2):
 name += chr(int(serial[i:i+2], 16) ^ (16 * (i / 2 % 3 + 1)))

print name

flag:K3yg3nm3

0x02 Easy Unpack

加了壳的程序寻找OEP，用OD打开，跟到0040A1FB时有一个大跳

跳过去就是原始入口

flag:00401150

0x03 Music Player

这题是让播放器的播放时间大于1分钟，有两个检测点，首先当播放时间大于1分钟的时候会弹个对话框，用OD加载对rtcMsgBox下断，之后当时间大于1分钟的时候会断到，往上翻发现判断时长的指令

0xEA60等于60000，这里应该就是60000ms，把这个值改大点，然后再次运行弹出一个系统错误的对话框，对kernel32.RaiseException下断，之后就可以通过查看调用栈找到出错点在哪

跳过去看到jge

最后把jge改成jl就拿到flag了

flag:LIstenCare

0x04 Replace

这题用了混淆和自修改，大致流程是，把输入的值转换为整型放到eax，然后把eax的值存到0x4084D0，并+1、+1、+0x601605C7、+1、+1

 

之后把0x4084D0的值存到eax，还有一个自修改，把0x40466F的机器码改为0xC39000C6

把eax的值及其+1对应的地址的机器码修改为0x90

最后跳到0x401071，如果0x401071、0x401072是nop，那么就会显示Correct！

因此只要我们输入值+1+1+0x601605C7+1+1=0x401071，由此需要溢出，用0x100401071-1-1-0x601605C7-1-1

flag:2687109798

0x05 ImagePrc

思路有点奇特，IDA加载看到加载位图的操作

用PEiD查看是VC++6.0，于是用eXeScope导出资源

最后写个脚本显示这张位图

from PIL import Image

width = 200
height = 150

image_file = open('dump', 'rb')
data = image_file.read()
image = Image.frombuffer('RGB', (width, height), data, 'raw', 'RGB')
image = image.transpose(Image.FLIP_TOP_BOTTOM)
image.show()
image_file.close()

flag:GOT

0x06 Position

这题有多解，flag长度为4，最后一个字符是p，仅仅判断每个字符的低5位

两位相加的结果如下表所示

0 1
0 6 7
1 7 8

当1+0或者0+1时都为7，这就是导致有多解的原因，最后自己选择一个正确的就行

flag:bump

0x07 Direct3D FPS

这题很有意思，是一个射击游戏，用IDA加载搜索字符串看到Game Over!和Game Clear!，猜测游戏通关后显示Game Clear!，通过交叉引用看到这是对话框的标题，内容是一串乱码

再看内容的交叉引用，发现有一处对内容做了异或操作，等同于0x409184[eax * 0x210] ^ 0x407028[eax]

再用OD动态调试，发现0x409184[i * 0x210] = i * 4，写个脚本拿到flag

cipher = [67, 107, 102, 107, 98, 117, 108, 105, 76, 69, 92, 69, 95, 90, 70, 28, 7, 37, 37, 41, 112, 23, 52, 57, 1, 22, 73, 76, 32, 21, 11, 15, 247, 235, 250, 232, 176, 253, 235, 188, 244, 204, 218, 159, 245, 240, 232, 206, 240, 169]
plain = ''
for i in xrange(len(cipher)):
 plain += chr(cipher[i] ^ (i * 4))
print plain

flag:Thr3EDPr0m

0x08 Ransomware

这题是一个加密程序和一个加密后的文件，需要通过分析加密算法来解密加密文件，原文件是exe程序。首先发现加了UPX壳，用UPXShell脱壳后用IDA分析，初始分析的过程会有点卡，发现sub_401000和_main函数中有很多花指令

写个IDAPython脚本去掉sub_401000和_main函数中的花指令

start = 0x00401006
end = 0x004135CE
for i in xrange(start, end):
 PatchByte(i, 0x90)
start = 0x004135E9
end = 0x0044A775
for i in xrange(start, end):
 PatchByte(i, 0x90)

sub_401000去花后是一个空函数，反编译_main函数发现其中多次调用了sub_401000函数，这应该就是在IDA初始分析时卡的原因

这时已经可以分析了，但由于有点小强迫症，所以再写个脚本去掉调用sub_401000函数的指令

from idaapi import *

ea = 0x0044A775
func = get_func(LocByName('_main'))
while ea <= func.endEA:
 ea = FindCode(ea, SEARCH_DOWN | SEARCH_NEXT)
 if GetMnem(ea) == 'call' and GetOpnd(ea, 0) == 'sub_401000':
 for i in xrange(5):
 PatchByte(ea + i, 0x90)

之后再反编译_main函数，看到了清晰的加密流程

加密流程就是对原文件每个字节循环异或key的每个字节，然后再取反，由于原程序是exe程序，根据PE结构可以知道加密文件特定偏移处所对应的明文

对应

然后写个脚本获得key为letsplaychess

cipher_data = [
 0xC7, 0xF2, 0xE2, 0xFF, 0xAF, 0xE3, 0xEC, 0xE9, 0xFB, 0xE5, 0xFB, 0xE1,
 0xAC, 0xF0, 0xFB, 0xE5, 0xE2, 0xE0, 0xE7, 0xBE, 0xE4, 0xF9, 0xB7, 0xE8,
 0xF9, 0xE2, 0xB3, 0xF3, 0xE5, 0xAC, 0xCB, 0xDC, 0xCD, 0xA6, 0xF1, 0xF8,
 0xFE, 0xE9
]

plain_data = [
 0x54, 0x68, 0x69, 0x73, 0x20, 0x70, 0x72, 0x6F, 0x67, 0x72, 0x61, 0x6D,
 0x20, 0x63, 0x61, 0x6E, 0x6E, 0x6F, 0x74, 0x20, 0x62, 0x65, 0x20, 0x72,

 origin.write(chr((ord(enc_data[i]) ^ 0xFF) ^ ord(key[i % len(key)])))

最后运行解密后的exe程序拿到flag

flag:Colle System

0x09 Twist1

这题加了壳又各种反调试，心好累，首先用OD加载程序，用F4+F7可以来到跳到OEP的地方

程序开始是一个SetUnhandledExceptionFilter

然后401220是接受输入的函数，401240是验证函数，如果返回值为1就输出Correct!，返回值为0就输出Wrong

跟进401240函数，它先读取输入的值存到一个数组中，然后获得ZwQueryInformationProcess的地址，并把此地址处的16个字节复制到409150

复制完后，409150函数就相当于ZwQueryInformationProcess函数，之后就是ZwQueryInformationProcess的反调试，分别是ProcessDebugPort(0x7)、ProcessDebugObjectHandle(0x1E)、ProcessDebugFlags(0x1F)，可以改成nop或改返回值进行绕过

之后是取输入值的第7位与0x36异或

之后有一个GetThreadContext来检测硬件断点，可以把获得的调试寄存器的值改为0或者nop来进行绕过，然后判断第7位跟0x36异或后的值是否跟0x36相等，由此可以得到flag的长度是6位

长度确定为6后，程序首先把第1位循环右移6位，然后与0x49比较，确定第1位为R

然后第3位与0x77异或再与0x35比较，确定第3位为B

然后第2位与0x20异或再与0x69比较，确定第2位为I

然后第4位与0x21异或再与0x64比较，确定第4位为E

然后第5位与0x46异或再与0x8比较，确定第5位为N

然后第6位循环左移4位与0x14比较，确定第6位为A

最后输出Correct!

flag:RIBENA
