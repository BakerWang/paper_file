Android内核漏洞分析

Moto • 2016年8月25日 • 技术控, 终端安全 • 阅读 1743

0x00

环境介绍：Nexus5+Kernel3.4.0-gd59db4e+Android4.4.3。

编译Kernel的时候打开LKM(Loadable Kernel Modules)，方法很简单：make hammerhead_defconfig后往.config文件加入以下配置项：

编写以下测试代码

#include <linux/module.h>
#include <linux/kernel.h>
int init_module(void)
{
printk(KERN_ALERT "Hello android kernel...\n");
return 0;
}
void cleanup_module(void)
{
printk(KERN_INFO "Goodbye android kernel...\n");
}
MODULE_AUTHOR("Wanglisong");
MODULE_LICENSE("GPL");
MODULE_VERSION("1");
MODULE_DESCRIPTION("Hello World")

编译得到Android_module.ko文件，使用insmod可以加载内核模块，使用rmmod可以卸载内核模块，使用dmesg可以查看内核消息。

在内核空间编程与用户空间编程有很大的不同，copy_to_user函数是将内核数据拷贝到用户空间，copy_from_user函数是将用户空间的数据拷贝到内核空间；我们知道这时必须要检查参数是否有效，例如char __user *buf是否是用户空间的一个合法的地址，通过查看内核源码我们看到copy_to_user，copy_from_user都会首先调用

具体的编写请参考《Linux设备驱动程序》第三章。

0x02

在Linux内核中使用struct task_struct来描述我们的进程，就是我们常说的进程描述符，它包含有内核管理一个进程所需的所有信息，包括进程的状态，thread_info地址，地址空间，文件信息，挂起的信号，凭证等。struct thread_info分配在进程内核栈的顶端，通过下面的代码可以很方便的得到thread_info的地址

代码还是很容易理解的，在arm架构中，THREAD_SIZE为2*4096（4096：页大小），thread_info又在栈顶，所以屏蔽sp的低13个有效位即可；在arm64架构中THREAD_SIZE为4*4096，所以屏蔽sp的低14个有效位即可。因为thread_info中有一个task字段，所以通过（task_struct *）current_thread_info()->task很容易得到我们的task_struct。struct cred 就是进程的凭证信息，我们关注 user&group ID, capabilities, LSM security context（这里即SELinux）。

分别以shell和root去打开设备文件然后用dmesg命令查看结果如下图：

所以将一个普通shell权限的进程提权至root我们需要将它的struct cred的值改写。通常有两种方法：在内核态调用commit_creds(prepare_kernel_cred((void*)0))或者patch thread_info->addr_limit(在x86 ubuntu32上该值为0xC0000000,ARM 上该值为0xBF000000)位0xFFFFFFFF,突破内核地址空间和用户地址空间的界限，得到任意读写的权限，然后去改写struct cred。
