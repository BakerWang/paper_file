构造一个支持多端口的中间人网络TAP

Tank • 2016年11月21日 • 极客 • 阅读 1217

问题

在红队（美国情报组织）期间，一旦获得物理访问权限，下一步典型操作便是连接到内网。许多工具（如cobalt strike）提供优异日志记录功能，但其它工具对此有所欠缺。此问题的解决方案是记录本地所有的网络流量，从而提供所有网络执行操作的完整概述。

但是，使用这种方法也存在一些潜在的问题：

  • 每次建立新连接时都需要手动启动去记录
  • 虚拟机需要能够执行对所述的日志记录的能力（存储器，处理能力等）
  • 测试团队的每个成员都需要每天整合其网络日志

这些问题的解决方案是创建一个单独的中间人网络设备，作为测试客户端和网络之间的中间记录器。

寻找合适的硬件

最适合的设备是Banana Pi R1（BPI-R1），它大量的硬件功能，在本次场景中我们最感兴趣的是BPI-R1开发本上的4x千兆LAN端口和1x千兆WAN端口。 此外，这个设备可以运行Debian，BPI-R1完全适配Debian。

攻击计划

BPI-R1上的所有网络接口都属于同一物理接口（eth0），而配置和分离通过VLAN标记来实现。 在本文中，我们将创建一个具有以下功能的多端口网络分流器（NETWORK TAP）：

  • 构建不超过3个透明网桥设备和内部客户端（而不是NAT和防火墙）
  • 记录网桥与外部加密USB驱动器之间传递的所有流量
  • 便于管理和使用的界面，更安全地卸载和调试

准备操作系统

第一步，下载Bananian发行版并将这款开源操作系统安装到SD卡上。

 dd bs=4M if=bananian-1604.img of=/dev/sda1

安装完成后，将SD卡插入BPI-R1并启动配置过程。 初始登录可以使用密码为pi的默认root用户执行。 下一步,更新系统并安装所需的实用程序：

bananian-config
apt-get update
apt-get upgrade
apt-get install tcpdump bridge-utils isc-dhcp-server cryptsetup

配置开关

一旦安装了操作系统，我们就可以配置交换机。 设备上的物理套接字排序如下：

LAN     WAN
2 1 0 4    3

我们将对这些端口使用以下配置：

  • 端口2（VLAN 103）; 设备管理接口
  • 端口1,0,4（VLAN 102）; 测试连接到内部客户端网络的设备
  • 端口3（VLAN 101）; 直接连接到内部客户端网络

将端口1，0和4桥接到端口3，从而使连接到我们分接器的每个设备与内部客户端网络具有直接和透明的连接。

配置文件是swconfig,需要将/etc/network/if–pre–up.d/swconfig文件修改为以下内容：

#!/bin/sh
 
#---------------------------#
# BPI-R1 VLAN configuration #
#---------------------------#
#
ifconfig eth0 up
 
swconfig dev eth0 set reset 1
swconfig dev eth0 set enable_vlan 1
swconfig dev eth0 vlan 101 set ports '3 8t'
swconfig dev eth0 vlan 102 set ports '4 0 1 8t'
swconfig dev eth0 vlan 103 set ports '2 8t'
swconfig dev eth0 set apply 1

一旦配置了VLAN标签，我们可以通过/etc/network/interfaces直接配置网络接口：

# interfaces(5) file used by ifup(8) and ifdown(8)
# Include files from /etc/network/interfaces.d:
source-directory /etc/network/interfaces.d
 
auto lo
iface lo inet loopback
 
auto eth0
iface eth0 inet manual
 
# 101 – internal client network
auto eth0.101
iface eth0.101 inet manual
up ifconfig eth0.101 up
 
# 102 – red team devices
auto eth0.102
iface eth0.102 inet manual
up ifconfig eth0.102 up
 
# 103 – management network
auto lo eth0.103
iface eth0.103 inet static
address 192.168.10.1
netmask 255.255.255.0
 
# bridge between 101 and 102
auto br0
iface br0 inet manual
bridge_ports eth0.101 eth0.102

重启设备后，我们可以确认接口和网桥是通过ifconfig和brctl show配置和启用的：

bridge name    bridge id            STP enabled    interfaces
br0            8000.03130740c28c    no             eth0.101

配置管理VLAN

为了便于配置并方便在没有HDMI监视器和USB键盘的站点上使用，我们可以在管理界面上配置DHCP服务器。使用INTERFACES=”eth0.103″更新/etc/default/isc-dhcp-server，/etc/dhcp/dhcpd.conf配置文件如下：

ddns-update-style none;
option domain-name "tap.local";
default-lease-time 600;
max-lease-time 7200;
authoritative;
log-facility local7;
 
subnet 192.168.10.0 netmask 255.255.255.0 {
    range 192.168.10.100 192.168.10.110;
    option subnet-mask 255.255.255.0;
    option broadcast-address 192.168.10.225;
    option routers 192.168.10.1;
}

还可以设置SSH在此接口上监听， /etc/ssh/sshd_config

ListenAddress 192.168.10.1

配置记录设备

虽然可以在SD卡本身上存储日志，但更便携和可靠的机制是写入外部USB设备。 由于可能潜在存储敏感客户端数据，此USB设备应在写入数据之前进行加密：

#!/bin/sh
 
# unmount and close if device already mounted and logging
killall -9 tcpdump 2>/dev/null
umount /root/tap-log 2>/dev/null
cryptsetup luksClose tap-log 2>/dev/null
 
# setup the device for writing
mkdir -p /root/tap-log
cryptsetup --verify-passphrase luksFormat /dev/sda1 -c aes -s 256 -h sha256
cryptsetup luksOpen /dev/sda1 tap-log
mkfs.ext4 /dev/mapper/tap-log
mount /dev/mapper/tap-log /root/tap-log
 
# log packets between bridge
tcpdump -i br0 -nn -w /root/tap-log/log-`date +'%s-%d-%m-%Y'` &
