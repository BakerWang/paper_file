Empyre：渗透Mac OSX的漏洞利用框架

SecToolkit • 2016年11月16日 • 工具 • 阅读 1283

简介

EmPyre是一款基于Python，远程入侵OSX系统的Post Exploitation框架工具。它内置有OSX漏洞利用模块，与Metasploit相比，让人耳目一新。

安装

git clone  https://github.com/adaptivethreat/EmPyre/

进入EmPyre目录运行安装脚本install.sh

就这样，EmPyre安装并准备就绪，现在只要进入EmPyre目录运行./empyre就可以启动界面。

EmPyre加载成功后你将看到如下的菜单

使用

创建监听

开始为受害系统创建回连监听进程

进入Listener菜单，键入options查看信息

其中，默认Listener进程为test，其它信息可以根据目标系统进行设置修改

设置完毕就可以运行

创建恶意文档

首先，我们需要创建恶意office宏文档。键入命令”main”回到主菜单，并使用“usestager”命令调用宏文档制作功能

我们只需告诉stager使用哪个我们创建的listener进程

然后就可以生成恶意宏代码

把这些输出的宏代码复制到word或excel宏文档中保存。

创建一个新的宏，然后将代码从EmPyre主机粘贴到宏

（注：如果攻击目标是64 位系统，需要把宏代码第1行“Private Declare Function“更改为”Private Declare PtrSafe Function” 。）

当受害者打开该恶意文档，会出现以下宏调用界面，如果点击enable macros按钮之后，恶意宏代码将会执行。

在EmPyre界面下可以通过agent查看回连信息

然后你就可以开始与上线的Mac主机进行交互

UKFOM…字符串是EmPyre对受害主机的独特标识，之后，可以使用以下命令控制主机

以下是一些可以利用的控制模块，其中troll/osx/say是我最喜欢用的

当然，你还可以使用help命令查看其它控制、执行和查询功能

视频演示

EmPyre Web Interface

EmPyre Web Interface是一款基于web接口的EmPyre程序，它允许用户从web界面管理EmPyre任务。

项目主页

https://github.com/interference-security/empyre-web

安装运行

获取EmPyre：https://github.com/adaptivethreat/EmPyre

./empyre --headless --restport port --username empyre_username --password empyre_password

Requirements

PHP Curl安装使用EmPyre Web Interface

安装PHP Curl

sudo apt-get install php5-curl

检查PHP Curl

命令行：

PHP脚本：

<?php
echo (function_exists('curl_version') ? "Curl found": "Curl not found");
?>

提醒

CSRF保护没有执行，因为它影响到EmPyre Web Interface的工作。它会在下一版本中实现。
