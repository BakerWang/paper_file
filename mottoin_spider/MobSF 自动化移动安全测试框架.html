MobSF：自动化移动安全测试框架

SecToolkit • 2016年11月23日 • 工具 • 阅读 1843

项目主页

https://github.com/ajinabraham/Mobile-Security-Framework-MobSF/

简介

移动安全框架 (MobSF) 是一款智能化、自动化化的开源移动应用（Android / iOS）自动测试框架，能够对以上两种移动应用进行静态和动态分析

它可以有效、快速地对应用APK 和IPA文件 及压缩的源代码进行审计分析。MobSF可以被用来对Android 和iOS的应用进行高效快速的安全分析。此框架支持二进制文件(APK & IPA)和源码压缩包。同时，MobSF 也能够通过其API Fuzzer功能模块，对 Web API 的安全性进行检测，如收集信息，分析安全头部信息，识别移动API 的具体漏洞，如XXE、SSRF、路径遍历，IDOR以及其他的与会话和API调用速率限制有关的逻辑问题。此外，MobSF还集成了Androdi Tamer 和 BlackArch。

安装运行

运行环境

  • Python 2.7
  • Oracle JDK 1.7或以上版本
  • Oracle VirtualBox
  • iOS IPA分析（需在 Mac系统上执行）所需命令行工具（ Mac系统）
  • 硬件配置：4GB 或以上内存，5G硬盘空间。

下载

下载MobSF最新版：https://github.com/ajinabraham/Mobile-Security-Framework-MobSF/releases

下载MobSF VM 0.3 ova 文件：https://goo.gl/QxgHZa

安装

该框架目前暂时只于Windows 7, 8, 8.1, 10, Ubuntu, OSX Mavericks等系统平台上进行测试

  • Windows：解压MobSF压缩文件到C:\MobSF
  • Mac：解压MobSF压缩文件到/Users/[username]/MobSF
  • Linux：解压MobSF压缩文件到/home/[username]/MobSF

配置静态分析器

通过pip安装MobSF Python 依赖包

Windows

C:\Python27\Scripts\pip.exe install -r requirements.txt

如果pip.exe在脚本目录中不可用请下重新安装Python 2.7

Unix

pip install -r requirements.txt --user

运行

python manage.py runserver

如果需要在具体端口上运行，可以运行以下命令：

python manage.py runserver PORT_NO

如果要将MobSF放到一个公网IP地址上可以运行以下命令：

python manage.py runserver IP:PORT_NO

配置MobSF VM

动态分析器目前只支持针对 Android APK文件进行分析，硬件环境要求为需要计算机拥有4GB 内存和支持完全虚拟化。

首先，配置动态分析器我们需要获取以下4个方面的信息，

  • VM UUID
  • 快照 UUID
  • 主机／代理 IP
  • VM／设备 IP

步骤如下

  • 打开VirtualBox的，选择文件- >导入设备，选中MobSF_VM_X.X.ova文件

  • 在进行导入过程中，请勿更改任意配置，一切按照默认设置进入下一步
  • 一旦OVA文件导入成功，我们将在VirtualBox上看到一个以MobSF_VM_X.X命名的新的条目
  • 接下来右击MobSF VM，并选择设置一项，选中到网络选项卡，这里我们需要配置两个网络适配器

适配器1 启用并于attached to 中，选择Host-only Adapter模式，重命名适配器名称，因为我们需要通过该名称来识别主机／代理 IP，配置如下图；

适配器2 启用并于attached to中，选择NAT模式，配置如下图：

  • 保存设置，启动MobSF VM。当该VM启动的时记下VM的 IP地址

  • 一旦该虚机启动后，它会停留在一个锁屏状态上，解锁密码默认为1234

  • 获取 主机／代理 IP

Windows :在命令提示符中输入ipconfig，记下与适配器1 相同名称的适配器IP地址

Unix :在命令提示符中输入ifconfig，记下与适配器1 相同名称的适配器IP地址

注：VirtualBox的IP和MobSF VM IP应该在相同的网段。如果您MobSF VM的IP和适配器IP是在不同的网络范围请参阅：当MobSF VM VirtualBox的与主机适配器不是在同一个网络范围怎么办？

  • 接着在MobSF 虚机中选中Wi-Fi设置，设置代理IP（上一步获取到的IP）和端口（1337）

  • 保存设置，并回到MobSF虚机的Home界面上，等待大约30秒的时间，之后保存好 MobSF虚机的快照

  • 一旦快照保存，右击MobSF VM，然后选择Show in Explorer或Show in Finder

  • 在任何文本编辑器打开MobSF_VM_X.X.vbox文件，并记下VM UUID和快照UUID

这里的uuid是虚拟机的UUID，currentSnapshot是快照的UUID。

  • 现在我们有了配置动态分析器所需要的信息（主机/ 代理IP，VM IP，VM UUID和快照UUID）

打开MobSF/settings.py文件，并将其中参数的值设置如下

  • UUID = VM UUID
  • SUUID = Snapshot UUID
  • VM_IP = VM IP
  • PROXY_IP = Host/Proxy IP
