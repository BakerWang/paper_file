DRAKVUF：黑盒二进制分析平台

Sam • 2016年11月10日 • 逆向破解 • 阅读 1766

介绍

DRAKVUF是一种基于虚拟化的无代理黑盒二进制分析系统。DRAKVUF允许对任意二进制文件（包括操作系统）进行深入的执行跟踪，在虚拟机内不需安装任何用于分析的特殊软件。

硬件要求

DRAKVUF使用Intel CPU中的硬件虚拟化扩展。需要具有虚拟化支持（VT-x）和扩展页表（EPT）的Intel CPU。DRAKVUF不会在其他CPU（如AMD）或没有所需虚拟化扩展的Intel CPU上工作。

支持的平台

DRAKVUF目前支持Windows7 32bit和Windows7 64bit两个版本。

恶意软件分析

DRAKVUF为隐匿恶意软件分析提供了一个完美的平台，因为其足迹从恶意软件的角度几乎无法解决。虽然DRAKVUF主要是针对恶意软件分析开发的，但它不限于该功能，还可以监视任意二进制文件的执行。

当前状态

目前系统提供以下核心功能：

  • 无代理启动二进制执行。
  • 监视Windows内部内核函数。
  • 客户端多虚拟CPU的支持。
  • 跟踪堆分配。
  • 跟踪正在访问的文件。
  • 在删除文件之前从内存中提取文件。
  • 通过写时拷贝内存和磁盘克隆分析VMs

请注意，DRAKVUF仍处于早期开发阶段，缺乏其他二进制分析器具有的一些功能。DRAKVUF还有很多需要改进和扩展的地方。 以下只是列举几个：

  • 结构化日志到JSON / MySQL / MongoDB。
  • 提取的文件自动提交到VirusTotal。
  • 支持Linux。
  • 使用Xen内存共享对未使用的内存进行运行时重复数据删除。
  • 集成到恶意软件分析操作中，如HOLMES。

DRAKVUF是一个开源项目，修补程序和错误报告会在在Github页面显示！有关使用项目的更多信息，请参见DRAKVUF Wiki。

安装指南

UPDATED 10/07/2016：因为最近更改了Microsoft的符号服务器，创建Rekall配置文件需要一个新版本的Rekall。所需的Rekall版本现在可作为DRAKVUF中的子模块使用。

UPDATED 10/07/2016：如果您使用最新的DRAKVUF主分支，请更新LibVMI，最近有一些更改需要对它进行更新！

在基于Debian的Linux发行版上构建Xen和DRAKVUF通常需要以下软件包。该系统主要在Debian Jessie和Ubuntu 14.04 LTS测试。

sudo apt-get install wget git bcc bin86 gawk bridge-utils iproute libcurl3 libcurl4-openssl-dev bzip2 module-init-tools pciutils-dev build-essential make gcc clang libc6-dev libc6-dev-i386 linux-libc-dev zlib1g-dev python python-dev python-twisted python-gevent python-setuptools libncurses5-dev patch libvncserver-dev libssl-dev libsdl-dev iasl libbz2-dev e2fslibs-dev git-core uuid-dev ocaml libx11-dev bison flex ocaml-findlib xz-utils gettext libyajl-dev libpixman-1-dev libaio-dev libfdt-dev cabextract libglib2.0-dev autoconf automake libtool check libjson-c-dev libfuse-dev libsystemd-daemon-dev checkpolicy

我们将安装一个略微修改的Xen 4.7版本，其中包括未反向运行的关键错误修复。

cd ~
git clone https://github.com/tklengyel/drakvuf
cd drakvuf
git submodule init
git submodule update
cd xen
./configure --enable-githttp
make -j4 dist-xen
make -j4 dist-tools

要安装Xen，要安装Xen，使dom0获得4GB RAM分配和两个专用CPU内核（将其调为首选）：

sudo su
make -j4 install-xen
make -j4 install-tools
echo "GRUB_CMDLINE_XEN_DEFAULT=\"dom0_mem=4096M,max:4096M dom0_max_vcpus=4 dom0_vcpus_pin=true hap_1gb=false hap_2mb=false altp2m=1 flask_enforcing=1\"" >> /etc/default/grub
echo "/usr/local/lib" > /etc/ld.so.conf.d/xen.conf
ldconfig
echo "none /proc/xen xenfs defaults,nofail 0 0" >> /etc/fstab
echo "xen-evtchn" >> /etc/modules
echo "xen-privcmd" >> /etc/modules
update-rc.d xencommons defaults 19 18
update-rc.d xendomains defaults 21 20
update-rc.d xen-watchdog defaults 22 23

要启用加载DRAKVUF所需的Xen XSM模块，需要编辑GRUB Xen发现规则脚本（通常位于/etc/grub.d/20_linux_xen），在模块指定内核的多重入口后添加下面内容（即策略应该是最后加载的模块）：

module ${rel_dirname}/xenpolicy-4.7.0

完成这些步骤后，需进行以下设置：

update-grub
reboot

还要确保你运行一个相对较新的内核：

uname -r

一旦你启动进入Xen，验证一切工作：

sudo xen-detect

输出应为：在Xen v4.7上运行PV context：

xl list

输出应该类似：

Name                                        ID   Mem VCPUs	State	Time(s)
Domain-0                                     0  4096     2     r-----     614.0

设置LVM卷组以保存VM磁盘，然后创建卷：

lvcreate -L20G -n windows7-sp1 vg

使用以下模板从ISO安装Windows 7（根据需要调整）：

arch = 'x86_64'
name = "windows7-sp1"
seclabel='normal:vm_r:drakvuf_domU_t'
maxmem = 3000
memory = 3000
vcpus = 1
maxcpus = 1
builder = "hvm"
boot = "cd"
hap = 1
acpi = 1
on_poweroff = "destroy"
on_reboot = "destroy"
on_crash = "destroy"
vnc=1
vnclisten="0.0.0.0"
usb = 1
usbdevice = "tablet"
altp2mhvm = 1
shadow_memory = 16
audio=1
soundhw='hda'
vif = [ 'type=ioemu,model=e1000,bridge=xenbr0,mac=00:06:5B:BA:7C:01' ]
disk = [ 'phy:/dev/vg/windows7-sp1,hda,w', 'file:/path/to/your/windows7.iso,hdc:cdrom,r' ]

在drakvuf目录创建LibVMI文件夹并切换到该目录：

cd ~/drakvuf/libvmi
./autogen.sh
./configure

确保输出以下内容：

Feature      | Option                    | Reason
-------------|---------------------------|----------------------------
Xen Support  | --enable-xen=yes          | yes
Xen Events   | --enable-xen-events=yes   | yes
KVM Support  | --enable-kvm=no           | libvirt missing
File Support | --enable-file=yes         | yes
Shm-snapshot | --enable-shm-snapshot=no  | no
-------------|---------------------------|----------------------------

Tools        | Option                    | Reason
-------------|---------------------------|----------------------------
Examples     | --enable-examples=yes     | yes
VMIFS        | --enable-vmifs=yes        | yes

Extra features
----------------------------------------------------------------------
Support of Rekall profiles: yes

构建和安装LibVMI：

make
sudo make install
sudo echo "export LD_LIBRARY_PATH=\$LD_LIBRARY_PATH:/usr/local/lib" >> ~/.bashrc
cd tools/pyvmi
python setup.py build
sudo python setup.py install

下载Volatility：

cd ~
git clone https://github.com/volatilityfoundation/volatility
cd volatility
cp ~/drakvuf/libvmi/tools/pyvmi/pyvmiaddressspace.py volatility/plugins/addrspaces
python setup.py build
sudo python setup.py install

构建和安装Rekall：

cd ~/drakvuf/rekall/rekall-core
python setup.py build
sudo python setup.py install

重要部分是：

现在生成Rekall配置文件：

cd /tmp
rekall fetch_pdb pdb_filename ntkrpamp 684da42a30cc450f81c535b4d18944b12
rekall parse_pdb ntkrpamp.pdb > windows7-sp1.rekall.json
sudo mv windows7-sp1.rekall.json /root

准备好这个配置文件，我们可以创建LibVMI配置：

sudo su
printf "windows7-sp1 { \n\
    ostype = \"Windows\"; \n\
    rekall_profile = \"/root/windows7-sp1.rekall.json\"; \n\
}" > /etc/libvmi.conf
exit

现在我们准备好构建和安装DRAKVUF：

cd ~/drakvuf
autoreconf -vi
./configure
make

简单地跟踪系统的执行：

sudo ./src/drakvuf -r <rekall profile> -d <domid>

例如：

sudo ./src/drakvuf -r /root/windows7-sp1.rekall.json -d 7

要查看所有可用选项：

./src/drakvuf
