漏洞分析：CVE-2017-0497 via ele7enxxh

Jarry • 2017年3月27日 • 漏洞分析 • 阅读 614

当终端设备尝试解析大小尺寸为0的TIFF文件时，SIGFPE可以在SkRawCodec.cpp的GetPixel上进行。

  • CVE ID：CVE-2017-0497;
  • Android ID：A-33300701;
  • 严重程度： Moderate;
  • Google设备更新：全部;
  • 更新的AOSP版本：7.0/7.1.1。

描述

http://androidxref.com/7.0.0_r1/xref/external/skia/src/codec/SkRawCodec.cpp#689

if (imageSize.h / width > maxDiffRatio || imageSize.h < width ||
  imageSize.v / height > maxDiffRatio || imageSize.v < height) {
  return SkCodec::kInvalidScale;
}

没有检查TIFF文件的大小信息，宽度和高度可以为零。

Attack vector

PoC：https://github.com/ele7enxxh/poc-exp/raw/master/CVE-2017-0497/CVE-2017-0497.tiff

触发此漏洞的方法如下:

BitmapFactory.decodeFile（“CVE-2017-0497.tiff”）;
