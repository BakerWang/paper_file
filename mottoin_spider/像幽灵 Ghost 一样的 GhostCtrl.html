像幽灵（Ghost）一样的“GhostCtrl”

M0tto1n • 2017年7月20日 • 安全报告 • 阅读 348

六月底我们报道了关于以色列医院发现的利用LNK文件窃取信息的恶意软件（RETADUP蠕虫）的分析报告。不过最新的观测发现，RETADUP蠕虫只是攻击活动的一部分，实际的攻击远比我们之前认为的更加危险，至少在影响力方面是如此。

本文中，我们重点分析一下最新发现的一款Android 恶意软件，一旦感染，就可以直接接管用户的移动设备。

趋势科技将这个Android 后门命名为“GhostCtrl”，这一恶意软件能够静默的控制受感染的设备。

目前我们发现了三个“GhostCtrl”版本的样本文件。第一个版中，恶意软件具有窃取信息和控制部分设备的功能，没有混淆；第二个版本中恶意软件支持对更多设备进行劫持；第三个版本结合了早期版本的特性。根据推测，这一恶意软件家族在未来将继续发展。

“GhostCtrl”简直就像幽灵（Ghost）一样

研究人员认为，GhostCtrl是跨平台远控工具OmniRAT的变种、，或者至少是基于OmniRAT开发出来的。OmniRAT在2015年11月登上过新闻头条，它是一款非常流行的、功能强大的、商业版的远程控制工具，适用于 Android,、Windows、 Linux 和Mac 等操作系统，终身许可证售价为 25～75 美元，可以预见的是各种OmniRAT的破解教程在各种地下论坛广为传播，一些成员甚至为它提供补丁。

我们发现一个显著的标识，可以证明GhostCtrl这款恶意的APK是OmniRAT的一个分支，考虑到OmniRAT是商业运作的（ RAT 即服务），能够在编译期间进行修改（或移除）。

图1: GhostCtrl第三版中的resources.arsc文件中，透露了它是OmniRAT的一个变种（快照）

GhostCtrl的持久性机制

GhostCtrl会伪装成一个合法的或流行的应用程序，使用目标APP的名称、彩信、whatsapp、甚至是Pokemon GO进行传播。用户安装了这些应用程序后，程序会通过base64解码的方式从资源文件中读取并加载一个恶意的APK。

动态点击了APK安装包之后，恶意的APK将请求用户安装它，用户想要避免安装是很棘手的，因为即使用户取消“请求安装页面”的提示符，这一消息仍然会立即弹出。恶意的APK并没有桌面图标，安装之后，恶意软件的主程序将启动服务，在系统后台运行。

图2:  APK安装包引导恶意APK的过程。

恶意的APK具有后门功能，通常会命名为com.android.engine误导用户认为它是一个合法的系统应用程序，通过socket（设备之间的端点通信）与 C&C服务器进行联系，新的 Socket是“hef–klife[.]ddns.net”, 3176。

GhostCtrl能够完全掌控受感染设备的一切

C&C传输过来的命令是加密过的，恶意程序在设备本地进行解密。有趣的是，我们还发现后门程序会连接到另外的一个域，而不是直接连接到C&C服务器的IP地址，这可能是为了隐藏它们的流量。检测到几个动态的 DNS服务器，它们有时候会指向相同的C&C的IP地址：

  • hef–klife[.]ddns[.]net
  • f–klife[.]ddns[.]net
  • php[.]no-ip[.]biz
  • ayalove[.]no-ip[.]biz

我们观察到一个值得注意的命令，它包含了执行代码和Object数据，使得攻击者能够指定目标和内容，为网络犯罪份子提供了更多便利。这一命令允许攻击者对设备进行操作，即使未获得设备所有者的同意或感知。

下面列出一些执行代码，以及它们各自的作用：

  • ACTION CODE =10, 11: 控制设备的WI-FI状态
  • ACTION CODE= 34: 实时监控手机传感器的数据
  • ACTION CODE= 37: 设置手机的UiMode（UI模式，比如夜间模式或开车模式）
  • ACTION CODE= 41: 控制设备的振动功能，包括振动模式和启动时间
  • ACTION CODE= 46: 下载图片壁纸
  • ACTION CODE= 48: 列出当前目录下的文件信息，并上传到 C&C 服务器
  • ACTION CODE= 49: 在特定目录中删除一个文件
  • ACTION CODE= 50: 在特定目录中重命名一个文件
  • ACTION CODE= 51: 将目标文件上传到 C&C 服务器
  • ACTION CODE= 52: 创建一个特定的目录
  • ACTION CODE= 60: 使用文本转化为语音的功能（将文本转换成语音/音频）
  • ACTION CODE= 62: 向攻击者指定的号码发送SMS / MMS，可以自定义信息的内容
  • ACTION CODE= 68: 删除浏览器历史记录
  • ACTION CODE= 70: 删除 SMS消息
  • ACTION CODE= 74: 下载文件
  • ACTION CODE= 75: 向攻击者指定的号码打电话
  • ACTION CODE= 77: 打开活动视图相关的应用程序，攻击者也可以指定统一资源标识符（URI），用于打开浏览器、地图、拨号视图等
  • ACTION CODE= 78: 控制系统的红外线发射器
  • ACTION CODE= 79: 运行由攻击者指定的shell命令并上传输出结果

另外一个独特的C&C命令是一个整型命令，负责窃取设备的数据。犯罪份子会收集并上传不同类型的、敏感的、有价值的信息，包括通话记录、短信记录、联系人、电话号码、SIM 序列号、位置信息和浏览器书签等。

与其他 Android 信息窃取软件相比，GhostCtrl窃取的数据类型范围非常广泛，除了上面提到的那些信息之外，还可以窃取诸如Android OS版本，用户名，Wi-Fi，电池／蓝牙／音频的状态，UiMode，传感器，相机、浏览器、搜索上的数据、服务进程、活动信息、壁纸等相关的数据。

它还能够从攻击者指定的电话号码中截取文本信息，最惊人的是能够偷偷的录制语音或音频，然后在一个特定的时间上传到 C&C服务器上。所有窃取的信息在上传到 C&C 服务器之间都会被加密。

图3: 上传后删除某些信息（代码示例）

图4:窃取信息的大部分功能模块都在“transfer” Package中

其他的 C&C 命令都是自定义的，比如“account”、“audioManager” 和“clipboard”等，这些命令都会触发恶意的动作。值得注意的是，这些命令在Android RATs中并不常见。

  • 清除/重置攻击者指定的帐户的密码。
  • 让手机播放不同的音频效果
  • 指定剪贴板中的内容
  • 自定义通知和快捷链接的样式和内容等
  • 控制蓝牙搜索并连接到另一台设备
  • 将可访问性的参数属性设置为true，并终止正在进行的电话呼叫。

GhostCtrl版本的迭代情况

GhostCtrl的第一个版本建立了一个框架，实现了获取管理员级别权限的功能；在第二个版本中增加了功能模块；经过第二版和第三版的迭代，恶意软件的劫持功能逐渐增加。

图5:GhostCtrl的第一个版本的框架，能够获得管理员级别的特权

图6: 第一版（左）和第二版（右）的后门功能比较

图7: GhostCtrl第二版中获取设备管理员权限的功能（代码示例）

GhostCtrl第二版中，也具有移动端勒索软件的功能，它可以锁定设备的屏幕、重置密码、Root受感染的设备、劫持摄像机、创建计划任务用以拍照或录制视频并以 mp4的格式将它们秘密的上传到  C&C 服务器上。

图9:GhostCtrl能够root受感染的设备（代码示例）

图10:GhostCtrl第三版本的攻击链

在GhostCtrl第三版中，wrapper APK会先释放一个加壳的APK，后者将脱壳后会释放 main APK、一个可执行的Dalvik（DEX）、一个 ELF 文件。 DEX 和 ELF 文件都具有解密功能，在恶意APK 的Main函数运行时，解密恶意APK调用的字符串和API。攻击链如此冗长，使得对它的检测变得更具挑战性，wrapper APK隐藏了加壳的APK以及资源目录中的 DEX 和 ELF 文件，这无疑又加重了被检测到的难度。

缓解

正如文章开头所言，GhostCtrl后门程序再加上一个信息窃取蠕虫，影响效果相当明显。攻击者视图隐藏他们的行踪，进一步可以推测他们的目的并非单纯的感染终端设备。随着移动设备在企业和终端用户的日益普及，GhostCtrl的威胁确实足以引起恐慌。

不过，针对GhostCtrl的防护，除了重视其危害之外，企业应当部署多层安全机制，以便更好地管理数据的风险。信息安全专业人员和 IT ／系统管理员可以采取一些最佳实践，以更好的保护 BYOD 设备，比如：

  • 保持设备更新；Android补丁是零散的，组织可以根据需求制定或配置设备更新策略，平衡生产力和安全性；
  • 应用最小特权原则，防止未经授权的访问和安装可疑应用程序；
  • 实现一个应用程序信誉系统，有效检测和阻止恶意的或可疑的应用程序；
  • 部署防火墙，入侵检测系统和入侵防御系统，在终端和移动设备层面加强防护，阻止恶意软件的恶意网络活动；
  • 实施并加强移动设备管理策略，进一步减少潜在的安全风险；
  • 使用加密、网络分割和数据隔离等方式来限制数据暴露或损坏可能造成的进一步的影响；
  • 定期备份数据，防止设备丢失、盗窃或被恶意加密；
