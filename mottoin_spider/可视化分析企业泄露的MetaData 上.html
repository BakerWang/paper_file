可视化分析企业泄露的MetaData（上）

M0tto1n • 2017年7月26日 • 安全报告 • 阅读 362

概述

本文中将探讨文档的元数据（MetaData）是什么？为什么它能为高级攻击者提供丰富的信息源？文件的元数据可以收集各种高度敏感的数据，例如：用户名、公司使用的软件、文件共享的位置等等。另外，本文还将展示如何寻找您所在公司或组织正在泄露的元数据，收集这些元数据并创建一个漂亮的Splunk仪盘表，帮助你完善一种全新的威胁情报和态势感知可视化平台。

文档的元数据是指什么？

如果你已经对元数据的概念了然于胸，可以跳过这一部分，直接进入“绘制数据足迹”章节。

如今，几乎所有的组织每天都会生产一些被批准发布的文件，这里存在一个问题：作为一个公司，事实上发表的数据比你想象的要多得多。每次创建文档时，用于创建文档的软件都会插入所谓的元数据（MetaData），简而言之，所谓的元数据就是指描述有关数据的数据。

这是什么意思呢？意思就是，元数据并非文件内容，而是：

  • 这份文件是用什么软件创建？（如Microsoft Word 2007）
  • 这份文件是哪个用户创建？（如MottoIn, u12345）
  • 创建这份文件的计算机名称？（如 Super MottoIn Lab PC 2017）
  • 这份文件的创建时间？
  • 这份文件上一次修改的时间？
  • 这份文件何时被打印？
  • 文件输出时的磁盘／共享文件夹的路径？（如\\DOC_SERVER\DOCS\PATENTS\Wormhole_travel_engine.docx）
  • 其他更多

这些信息是在创建或修改文档时，被软件自动插入到文档中的。插入信息的数量和种类依赖于所使用的软件及其配置。很少有组织会限制这类信息的传播。因此，这些敏感信息被暴露出来，你的竞争对手非常愿意花费时间去分析这些信息。

元数据&侦查：黑客的金矿

一般来讲，组织最可能面临的攻击者有三种：

  • 脚本小子
  • 先进的攻击者／APTs
  • 内鬼

第三类人士不在本文的讨论范围之内，而前两种人则有几个有趣的差别。一个脚本小子和一个先进的攻击者之间最大的区别是他们肯在KillChain的侦查阶段花费多少时间。

侦查是网络攻击活动中最重要的阶段之一，往往决定了攻击是否能够成功。此时，攻击者专注于绘制战场的地图，寻找目标即收尾所有可用的信息，以提高攻击计划成功的可能性。攻击者通常会寻找一个脆弱点，可以是一个资产（连接到Internet上但未打补丁的服务器），也可以是一个人（公司财务部门的某位员工正在使用的老旧的windows XP），试图以此为突破口实施攻击。

可以通过子域名枚举的方式发现易受攻击的资产，发现公司暴露在公网上的IP段，通过漏洞扫描检测脆弱点，一旦发现有突破口，问题就变得复杂了。可以使用OSINT（一个开源的情报平台）描绘出员工或资产的画像。OSINT通常可以显示员工名单甚至完整的组织结构图。那么，攻击者如何从目标组织中找出最容易受到攻击的人呢？这个问题比较考验功力了。

我们经常会在行业报告中看到攻击者如何使用钓鱼攻击成功捕获受害者1号并进一步入侵公司的案例描述。钓鱼网络攻击可以向无辜者发送一个电子邮件，附件中Word文档包含了一个精心设计的VBA 宏脚本，可以加载自定义的PowerShell RAT，攻击者准备好邮件的内容措辞，期待目标可能会打开附件。这次攻击会成功吗？可能吧，也许吧，毕竟谁也无法保证，但相对于发送几百封、甚至几千封的邮件而言，攻击者更希望增加每一次攻击的成功率，毕竟如果邮件过多，会引起公司安全团队的警惕。那么该要怎么做，才能增加每次投放的成功率呢？

这里有一个简单的清单，列出了一些黑客们关注的注意事项，掌握了它们可以增加入侵成功的机率，也有利于post-exploitation阶段的操作：

  • 目标使用的是什么软件？如果正在使用的是LibreOffice而不是Microsoft Word的话，发送一个VBA宏有个毛用？
  • 目标的操作系统是什么？开发利用一个关于Windows如何解析TTF字体的漏洞，但目标使用的是Mac OS系统，那也白忙活了。
  • 目标的用户名和电子邮件地址是什么？这有助于在post-exploitation阶段站稳脚跟，同时保持隐蔽性。
  • 公司大部分文档的共享文件夹在什么位置？攻击者可以在攻破一个目标之后再继续横向运动，也可以计划一个有针对性的勒索攻击。
  • 哪些承包商在为目标公司工作？众所周知，先进的攻击者有时会选择承包商，如果这方面的安全措施不太严格的话。

现在，你会把所有这些敏感信息发布在你公司的网站上，允许任何感兴趣的人下载并使用吗？不会的？好，那么请允许我告诉您，如果您正在通过网站发布或传输一些文件，但是却没有删除这些文件的元数据，那么所有这些信息都可以在网络上被找到，我们打赌你甚至不知道它在那里，姑且将这些数据被称之为暗数据（dark data）。

公司应避免发布这些暗数据，因为这样做会给公司带来巨大的安全风险。下面的图片是一个显示敏感信息关联关系的示意图：

从安全人员的角度出发，类似上图中的信息正是安全团队应该收集的威胁情报类型（当然了，攻击者们对此更感兴趣）。从TI 供应商那里购买所有APT的情报是一件很酷的事情，不过它们的成本太高了，而且大部分的APT攻击不会出现在你的组织环境中。所以，从实际出发，了解你的对手如何感知组织的安全形势？你公司的数字足迹是什么？这是安全人员首先需要关注的东西，至少你要知道需要保护的内容是什么。

那么，该怎么做呢？请继续阅读下去，接下来将为您讲述具体的操作方法。

绘制数据足迹

本文中，假设我们是whitehouse.gov和 usa.gov站点的安全分析师，以此为例，绘制元数据泄露及相关背景调查的数字足迹，这里我们也建议您对自己的公司做同样的操作，可能您会惊讶于得到的结果。

首先，获取我们感兴趣的网站上发布的文档，这里可以用到的技术包括：

  • 对站点进行爬虫或制作镜像，下载其所有的内容；
  • 使用 Google／Bing或其他搜索引擎寻找公开的文件；
  • 条件允许的话，请贵公司的相关团队提供一份已经发布的文件的副本；

最简单、粗暴的方法就是使用wget命令制作站点镜像，将整个网站下载到本地磁盘，命令如下：

wget -mk http://www.example.com/

这样做的缺点是也会下载到HTML和其他的我们不感兴趣的内容，不过，这些无关内容可以通过使用一个Bash脚本过滤掉；

使用Google 或 Bing 等搜索引擎进行搜索的方法更有针对性，也可以使用类似 FOCA 这类信息收集工具检索文档。

site:example.comfiletype:pdf

小贴士: 文件类型搜索方面, Bing的效果优于 Google.

filetype 也可以制定为office文件、图片格式等，这样做可以避免从目标网站及其子域名的站点上下载无关的内容。另一方面，它依赖于一个搜索引擎，由于 robots.txt的缘故不也会排除到一些有趣的内容。

如果你想要实现自动化的搜索并下载这些文件，推荐一个工具给你： snitch,使用snitch时，你可以通过下列命令检索感兴趣的文件：

python2 snitch.py -C “site:whitehouse.gov filetype:pdf” -P 100

如果你想要得到一个更完整的数字足迹，建议你结合上述两种方法：爬行／镜像你的网站，并通过搜索引擎丰富你的结果。

一旦你获得了想要的文件，可以使用一个名为 exiftool 的工具从文件中提取出元数据（命令行下使用 –j参数，将结果输出为JSON格式），将输出结果倒入到 Splunk 中，你会得到一个所有数据的美丽的／可怕的仪表盘，然后就可以更好的进行分析了。

Download the bash script ，这个脚本的作用是：

  • 指定将网站内容下载到具体的文件夹，并运行 exiftool 工具提取文件中的元数据，将结果转化成JSON格式，并过滤掉无关的内容（如 HTLML或txt文件）；
  • 通过第二个参数，可以指定一个包含URL列表的文件，用于下载并分析元数据（比如一个Google搜索引擎的文档清单）

一旦你获取到了metadata.json 文件，可以将去导入到运行在Docker容器中的Splunk大数据分析平台上。

平台的搭建非常简单：

首先从官方的Docker上拉取一个DockerSplunk镜像：

docker pull splunk/splunk

然后，启动它：

docker run -d -e “SPLUNK_START_ARGS=–accept-license” -e “SPLUNK_USER=root” -p “8000:8000” splunk/splunk

Splunk在本地的8000端口运行之后，在浏览器地址栏输入：localhost:8000就可以进入初始设置（记得改变默认的admin密码）。完成之后，导入您的元数据进行分析：

  • 单击主仪表板或“设置”面板菜单上的“Add Data”；
  • 点击“Upload”；
  • 转到上传源文件的页面，可以选择保留默认设置，Splunk会自动化的检测 JSON格式的文件并自动解析；
  • 跳转到下一个页面，在索引部分创建一个新的索引，填写这一新建索引的名称，如document_metadata，其他设置可以保持不变。这里需要注意的是，这个索引名称在后边的搜索查询中还会用到，您可以根据自己的情况为它命名；
  • 确认各项内容无误，完成数据导入任务。

此时，您的数据收集工作已经完成了，接下来将进入更为有趣的数据分析和可视化部分，我们在下一篇文章中会重点描述，此时的你，可以动手做起来了，收集您公司泄露在外的元数据，并搭建好Splunk数据分析的环境。
