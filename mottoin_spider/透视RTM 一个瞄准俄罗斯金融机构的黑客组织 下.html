透视RTM：一个瞄准俄罗斯金融机构的黑客组织（下）

M0tto1n • 2017年6月13日 • 安全报告 • 阅读 236

有几个黑客组织，专门瞄准俄罗斯的企业进行网络攻击活动。最近我们发现，这些攻击活动有一个趋势，犯罪分子通常会利用简单的工具入侵目标，一旦获得访问权限，之后的工作都是手工完成的，攻击者会慢慢的了解网络架构，并针对这些受害目标的情况定制特定的用于窃取资金的工具。使用这种攻击方式的比较有名的组织有：Buhtrap, Cobalt 和Corkow。这份报告中，主要探讨一下另一个新的组织，我们称之为RTM。RTM 组织使用Delphi语言编写定制的的恶意软件，这个我们在稍后章节中会详细介绍。第一次检测到RTM的活动可以追溯到2015年，这个组织会在它们的目标系统上部署好几个定制的不同的模块。该组织主要关注远程银行系统（remote banking systems，RBS）的用户，目标范围是俄罗斯及其周边的国家。

本报告中，我们会详细介绍RTM的攻击目标、使用的工具，同时对他们正在执行的操作类型进行解析。

4.技术分析

这一部分，我们重点以SHA-1 值为AA0FA4584768CE9E16D67D8C529233E99FF1BBF0 和48BC113EC8BA20B8B80CD5D4DA92051A19D1032B的RTM样本为例，描述一下RTM银行木马的主要功能，包括持久性机制、定制的RC4算法、网络协议、间谍能力以及一些其他方面的功能。

4.1 安装和持久性

4.1.1 可执行文件

RTM恶意软件的核心是一个DLL，它被一个 .exe文件安装在磁盘上，这个可执行文件一般会被打包，其中包含DLL代码，一旦被启动，它只提取其中的DLL，并使用下列命令使之运行：

rundll32.exe “%PROGRAMDATA%\Winlogon\winlogon.lnk”,DllGetClassObject host

感染主机后不久，恶意软件就会建立它的持久性机制，根据受害者的用户权限级别，有两种不同的方式可以实现这一点：如果受害用户是管理员，恶意软件就会在HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\ registry添加一条名为“Windows Update”的记录，“Windows Update”值中存储的数据是一些命令，将在用户会话开始时被执行。

HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\Windows Update  [REG_SZ]

= rundll32.exe “%PROGRAMDATA%\winlogon.lnk”,DllGetClassObject host

它还将尝试在windows任务调度程序中添加一个任务，这项任务将启动winlogon.lnk DLL，与前边提到的导出和参数相同。

如果受害者是一个普通用户，它将在HKCU\SOFTWARE\Microsoft\Windows\ CurrentVersion\Run\ registry创建“Windows Update” 记录，使用相同的值，如下所示：

rundll32.exe “%PROGRAMDATA%\Winlogon\winlogon.lnk”,DllGetClassObject host

4.2 RC4 算法的改进

尽管RC4算法的弱点很明显，但恶意软件制作者依然经常使用这个算法。然而RTM开发者对它稍微做了一些修改，以减缓安全研究人员的分析工作。这种改进过的RC4算法被RTM恶意软件广泛采用，主要适用于加密字符串、网络数据、配置和模块。

4.2.1 不同之处

原始的RC4算法主要分为两部分：密钥调度算法（KSA）和伪随机数生成算法（PRGA）。KSA部分包括使用密钥初始化S 状态向量的值。PRGA部分使用上一步被初始化过的S 向量的值，对明文进行处理，得到密文。

在RTM恶意软件中，制作者在初始化S向量和对明文进行加密之间增加了一个步骤，在加密数据之前，S向量会和一个四字节的数值进行异或运算，这个附加的四字节的值是可变的，并且在数据加密和解密的过程中也会用到。执行这个过程的具体函数如下所示：

在我们分析的样本中，有超过25个不同的RC4 密钥，对每个字符串进行异或运算的密钥值也是不同的。分割的每个字符串的整数字段的值统一都是0xFFFFFFFF。

执行的开始阶段，恶意软件会将字符串解密成一个全局变量，当它需要访问一个字符串时，会基于一个基址和一个偏移量动态的计算出解密字符串的地址。

字符串中包含的有关恶意软件功能的有趣嘻嘻，在报告的6.8节会有一些示例。

4.3 网络

不同版本的恶意软件与C&C服务器联系的方式发生了变化。2015年10月至2016年4月期间，发现的最初的几个样本使用传统的域名服务，其托管在livejournal.com上使用RSS feed的方式更新C&C列表。从2016年4月份开始，我们观察到恶意软件的C&C服务器开始使用不断变化的.bit域名，这有利于对抗域名撤除，在4.3.2部分有进一步的解释。第一个RTM C&C 的.bit 域名（fde05d0573da.bit）在2016年3月13日被注册，这个日期也证实了我们上述推断。

所有我们检测到的C&C URLs使用了一个相同的路径：/r/z.php，由于这条路径并不常见，所以它有助于识别网络流量中的RTM请求。

4.3.2 .bit域名

最近发现的一些RTM恶意软件的制作者将C&C服务器切换到了 .bit顶级域名（TLD），这个域名不在ICANN的顶级域名列表中，而是依赖于域名币（Namecoin），Namecoin是一个基于比特币技术的分布式域名系统。尽管之前在Necurs   僵尸网络的某个版本中看到过类似的情况，但是恶意软件作者并不经常使用.bit 顶级域名，

与Bitcoin区块链不同，Namecoin区块链的用户可以存储数据，这一特定的主要使用场景之一就是 .bit 顶级域名。用户可以将注册的域名存储在Namecoin 区块链中，区块链中还包括域名进行IP地址的解析功能，因此，TLD被认为是“censorship-resistant“（抗审查的），只有注册者才能改变.bit域名的方案内容，这意味着对使用TLD的恶意域名的审查工作会更加复杂。

然而，RTM组织的恶意软件不嵌入任何需要读取Namecoin blockchain的模块，它依赖于类似dns.dot-bit.org 或 OpenNic这些DNS层的服务器去完成.bit域名的解析。因此，它与使用的DNS服务器具有相同的弹性。我们观察到C&C域名失效与博客内容紧密相关。

使用.bit TLD的另一个优势是成本低廉，注册这样一个域名，运营商只需要支付0.01 NC，相当于0.00185美元（这个数字的统计时间为2016年12月5日），相比之下注册一个.com域名至少需要花费10美元。

4.4 指纹信息

RTM恶意程序是银行木马，所以它的制作者需要采集受害者机器上的一些信息。僵尸程序收集的信息分为两部分，一方面是与操作系统属性相关的通用信息，另一方面，恶意软件会检测受害者机器上是否安装了与俄罗斯远程银行系统有关的产品。

4.4.1 通用信息

4.5 监控

当恶意软件启动时，会向C&C服务器发送各种指纹信息，包括银行软件的相关情况。只有当初始扫描工作借宿直邮，恶意软件开始监控系统时，才会出现指纹采集。

4.5.1 远程银行系统

前面讨论过的TBdo类还负责监控银行系统相关的事件，在最初的扫描过程中，使用DDE检查IE和Firefox的标签；另外一个类Tshell，这个类使用COM接口IShellWindows, iWebBrowser, DWebBrowserEvents2和 IConnectionPointContainer监控shell窗口（IE浏览器和资源管理器窗口）。当用户导航到一个新的网页时，恶意软件会收到通知，然后根据对当前URL地址和目标URL地址进行比对，如果找到匹配项，会每隔5秒连续进行6次截图，最后将截图发给C&C服务器。

它还会检查一些与银行软件相关的窗口标题，这些标题的完整列表如下图所示：

4.5.2 智能卡

RTM恶意软件能够监控连接到受害主机上的智能卡读取器，有些国家使用这些设备来验证与银行有关的订单，因此，如果这种装置连接到计算机上，它可能表明这些计算机正在执行银行交易。

然而，与其他银行木马不同，RTM恶意软件不能与这些智能卡直接交互，我们在监测到的样本中没有看到具有这类功能的模块。

4.5.3 键盘记录

记录键盘按键是监控受害者计算机的重要一环，此外RTM恶意软件不仅监控键盘，还会监控虚拟键盘和剪贴板，RTM开发人员似乎不想错过任何信息。

RTM恶意软件使用SetWindowsHookExA函数监控键盘和鼠标记录，当有事件发生时，会记录按键或对应的虚拟键盘上的按键，以及软件名和日期，然后将缓冲区信息发送到C&C服务器。

使用SetClipboardViewer函数监控剪贴板上的内容，只记录剪贴板上的文本内容。

4.5.4 截图

最后一个监控特性是截图。当shell监视类检测到感兴趣网站或检测到银行软件窗口时，就会使用截图功能。恶意软件使用GDI库完成截图功能，然后将信息发送给C&C服务器。

4.6 卸载

C&C服务器可以发送命令停止恶意软件所有活动并清理计算机。恶意软件会清除受害者主机上的恶意软件执行过程中创建的所有文件和注册表项，然后使用一个额外的加密的DLL，杀掉恶意软件并删除 winlogon 文件，最后，关闭计算机。这个DLL文件被开发者命名为erase.dll，如下图所示：

C&C服务器还可以发送一个名为“uninstall-lock” 的更具破坏性的命令。如果恶意软件拥有管理员权限，它会尝试擦除MBR，如果失败了，它会尝试将MBR转移到一个随机路径。如果成功，就按季在关机后将不能正常启动。这种情况下，可能需要重装整个系统，这样一来就可能会清除可供取证的证据。如果恶意软件的权限较低，它就会在主DLL中编写一个 .exe文件，这个 .exe 文件很小，代码只实现了关闭计算机的功能，会在注册表的HKCU\CurrentVersion\Run位置创建一个值，每次用户开始一个会话，计算机就会关闭。

4.8 其他功能

恶意软件还有许多其他的功能，下文中对它们做一下简单介绍。

4.8.1  附加模块

恶意软件可以运行其他的DLL文件的附加模块，C&C 服务器通过网络传输它们，作为外部程序运行。它们也可以映射到内存中，并以新线程的方式启动。它们存储在.dtt文件中，使用了自定义的RC4算法加密，与网络传输相同的密钥。

我们在受害者主机上检测到了VNC 模块(8966319882494077C21F66A8354E2CBCA0370464)，浏览器数据渗漏模块 (03DE8622BE6B2F75A364A275995C3411626C4D9F)和1c_2_kl 模块 (B1EE562E1F69EFC6FBA58B88753BE7D0B3E4CFAB)

对于VNC 模块，C&C服务器之后会发送一个命令，让它去链接一个VNC 服务器（某个特殊IP的44443端口）。

浏览器数据渗漏模块使用了TBrowserDataCollector类，可以读取IE浏览器历史记录，然后将URL访问记录的完整列表发送给C&C 服务器。

最后发现的1c_2_kl模块能够与1C 软件套件进行交互，此模块分为两部分：一个主DLL 和两个客户端（32 和64-bits），它们通过一个WH_CBT 事件钩子注入到每个进程中。如果客户端注入到一个进程后，会钩住CreateFile 和 WriteFile函数。

每当CreateFile函数被调用时，它在内存中的保留的文件路径也就是1c_to_kl.txt文件的路径。然后，当截获一个WriteFile 函数的调用时，它会调用真正的WriteFile 函数并发送1c_to_kl.txt 文件的路径到主DLL 模块，传输方式是使用一个定制的WM_COPYDATA 窗口消息。

最终，主DLL 打开并解析文件以识别支付订单的内容，提取文件中包括的交易的总量和数量，然后将这些信息发送给C&C 服务器。截止到目前，我们推测这个模块仍然在不断的发展，因为它包含了调试信息并且不能自动修改1c_to_kl.txt文件。

这个虚假的错误消息很容易让用户上当，同样的，这个设计与常规的windows错误消息很相似，但是上述对话框中同样存在一些拼写错误。用户点击两个选项中的人一个，都会尝试提升恶意软件的权限。

用户点击某一选项后，恶意软件使用ShellExecute函数的RunAs选项启动DLL。这将会使用管理员权限运行程序。然而，此时用户会接受到windows系统发来的一个真正的UAC提示，询问晕乎是否允许特权提升，如果允许的话，恶意软件将获得管理员权限。

4.8.3 证书

恶意软件海可以向Windows Store 添加一个证书，通过在csrss.exe进程的对话框中自动的点击“Yes” 使得添加的证书合法化。这种方式并不新鲜，Retefe银行木马也能通过自我验证来安装一个新证书。

4.8.4 反向连接

恶意软件作者使用TCP通道实现反向连接，我们还没有看到这个功能的任何操作，不过它可以被用来远程控制受感染的计算机。

4.8.5 主机文件管理

C&C服务器能够向恶意软件发送一个命令，用以修改windows 的etc/hosts文件，windows系统上的etc/hosts文件可以自定义DNS解析。

4.8.6 查找并发送文件

C&C 服务器还可以请求恶意软件查找并上传某一个文件。比如，在我们调查的过程中，接到了C&C服务器请求1c_to_kl.txt文件的命令。如前所属，这个文件是由一个名为1C: Enterprise 8的会计软件生成的。

4.8.7 升级

最后，恶意软件制作者通过发送一个新的DLL来更新恶意软件。

5.结论

我们对RTM恶意软件的研究表明，俄罗斯银行系统是该组织的攻击目标。其他组织如Buhtrap, Corkow 和 Carbanak，已经成功的从俄罗斯的金融机构及其客户处盗取了资金，而RTM组织是这个游戏的另一个参与者。

根据我们的检测，RTM恶意软件的片段早在2015年年底或更早就开始使用了，即使它并未使用一些未知的技术，但是RTM恶意软件已经实现了全方位的间谍能力，包括读取智能卡、键盘记录，并实时监控银行相关的活动，它还可以搜索会计软件（如1C: Enterprise 8）创建的导出文件。另外，RTM恶意软件的C&C 基础设施具有高度的灵活性，使用了无从追踪的.bit 顶级域名。
