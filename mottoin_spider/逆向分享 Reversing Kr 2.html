逆向分享 – Reversing.Kr (2)

Moto • 2016年8月31日 • 专题, 逆向破解 • 阅读 1933

接上一篇逆向分享 – Reversing.Kr (1)，这一篇也是Windows平台下的，在题目难度上要增加一点。

Windows

0x0A WindowsKernel

这题的验证在驱动中，首先用IDA分析应用程序，它先加载驱动，然后DeviceIoControl与驱动交互，当驱动返回1的时候就弹出Correct!

接下来分析驱动，驱动通过缓冲I/O来访问用户模式缓冲区，最终需要让0x13024处的值为1

验证的流程是先用READ_PORT_UCHAR(0x60)来获得键盘输入的scan code，这里是set 1，之后有三层来验证break code(释放按键时产生)

第一层验证过程，可以得到前四个输入值的break code为0xA5、0x92、0x95和0xB0

第二层验证过程，可以得到之后四个输入值的break code与0x12异或后为0xB2、0x85、0xA3和0x86

第三层验证过程，可以得到最后四个输入值的break code与0x12异或再与0x5异或后为0xB4、0x8F、0x8F和0xB2

这题完全可以用静态分析的方式kill，但我还是试了一下双机调试

flag:keybdinthook

0x0B AutoHotkey1

这题是需要找到DecryptKey和EXE’s Key，首先发现加了UPX壳，于是用upx -d脱壳，打开脱壳之后的程序弹出EXE corrupted

用IDA加载脱壳后的程序，搜索字符串EXE corrupted，发现有两处地方调用，往上看发现都调用了同一个函数，如果返回值不为0就显示这个字符串

然后跟进这个函数4508C7，发现有一个自校验，如果程序有改动，会通不过，这个函数通过自校验后还调用了一个函数450ABA，不知道这个函数是干嘛的。接下来用OD加载脱壳的程序，然后把调用这个函数之后的跳转指令从je改成jne

然后F9，显示Could not extract script from EXE.。重新加载跟进450C9F函数发现又有一个自校验函数450DCA，发现其中调用了好几次450ABA函数，并把函数处理之后的值与字符串比较，于是猜测这是解码函数

之后用OD加载原始的程序，用ESP定律法来到OEP，并在调用450ABA函数之后下断，发现我的猜测是对的，于是再重新加载原始程序，在跳到OEP之后，在450ABA下断看看解码出了哪些字符串，多次的F9+(Ctrl+F9)，发现在第一次的解码中看到了DecryptKey=220226394582d7117410e3c021748c2a

在最后一次解码时发现了EXE’s Key=54593f6b9413fc4ff2b4dec2da337806

MD5解密一下分别是isolated和pawn

flag:isolated pawn

0x0C CSHOP

这题是一个C#的题，打开是一个空白的对话框，用dnSpy反编译，看到new了一个按钮，但大小却是0

可以修改IL指令把size改大点，然后按钮就显示出来了，其实不改也行，由于按钮的TabIndex是0，当焦点在按钮上时，按空格键相当于点击按钮，所以打开程序后按下空格flag就出来了

flag:P4W6RP6SES

0x0D PEPassword

这题给了一个原始文件和对其用PE Password加密后的文件，首先用OD加载Packed.exe，F7一步一步跟到消息循环处

然后需要找到键盘按键的事件回调函数，先bp TranslateMessage MSG==WM_KEYDOWN，在对话框输入一个值断下后，OD中Alt+M在409000处下内存访问断点，之后一直F9就来到了事件回调函数

调用4091D8之前会把输入的值存放到esi，然后4091D8会根据输入的值做一个运算，返回值与0xE98F842A比较，如果相等就会解密数据

这里可以尝试爆破出正确的password，但想来是很有难度的，于是把4091A6处的jnz指令改成nop来到解密函数409200

解密的数据从401000开始，一次解密4个字节，在这之前又根据输入的password计算了两个值，分别放在eax和ebx，由于有原始数据和加密后的数据，因此可以得到正确的初始eax，然后就是爆破ebx，原始文件的401000处的前12个字节是81 EC 4C 01 00 00 56 57 B9 08 00 00，加密文件的401000处的前12个字节是17 2E E6 B6 05 7E 0C 0D 9E 15 C5 99，因此前三个eax的值为0xb7aac296、0x5a5a7e05、0x99c51d27，这里用C写爆破要方便一点

#include <stdio.h>

void get_eax(int *result) {
 int result_eax = 0, result_ebx = 0, input_eax = result[0], input_ebx = result[1];
 __asm {
 mov eax, input_eax
 mov cl, al
 mov ebx, input_ebx
 rol ebx, cl
 xor eax, ebx
 mov cl, bh
 ror eax, cl
 add ebx, eax
 mov result_eax, eax
 mov result_ebx, ebx
 };
 result[0] = result_eax;
 result[1] = result_ebx;
}

int main() {
 int first_eax = 0xb7aac296;
 int second_eax = 0x5a5a7e05;
 int third_eax = 0x99c51d27;
 int i = 0;
 int result[2] = { 0 };
 while (i <= 0xffffffff) {
 result[0] = first_eax;
 result[1] = i;
 get_eax(result);
 if (result[0] == second_eax) {
 get_eax(result);
 if (result[0] == third_eax) {
 printf("ebx:0x%p\n", i);
 break;
 }
 }
 i++;
 }
 return 0;
}

爆破出初始的ebx为0xc263a2cb

最后把开始解密时的eax和ebx设为0xb7aac296和0xc263a2cb

F9就出flag了

0x0E AutoHotkey2

这题是让我们修复程序，跟AutoHotkey1的程序很像，由于之前分析过，upx脱壳后直接定位到第一个校验函数4508C7，它先是计算文件的校验值，再异或0xAAAAAAAA，与文件的最后四个字节比较，校验值正确后读以文件倒数5-8个字节为地址处的16个字节与A3 48 4B BE 98 6C 4A A9 99 4C 53 0A 86 D6 48 7D比较

搜索下得到那16个字节的偏移应为00032800，因此把文件倒数5-8个字节修改为00 28 03 00，然后用OD得到正确的校验值

把最后四个字节改成33 C6 2B 36

0x11 CRC1

这是一个CRC64的题，401000函数是建表，建表时的异或值为0xc96c5795d7870f42

然后401070函数计算输入的8个字符的crc64值，最后与0x676F5F675F695F6C比较

最笨的办法就是计算(127-92)^8次crc64值与0x676F5F675F695F6C比较，但这样感觉要很久，仔细看算法可以发现crc是可以倒着算的，首先建表时

int init_crctable() {
unsigned long long t = 0, b = 0;
int i = 0, j = 0;
for (i = 0; i < 256; i++) {
t = i;
for (j = 0; j < 8; j++) {
b = t & 1;
t = t >> 1;
if (b) {
t ^= 0xc96c5795d7870f42;
}
}
crctable[i] = t;
uncrctalbe[i] = t >> 56;
}
return 0;
}

然后通过下面的代码就可以往回算

int find_index(unsigned long long crc) {
int i = 0;
unsigned long long find_crc = crc >> 56;
for (i = 0; i < 256; i++) {
if (find_crc == uncrctalbe[i]) {
return i;
}
}
return -1;
}

unsigned long long uncrc(unsigned char ch, unsigned long long crc) {
int index;
index = find_index(crc);
crc ^= crctable[index];
crc = crc << 8;
crc += (ch ^ index);
return crc;
}

unsigned long long get_uncrc(unsigned char *uncrc_data, int start, int end, unsigned long long crc) {
int i;
for (i = start; i >= end; i–) {
crc = uncrc(uncrc_data[i], crc);
}
return crc;
}

每部分生成8100多万个值，由于内存较小用map分块遍历来找到两部分中相等的那个值

#include <utility>

using namespace std;

map<unsigned long long, bool> crc_value;

void read_file() {
char crc_data[240000000] = { 0 };
FILE *file;
size_t read_size;
int n, count = 0;
unsigned long long crc;
file = fopen(“first4.txt”, “r”);
fseek(file, 3 * 240000000, SEEK_SET);
read_size = fread(crc_data, 1, 240000000, file);
n = read_size / 24;
for (int i = 0; i < n; i++) {
if (count % 1000000 == 0) {
printf(“current:%d\n”, count);
}
*(crc_data + i * 24 + 18) = 0;
sscanf(crc_data + i * 24, “0x%016llx”, &crc);
printf(“0x%016llx\n”, crc);
crc_value.insert(pair<unsigned long long, bool>(crc, true));
count++;
}
printf(“%s\n”, “read_end”);
fclose(file);
}

void crc_find() {
char crc_data[240000000] = { 0 };
FILE *file;
size_t read_size;
int n, count = 0;
unsigned long long crc;
file = fopen(“last4.txt”, “r”);
while (read_size = fread(crc_data, 1, 240000000, file)) {
n = read_size / 24;
for (int i = 0; i < n; i++) {
if (count % 1000000 == 0) {
printf(“find-current:%d\n”, count);
}
*(crc_data + i * 24 + 18) = 0;
sscanf(crc_data + i * 24, “0x%016llx”, &crc);
printf(“0x%016llx\n”, crc);
if (crc_value.count(crc) >= 1) {
printf(“0x%016llx\n”, crc);
return;
}
count++;
}
}
fclose(file);
}

void main() {
read_file();
crc_find();
}
