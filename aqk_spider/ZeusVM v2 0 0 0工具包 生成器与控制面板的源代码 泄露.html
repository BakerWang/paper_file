ZeusVM v2.0.0.0工具包(生成器与控制面板的源代码)泄露

2015-07-09 15:44:56 阅读：13317次 点赞(0) 收藏 来源： 安全客

     背景

     KINS(更准确地说应该是ZeusVM) v2.0.0.0工具包(生成器与控制面板的源代码)被泄漏了出来，并在网络上被疯传。在2015年6月26日，我们得知(感谢Xylit0l)了这个事件，并且经过多次的内部讨论之后，我们认为：“毕竟知道此次泄漏事件的还是坏人居多...”。今天，我们决定通过这篇文章来提高公众对于此类事件的警惕。所以，这篇文章是为了给那些至今还不知道此次工具包泄漏事件的人传递一条信息，或者提醒那些想要保留从可信任来源处泄漏出来的数据的人。

     在这篇文章中，除了警告用户之外我们还想告诉大家的是，KINS v3目前正在黑市上出售，售价在5000左右。

     这篇文章的完成需要的是团队合作，一同参与测试的还有我的两个朋友，Xylit0l[链接]和unixfreaxjp[链接]，他们正在编写报告，进行测试，记录数据，进行截图，而且也在尽可能地将那些扩散了这个泄漏工具包的网站下线。

     被泄漏的到底是什么？

     被泄漏出来的实际上并不是KINS v2.0.0.0版本的源代码，而是KINS v2.0.0.0中生成器的二进制源码，详情请看下面的截图(截图由Xylit0l截取)：

     而且，控制面板的源代码可以在本文所带的视频中看到。在下面的图片(截图由Xylit0l截取)中显示的是使用了web面板的KINS，这是一种典型的ZeuS僵尸控制面板。

     所有发布的图片和信息都是MMD的研究材料，有法律免责声明à[链接]。

     即使程序员使用的名称是KINS 2.0.0.0，但是该工具的生成器配置文件以及生成器生成的二进制代码都是来自于ZeusVM(2.0.0.0版本)的，而且这些数据都完全不同于之前版本的KINS工具包，详情可以查看链接。

     在之前所发布的KINS版本中，并没有使用隐写术(通过将某些消息嵌入至其他看起来没有意义的数据中来隐藏信息的一种方式，在这个情况下，ZeusVM将配置代码嵌入了JPG文件之中)，但是这个版本使用了隐写术，而且生成的ZeusVM bot二进制代码与ZeusVM v1和v2版本中的样例代码(代码可以在内核模式代码库中找到)非常相似。这也就是我们使用ZeusVM作为这篇文章标题的原因，有证据显示，KINS在他们的恶意软件中采用了ZeusVM技术。如果你不了解ZeuS恶意软件的发展历程，那么你也许会感到有些迷惑，但是目前已经发现的东西以及专家分析的结果都是既定的事实了。

     解释的已经够多了，下面向大家展示这个工具包！

     隐写术的概念验证是什么？

     下面的视频向大家演示了KINS/ZeusVM 2.0.0.0中所使用的隐写术技术。此次演示向大家展示了生成器可以将KINS 2/ZeusVM2中经过加密的配置文件代码注入到任意的JPG文件中，这样一来，这种JPG文件就可以伪装成生成器的配置文件了。我们使用十六进制编辑器对注入代码数据“之前”和“之后”的JPG文件进行了详细的比对。为了设计出针对其模版配置的正确注入测试方案，Xylit0l与我讨论了很多…最终我们成功了。在这里我要向大家道歉，因为我们将这个恶意软件大卸八块了，而且也并没有使用它们。

     那么，这意味着什么呢？

(1)         我们将会在我们的互联网中看到更多ZeusVM(2.0.0.0版本)僵尸网络的活动，因为这个恶意软件以及配置文件生成器已经像空气一样，人人都可以获得，而且现在也变成公开的了。不仅普通的网络犯罪分子可以操作这种僵尸网络，而且任何得到了这个工具包的人都可以生成ZeusVM 2.0.0.0的二进制代码，并且通过其控制面板来设置僵尸网络；

(2)         而且，我们很快便会看到一个新版本的KINS/ZeusVM(v3)问世。

这对于安全社区来说是一条非常重要的信息。该工具包目前正在被广泛地传播和散布，所以我们必须非常努力地将某些泄漏了该工具包的下载站点一个一个地下线，如果不这样做的话，事态就会发展到一个不可收拾的地步了。请您帮助我们，不要在网络上传播和发布这个工具包。所有的反恶意软件以及所有威胁过滤行业的公司最好了解这个工具包的详细信息，并尝试申请得到这个泄漏出来的代码存档，这样他们就可以开始研究如何阻止和缓解由该工具包所产生的安全威胁了。如果你现在还没有得到这个工具包和生成器，你可以在我们的网站中获取到，或者从VirusTotal等其他受信任的网站获取。
