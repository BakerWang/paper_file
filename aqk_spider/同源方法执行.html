同源方法执行

2015-06-19 14:55:23 阅读：12299次 点赞(0) 收藏 来源： 360安全播报

该文章是对"Same Origin Method Execution" (SOME)的简要陈述。（SOME）是web应用程序攻击方法中的一种，它通过强制受害者在endpoints的域上执行任意页面的脚本方法来滥用回调endpoints(主要是 Flash applets和OAuth 会话的JSONP endpoints—通常重定向到- redirect_uri)。SOME的影响类似于XSS。但是有一些重要且有区别的利用方法。虽然有局限性，但是对特定web功能/网页来说也不限制UI or HTTP 响应头（很重要且有效）。

事实上，使用只有文字数字 字符和一个点可让攻击者劫持危险的web函数甚至除去敏感的用户数据，如私人相片和/或视频。受欢迎的域如Google, Yahoo, Microsoft (plus.google.com, maps.yahoo.com, yammer.com and so on)和非常流行的平台Wordpress和VideoJS（它将多数有漏洞的域转向SOME），其受SOME的影响。当然最近许多问题都已被修复了。（披露的响应细节如下）

BlackHat大会的Paper, 示例和课件

如果你希望深入研究技术细节，你可以读我的white-paper，或查看最新的Black-Hat陈述。该课件中含有SOME利用的示范视频-你可以看到我曾经使用SOME来劫持Google云的私人相片及视频的方法。

顺便一提，你也可以找到更多关于侵害的细节,其在我之前的文章中提到过：Stealing private photo albums from Google

攻击场景：

我们首先需要理解SOME攻击来涵盖相关知识，请支持我：

同源方法执行通过强制建立windows/frames来滥用user agents，依次重定向他们的文档。通过在callback内构造参数来使用引用将可重置目标文档的执行上下文并劫持一些其内部存在的web功能。

处理方法执行的Surface起始步骤：

首先攻击者不得不检测某个有漏洞的例子一个插件或一个弱点callback  endpoint文档。一个有弱点的范例是通过参数值应用某个函数名而导致浏览器的解析器执行。

笔记：响应消极的mime类型范例如当直接访问时应用/json是没有漏洞的。

navigation or a direct reference pointing to it, for example: 攻击者必须在相同域上的web页面选择目标。针对劫持目标页面的web功能（对象方法，javascript函数），攻击着应该使用DOM导航或一指向它的直接引用，例如：document.body.privateAlbum.firstChild.nextElementSibling.submit

创建exploit程序：

1.创建SOME并创建窗口引用，它不得不通过打开新的浏览上下文创建一利用程序的Surface/环境。

一旦准备好了环境，初始页面（MAIN）应该将它的文档重定向到在endpoint的域(http://www.vulnerable-domain.com/target_document)上任意需要的目标页面

随着重定向的进行，新的浏览上下文应该等待目标文档的DOM加载完成。

一旦准备就绪，由于劫持方法执行，新浏览上下文将把其文档重定向到弱点范例（用某随意的callback参数设置），例如：http://www.vulnerable-domain.com/flash-plugin.swf?callback=opener.document.body.privateAlbum.firstChild.nextElementSibling.submit

用于执行某个Alart的指定执行上下文的范例：

PoC:

@Main Page:

<script>
function startSOME() {
 window.open("step1.html");
 location.replace("http://www.vulnerable-domain.com/privateAlbum");
}
document.body.addEventListener("click",startSOME); //Popup Blocker trick
</script>

@step1.html:

<script>
function waitForDOM() {
      location.replace("http://www.vulnerable-domain.com/flash-plugin.swf?callback=opener.document.body.privateAlbum.firstChild.nextElementSibling.submit");    
}
setTimeout(waitForDOM,3000);
</script>

缓解及修复

Static Callbacks - 利用同源方法执行依赖滥用callback参数，许多web应用确实可以持续存在一样的的功能（没有动态设置callback）。因此，可应用时，网站应该使用修复后的callback值。以对抗外部的callback控制。

White-list approach 案例，每个endpoint支持设计的web应用多于一个单一的callback，或二者选一，维持高度灵活性很重要（在Flash Plugin中通用），它更好地设置白名单并匹配给出的callback参数值来对抗它。这将强制识别合法callback函数其可以执行UPDATE：我在一个JavaScript防御库上工作，其针对通用并简单的部署方案服务。

Cross-Domain Messaging –使用postMessage是为了通知事件并与跨域事件作为javascript callback执行交互（如果可应用）。

响应披露

Oct 30, 2014: Google已修复的范例– Google Plus.

Nov 06, 2014: Microsoft已修复的范例– Yammer.

April 21, 2015: Wordpress 已修复的范例– Plupload.

在infosec社区，Callback利用问题众所周知，即使最近的攻击事件没有得到足够的关注。因为网站持有者的一些原因，包括player们，仍然依赖于开发者的习惯和经验。更重要的是，现在“没有最好的练习”或被公开出来的标准。因此我已选择在BlackHat会议上流放一些SOME和callback endponits的相关提示。由顶级研究者Aleksandr Dobkin和Roman Shafigullin发现了SOME的多方面影响。他们在这都值得信任。

Google Bug Bounty认为该攻击的影响类似于XSS bug带来的问题。

Microsoft Bug Bounty认为该攻击的影响类似于XSS bug带来的问题。

Wordpress在安全相关问题中已将这类”XSS”攻击(Apr. 2015 - WordPress 3.9-4.1.1 - Same-Origin Method Execution)分类

同源方法执行可导致其目标出现严重的后果，记住该种攻击的风险和XSS一样高，其在top 10 OWaSP项目中风险排名第三-那是XSS。总结，它对于抛弃错误观点来说相当重要，狭隘的字符集以某种方式担保你的应用的安全性，更重要的是我们应该让开发者更多地关注SOME问题。
