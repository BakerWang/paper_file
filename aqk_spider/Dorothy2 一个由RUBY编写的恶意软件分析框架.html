Dorothy2－一个由RUBY编写的恶意软件分析框架

2015-06-25 16:50:26 阅读：6412次 点赞(0) 收藏 来源： 360安全播报

框架介绍

Dorothy2是一个用于分析可疑二进制代码的框架。它的特点在于灵活的模块化环境以及网络分析研究框架的交互性。除此之外，它还可以通过对比进程创建的基准线来识别新的进程。静态代码分析以及新型系统行为分析等功能将很快在下一版本中加入。Dorothy2使用预配置的分析机制来分析恶意代码。代码分析需要以下几点确定因素：

l  确定的沙盒操作系统类型

l  确定的沙盒操作系统版本

l  确定的沙盒操作系统语言

l  修复分析超时(重置虚拟机需要等待的时间)

l  所需的截图数量(以及截图间隔)

l  支持的插件名单，以及操作系统的执行方式。

框架的使用简介帮助研究人员能够对不同使用环境下的代码集合进行分析。众所周知，某些恶意软件被设置为只在某种特定环境下运行。CSIRT可能只会在某一客户的某一特定环境下使用它们来测试可疑的恶意软件。

我在2009年2月公布了Dorothy项目，这是我完成学士学位所做的最后一个项目，而 Dorothy2正是这个项目的延续。你可以在意大利的Honeyproject网站上找到关于整个项目的更多信息。

整个框架主要由5个模块组成，它们甚至可以分开单独运行。下面的图片显示了框架中各个模块的整体架构，以及它们之间是如何进行链接的。

二进制代码读取模块(BFM)

该模块主要负责从配置源中获取二进制代码。通常情况下，一个“二进制源”可能是系统文件夹，一个电子邮箱，或者是一个使用了SSH的主机。一旦获得了二进制代码，BFM模块将会把其加入分析队列。

Dorothy的分析引擎

该模块负责对队列中的数据进行分析，分析主要通过在沙盒中执行预设的二进制代码，然后存储生成的网络流量数据，最后将截图存入分析文件夹中。

(网络)数据提取模块

该模块负责研究和分析pcaps文件，并将相关信息(流数据，IP的地理信息等等)存入Dorothive中。除此之外，它还会提取沙盒通过HTTP／HTTPS所下载的所有文件并将这些文件存储至二进制文件的分析文件夹中。

伪造的Webgui界面

一个伪造的Sinatra应用程序将会为所有需要的数据提供交互式概览。警告：这个模块应当在一个受控环境中运行。作者强烈建议，不要将这个模块暴露在互联网中。

这个ppt为大家提供了我们所使用的僵尸渗透网络模块的概述。

根据GPL 3通用开源许可证的条款，前四个模块公开发布在了密网项目联盟上。所有由该框架所生成的数据－例如二进制信息，时间戳，网络分析详情－都被保存在了一个postgres数据库中，以便未来分析使用。

我于2009年底开始编写这个项目，与此同时，我也在学习Ruby语言。从那时起，随着我的Ruby技术不断提高，我便同时对该项目进行了修改和提升。正因为如此，你可能会发现某些代码并不是很整洁。

要求

     Dorothy的运行需要下列软件的支持：

VMWare ESX >= 5.0 (提示: 如果你已经下载了ESXi, 你可以对ESX进行评价)

Ruby 1.9.3

Postgres >= 9.0

至少一台Windows虚拟机

一台用于运行网络数据分析引擎的类Unix设备 (NAM) (需要tcpdump/ssh)

pcapr-local (仅有doroParser会使用它)

MaxMind libraries (仅有doroParser会使用它)

关于操作系统

Dorothy被设计运行在任何一种*nix系统中。目前，它已经在OSX和Linux平台上测试成功了。

在Windows平台上使用虚拟机作为沙盒(已在XP系统下测试成功)。

只有pcapr-local严格要求使用Linux，如果你想要使用一台Mac来运行这个框架(与我一样)，你可以将它安装在NAM中(正如这篇文章建议的那样)。

安装

我建议大家按着下列步骤一步一步地完成：

1.   设置你的ESX环境

2.   安装框架所需的软件

3.   安装Dorothy和libmagic库

4.   开启Dorothy，并配置它

5.   使用Dorothy

一、    设置你的ESX环境

1.   基本配置(ssh)

Configuration->Security Profile->Services->Proprieties->SSH->Options->Start and Stop with host->Start->OK

2.   配置两个单独的虚拟网络，其中一个为沙盒专用的网络。

3.   配置沙盒所需的Windows虚拟机

l  禁用Windows防火墙

l  Windows访客系统必须安装VMware Tools

l  访客系统中一切都设置完之后，通过vSphere控制台为沙盒虚拟机创建备份。在运行完二进制代码之后，Dorothy将会使用它来恢复虚拟系统。

4.   创建一个NAM所需的unix虚拟机

l  安装tcpdump和sudo

#apt-get install tcpdump sudo

l  为dorothy创建专属用户(例如”dorothy”)

#useradd dorothy

l  在dorothy用户的主目录中创建一个目录用于存放网络数据

#su dorothy
$mkdir /home/dorothy/pcaps

l  为dorothy的用户添加执行和结束tcpdump的    权限：

#visudo
add the following line:
dorothy  ALL = NOPASSWD: /usr/sbin/tcpdump, /bin/kill, /usr/bin/killall

l  如果你正在为Linux设备安装Dorothy，我建议你安装同时再这个设备上安装pcapr。

#apt-get install ruby1.9.3 rubygems  tshark zip couchdb

l  开启couchdb服务器

#/etc/init.d/couchdb start

l  安装pcapr-local

#gem install pcapr-local

l  使用dorothy的系统账号开启pcapr-local，并对其进行配置。当出现提示时，插入下列文件夹路径来存储网络数据。

$startpcapr
....
Which directory would you like to scan for indexable pcaps? [/root/pcapr.Local/pcaps]
/home/dorothy/pcaps

除此之外，请记住允许pcapr在所有的端口运行

What IP address should pcapr.Local run on? Use 0.0.0.0 to listen on all interfaces [127.0.0.1]    0.0.0.0

l  如果一切进行顺利，你应该能够访问下列网址

http//{ip-used-by-NAM}:8000

安装演示

1．      基本安装(强烈建议)

2.高级安装

如果你需要将Dorothy安装在多环境中，建议您进行高级安装。利用一个私人的VPN网络，多网络环境的所有沙盒流量都会存在于一个内部IP地址中。

二、    安装框架所需的软件

1.   安装postgres数据库

$sudo apt-get install postgresql-9.1

或者

http://www.postgresql.org/download/

2.   为Dorothy配置一个专用的postgres用户(或者使用默认的postgres用户作为代替，这完全取决于你)。

3.   安装下列软件包

$sudo apt-get install ruby1.9.3 rubygems postgresql-server-dev-9.1 libxml2-dev  libxslt1-dev libmagic-dev

对于OSX用户：mac的端口支持上述所有的软件。Libmagic的提示：使用brew：

$ brew install libmagic
$ brew link libmagic

如果你想要安装pcapr：

$sudo apt-get install tshark zip couchdb

三、    安装Dorothy和libmagic库

安装Dorothy gem

$ sudo gem install dorothy2

四、    开启Dorothy，并配置它

1.   安装MaxMind库

l  GeoLiteCity

l  GeoLite ASN

l  将GeoLiteCity.dat和GeoIPASNum.dat复制到Dorothy的etc/geo/文件夹

2.   运行Dorothy

$ dorothy_start –v

应当出现下列提示信息

[警告] Dorothy的配置文件不存在, 为了创建配置文件，请您回答下面几个问题。

3.   跟随提示完成配置

l  环境变量(db,esx服务器等等)

l  Dorothy源(得到新二进制代码的地方)

l  用于分析的ESX虚拟机

当你首次运行Dorothy时，它将会要求你填写这些问题，然后在etc/文件夹中创建系统所需的配置文件。但是，你也可以直接对这些文件进行修改和创建，你也可以在这个文件夹中找到配置的演示文件。

五、    使用Dorothy

1.复制一个.exe文件或.bat文件至$yourdorothyhome/opt/bins/manual/中。

2.  执行dorothy：

$ dorothy_start -v -s malwarefolder

六、    问题调试

我清楚地知道Dorothy的安装并非一件容易的事情。所以下面我为大家提供一些理解程序崩溃信息的提示。

1.        当你运行Dorothy 时，设置verbose标志(-v)，或者在进行额外调试时，设置-debug标志。

$dorothy_start -v -d -s malwarefolder

2.        如果发生了任何错误，你可以错误信息提交给我！

3.        在googlegroups.com上就dorothy2进行提问

致谢

感谢所有对Dorothy2项目作出贡献的人们。
