基于云的数据保护并不理想

2014-10-24 15:05:17 阅读：4188次 点赞(0) 收藏 来源： Brien Posey

Brien M. Posey是一位微软认证系统工程师，曾获微软Exchange Server、Windows Server 和Internet Information Server (IIS)的MVP奖。Brien担任一个全国连锁性医院的CIO，曾经在Fort Knox负责信息管理系。

基于云的数据保护有哪五大劣势?

在过去几年，使用云提供数据保护流行开来。但即便如此，基于云的数据保护也并非那么尽如人意。以下就是它的五大劣势方面：

1.数据安全

人们提出的最首要的异议大概就是云存储的安全漏洞了。如果数据存储在云中，在你的数据中心之外，并且你没有对数据保护的直接控制。这些都极大的增加了安全漏洞导致的云数据泄露的可能性。

2.隐私

另外一个拒绝基于云的数据保护的原因就是对于隐私数据的重视。毕竟，无法考证提供商是否会读取、甚至利用你的数据。不过至少出现过一起已证实的云存储提供商采集客户电子邮件地址数据的事故。

3.持续的成本

还必须考虑的一个重要因素就是持续上升的成本。云服务提供商一般基于消费空间大小，客户产生的I/O负载计费。这些费用将没有终结。如果一个组织将一份文件拷贝到云，那么云提供商将月复一月的为文件所占空间计费。

4.长久的恢复时间

另外一个为人诟病的方面就是基于云的灾难恢复时间。平心而论，确实有供应商提供即时恢复能力。许多组织都采用混合云，以一台现场设备恢复云中数据的方式实现。然而，如果恢复不得不从云中进行，那由带宽限制带来的结果将缓慢无比。

5.云提供商潜在单点故障

如果云提供商存在单点故障的话，基于云的数据保护就并不理想。云提供商很有可能使用冗余的硬件保护数据，但一旦提供商服务中断会发生什么?如果你计划采用云数据保护，那么不要犯集中式错误。使用云作为补充方案，或将数据存储于多个云。
