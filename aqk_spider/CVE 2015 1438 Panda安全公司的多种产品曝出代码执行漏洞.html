CVE-2015-1438 Panda安全公司的多种产品曝出代码执行漏洞

2015-07-06 15:05:40 阅读：7770次 点赞(0) 收藏 来源： 安全客

    

    CVE:CVE-2015-1438

     供应商:Panda安全公司

     产品:多种产品

     受影响版本:1.0.0.13(PSKMAD.sys驱动程序版本)

     修复版本:15.1.0(产品版本)

     漏洞报告人:Kyriakos Economou

     漏洞详情：

     对于复制到已分配空间的内核分页缓冲池和非分页缓冲池的数据，Panda公司安全产品的内核内存访问驱动程序无法对这些数据的大小进行验证。除此之外，攻击者已经可以控制非分页缓冲池的start-to-copy索引了，这样一来，攻击者就可以更加精确地对内核对象进行攻击了，而且他们还可以通过劫持程序的函数指针来控制EIP。

     技术细节：

b5ae8cc5 8b7508 mov esi,dword ptr [ebp+8]<-- ESI输入缓冲区

b5ae8cc8 0fb7460c movzx eax,word ptr [esi+0Ch] <-- EAX读取分页缓冲池中的数据大小，并依此来分配输入缓冲区

b5ae8ccc 6685c0 test ax,ax

b5ae8ccf 7674 jbe PSKMAD_b5ae4000+0x4d45 (b5ae8d45)

b5ae8cd1 668b4e0e mov cx,word ptr [esi+0Eh]

b5ae8cd5 668945f4 mov word ptr [ebp-0Ch],ax

b5ae8cd9 0fb7c0 movzx eax,ax

b5ae8cdc 6850534d45 push 454D5350h <-- 'PSME' 池标记

b5ae8ce1 50 push eax <-- 所分配的分页缓冲池大小

b5ae8ce2 6a01 push 1 <-- 指示分页池

b5ae8ce4 66894df6 mov word ptr [ebp-0Ah],cx

b5ae8ce8 ff159ca6aeb5 call dword ptr [PSKMAD_b5ae4000+0x669c]={nt!ExAllocatePoolWithTag}

b5ae8cee 8945f8 mov dword ptr [ebp-8],eax

b5ae8cf1 85c0 test eax,eax

b5ae8cf3 7550 jne PSKMAD_b5ae4000+0x4d45 (b5ae8d45)

...

b5ae8d45 8d4608 lea eax,[esi+8]

b5ae8d48 50 push eax <--指向的是输入缓冲区中存储的返回的符号链接目标长度

b5ae8d49 8d45f4 lea eax,[ebp-0Ch]

b5ae8d4c 50 push eax

b5ae8d4d ff36 push dword ptr [esi] <-- 处理符号链接对象

b5ae8d4f ff1514a7aeb5 call dword ptr [PSKMAD_b5ae4000+0x6714]={nt!ZwQuerySymbolicLinkObject} <--- 引起堆溢出，因为我们所控制的分页缓冲池大小已经分配完成了

b5ae8d55 894604 mov dword ptr [esi+4],eax

b5ae8d58 85c0 test eax,eax

b5ae8d5a 7d09 jge PSKMAD_b5ae4000+0x4d65 (b5ae8d65)

...

b5ae8d65 0fb745f4 movzx eax,word ptr [ebp-0Ch]

b5ae8d69 50 push eax <-- 符号链接目标的EAX 长度

b5ae8d6a 8b4610 mov eax,dword ptr [esi+10h] <-- EAX索引复制了符号链接目标，我们的输入缓冲区同样读取了这些数据

b5ae8d6d ff75f8 push dword ptr [ebp-8] <-- 分页缓冲池中符号链接目标的存储地址

b5ae8d70 03c6 add eax,esi <-- ESI指向我们的输入缓冲池(非分页缓冲池).

b5ae8d72 50 push eax <-- 这样一来，我们便控制了全部的地址，这些地址是符号链接目标复制的起始地址，它们会按缓冲区的低地址到高地址，将数据复制到缓冲区中

b5ae8d73 e864120000 call PSKMAD_b5ae4000+0x5fdc (b5ae9fdc) <-- jmp to {nt!memcpy} <--- 引起堆溢出，因为在调用DeviceIoControl 编程接口的过程中，我们可以控制非分页缓冲池所分配的起始复制地址。

     影响:

     攻击者可以利用上述的两个漏洞来出发系统的堆溢出，这样攻击者便可以使用内核权限来执行任意的代码了。然而，第二个堆溢出漏洞则显得更加的有意思，因为这个漏洞可以允许攻击者通过调用nt!ZwQuerySymbolicLinkObject内核函数，从而精确地控制符号链接目标起始复制的索引列表了。

     这样，攻击者便可以精确地对内核对象进行攻击了，通过对非分页内核缓冲池执行堆喷射，攻击者便可以控制这些对象了。攻击者还可以使用一部分符号链接目标作为指向用户内存空间的指针。
