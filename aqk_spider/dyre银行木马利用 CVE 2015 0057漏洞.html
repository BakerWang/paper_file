dyre银行木马利用 CVE-2015-0057漏洞

2015-07-13 10:54:01 阅读：10133次 点赞(0) 收藏 来源： 360安全播报

在Dyre银行木马的新变种正在利用微软的Windows（CVE-2015-0057）一个修补漏洞。我们首次观察到这些新的变种在6月17日，而在4个多月后，微软发布了补丁程序（MS15-010）。

CVE-2015-0057是一个释放后使用漏洞自由存在于Windows内核，可以被利用来执行本地权限提升的Win32k.sys中的组成部分。该漏洞被报告给微软通过Udi Yavo，以及补丁发布之后，Udi Yavo撰写一个简短博客，讲述漏洞的一些细节。然而，值得注意的是，对于CVE-2015-0057的攻击代码尚未被公开是很重要的。

除此之外,这些新变种Dyre还包括利用cve - 2013 - 3660作为后备,以防系统打补丁的cve - 2015 - 0057。

初步验证

首先，恶意软件首先检查它的权限级别。如果它已经具有管理员权限，它解密使用单字节XOR解密嵌入的资源FILE1。解密后的二进制然后被写入到文件系统并执行。

如果恶意软件不具有管理员权限，则进入到执行以下序列：

1.如果NT版本为5.x（Windows XP中/在Windows 2000 / Windows 2003中），它扫描模式“3036220”下面的注册表路径：

  SOFTWARE\Microsoft\Updates\Windows XP\SP0
    SOFTWARE\Microsoft\Updates\Windows XP\SP10
    SOFTWARE\Microsoft\Updates\Windows XP\SP3
    SOFTWARE\Microsoft\Updates\Windows XP\SP4

这个模式被用于检测是否Microsoft更新（KB3036220或MS-15-010）安装，因此CVE-2015-0057被修补。在这种情况下，Dyre检查，如果CVE-2013-3660修补以及通过扫描注册表中的：“KB2850851”，它对应于用于CVE-2013-3660的安全更新。它看起来这种模式中相同的注册表的路径，如上所述。如果不打补丁，那么攻击代码对应的CVE-2013-3660用于特权升级。

2.如果NT版本是6.0或6.1(Windows Vista和Windows 7 / Windows Server 2008),然后继续扫描下面的注册表路径模式“3036220”:

SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\Packages.

现在,让我们深入讨论这个二进制如何利用漏洞从cve - 2015 - 0057。

Exploitation of CVE-2015-0057

函数的主要弱点是:win32k !xxxEnableWndSBArrows用于管理窗口滚动条。在xxxEnableWndSBArrows易受攻击的代码修改其用于由利用修改其他关键数据结构先前分配tagPROPLIST结构的cEntries字段。

下面的步骤使用Dyre利用use-after-free脆弱性存在于win32k ! xxxEnableWndSBArrows:

1.通过调用子程序WIN32K分配大量的连续的内核内存！CreateProp（创建一个特定大小的tagPROPLIST结构）。

2.释放一些tagPROPLIST结构选择性，创建内存漏洞落入脆弱的结构。

3.通过调用子程序WIN32K！_InitPwSB分配SBINFO结构。 tagPROPLIST大小可以从用户模式进行控制。

攻击者设定配置tagPROPLIST的大小，SBINFO的大小。调用_InitPwSB后，一个SBINFO结构将落入一个漏洞在内存中创建步骤2。

4.在函数内,win32k !xxxEnableWndSBArrows,常规xxxDrawScrollBar启动一个用户模式的回调函数。

a）在用户模式回调（USER32！__ ClientLoadLibrary）的过程中，攻击者破坏了Windows ，从而释放了内存块。

b）之前返回到xxxEnableWndSBArrows（在内核模式），则攻击者将调用WIN32K！NtUserSetProp（在用户模式）。这一行动将占据先前与新tagPROPLIST结构释放内核内存，取代了以前分配SBINFO结构。

5.当用户模式回调返回到内核，例程WIN32K！xxxEnableWndSBArrows将使tagPROPLIST.cEntries field的变形（或指令）。这将增加tagPROPLIST.aprop数组的大小。该漏洞主要存在这里所使用的漏洞，在接下来的步骤。

图1示出tagPROPLIST结构中WIN32K的cEntries字段的修改xxxEnableWndSBArrows！

图1：修改tagPROPLIST.cEntries field

6.使用这种损坏tagPROPLIST对象,攻击者能够实现任意相对内存的写入。这种原始利用覆盖一个值在一个关键的GDI对象。

7.最后，攻击者调用它使用上述损坏GDI对象Win32k.sys中的函数，这导致的绝对地址的任意存储器写入。

8.攻击者然后能够修改内核函数指针指向他的在用户模式的shellcode缓冲器，导致特权升级。

图2总结了使用后释放内存覆盖和任意内存的序列中写道：

图2：任意内存的顺序写入

用于重定向控制流程以用户模式的shellcode和升级的特权的技术如下：

1. HalDispatchTable+为0x4将被覆盖在攻击中提到的上述步骤7。

图3示出的精确点是HalDispatchTable+为0x4被覆盖的用户模式的shellcode的地址。

图3：HalDispatchTable+为0x4覆盖和用户模式的Shellcode

2.从用户模式，它调用的API，ZwQueryIntervalProfile。在内核模式下，这个控制权转移给HalDispatchTable+ X04。因为在这个位置的值被覆盖用户模式的shellcode，控制转移到特权升级的shellcode。

这shellcode的使用复制系统进程的EPROCESS结构的令牌领域并将其存储在目标进程的EPROCESS结构的标准方法。以下步骤描述了这种方法：

1.呼叫PsLookupProcessByProcessId得到的指针到两个受害者的过程和系统进程的EPROCESS结构。

2.调用PsReferencePrimaryToken得到两个进程的标记。

3.遍历受害者进程的EPROCESS结构查找偏移，其中令牌被存储。

4.存储在这个偏移量提升自身的权限系统进程令牌。

类似的shellcode被用在这个二进制文件利用CVE-2013-3660（仅当安装了CVE-2015-0057的安全更新中使用），但是，我们观察到以下差异这表明，这些漏洞最有可能写的2个不同的作者或重新使用来自其他地方。

CVE-2015-0057:

它获取使用PsLookUpProcessByProcessId系统进程的EPROCESS结构。

它采用的为0x200迭代计数器扫描EPROCESS结构令牌偏移。

CVE-2013-3660:

它获取使用PsInitialSystemProcess系统进程的EPROCESS结构。

它采用的是0x300迭代计数器扫描EPROCESS结构令牌偏移。

Kernel Mode Protections

要注意，这利用不会经旁路绕过SMEP，因为它从内核模式执行用户模式回调如上所述是重要的。因此，Windows 8.1不受它的影响。

对于Windows NT的用户5.x中，WIN NT6.0和Windows NT6.1，宜安装更新KB3036220，因为这个可以随时被利用。

这种利用技术指标在用户模式

下面是一些这个指标漏洞在用户模式下：

1.调用SetPropA/ RemovePropA API的长序列。

2.创建的窗口在一个循环数组和设置使用SetPropA每个窗口的属性列表中。

3.从上面创建阵列摧毁几个窗口创建漏洞。

4.调用EnableScrollBar参数，如下所示：

EnableScrollBar(hWnd, SB_BOTH, ESB_DISABLE_BOTH);

5.调用ZwQueryIntervalProfile执行实际权限提升。

Conclusion

列入这一最新的特权升级方法都是利用Dyre的代码，表现了银行木马恶意软件作者的技术高深。
