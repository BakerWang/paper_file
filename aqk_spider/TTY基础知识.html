TTY基础知识

2015-08-24 14:40:35 阅读：7611次 点赞(0) 收藏 来源： 360安全播报

TTY揭秘

TTY子系统是为Linux和UNIX设计的。不幸的是,它的重要性经常被忽视,因此很难找到好的介绍性文章。我相信,一个基本的了解tty在Linux开发人员和高级用户中是至关重要。

事实上,tty子系统,从用户的角度来看——是由一段扭曲混乱东西组成。要了解这是怎么来的,我们必须先回到过去。

历史

1869年,发明了股票。这是一个机电机组成的打字机,它的目的是实时分发股票价格。基于ASCII的电传,这个概念逐渐演变的更快了。用打印设备连接形成一个大的网络,称为电报,这是用于传递商业电报的打字机,但是没有连接到任何电脑。

然而,与此同时, 计算机仍然相当巨大和原始的,但能够多任务,能够与用户实时交互。当命令行最终取代了旧的批处理模式,打印设备被用作输入和输出设备。

大量的电传打字机模型都略有不同,所以需要某种软件来做兼容层。在UNIX的世界,这种方法是让操作系统内核处理所有的底层细节,如字长,波特率,奇偶校验,流量控制,基本线编辑等的控制码。花哨的光标移动,色彩输出和其他高级功能可能在1970年代末由固态视频终端vt – 100来实现,这是留给应用程序。

如今,我们发现物理打印设备和视频终端几乎灭绝。除非你参观博物馆或硬件爱好者,你可能会看到用TTY模拟视频终端的软件模拟真实的东西。但是我们应当看到野兽仍潜藏在表面之下。

使用案例

 

在一个终端用户类型(物理电传)。该终端通过一条导线连接一个UART(通用异步接收器和发送器)的计算机。操作系统包含一个UART驱动程序管理的字节的物理传输,包括奇偶校验、流量控制。在那ïVE系统,串口驱动将提供输入的字节直接应用过程。

但这种方法将缺乏以下基本特征:

线路编辑。大多数用户打字时容易犯错误,所以退格键是经常用到的。依照UNIX设计理念,应用程序应该保持尽可能简单。为了方便,操作系统提供了一个编辑缓冲区和一些基本的编辑命令(退格)。高级应用程序可能禁用这些特性,通过原始模式而不是默认的规范模式。大多数交互式应用程序(编辑、邮件用户代理)在原始模式运行,并处理所有的编辑命令行本身。

顺便说一句,内核提供了几种不同的行规则。但是只有其中一个是连接到一个给定的串行设备。默认提供编辑功能,被称为N_TTY(drivers/char/n_tty.c,如果你觉得危险)。

会话管理。用户可能想要同时运行多个程序,并与之交互。如果一个程序进入了一个无休止的循环,用户可能想要杀死它或者暂停它。在后台启动的程序应该能够执行,直到他们尝试写到终端,在哪个点他们应该被暂停。同样,用户输入只应针对前台程序。操作系统实现这些功能的驱动(drivers/char/tty_io.c)。

一个操作系统进程是“活着”的(有一个执行上下文), 这意味着它可以执行操作。TTY驱动却没有活着;在面向对象的术语,TTY驱动是一个被动的客体。它有一些数据字段和一些方法,但它实际上可以做某件事的唯一方法是当它的方法被称为一个进程或内核中断处理程序的上下文中调用时。线约束同样是一个被动的实体。

一个特定的三UART驱动,线路规范实例和TTY驱动可称为tty设备,或有时只是TTY。, 一个用户进程可以通过操作相应的设备文件的写权限在/dev设备文件的要求影响任何tty设备的行为,所以当用户登录的时候在一个特定的终端,用户必须成为设备文件的所有者。这通常是通过登录程序,使用root特权运行。

现在,让我们退一步,看看所有这些属于流程模型。

流程

一个Linux过程可以在以下国家:

 

R       Running or runnable (on run queue)
D      Uninterruptible sleep (waiting for some event)
S       Interruptible sleep (waiting for some event or signal)
T       Stopped, either by a job control signal or because it is being traced by a debugger.
Z       Zombie process, terminated but not yet reaped by its parent.

通过运行ps,你可以看到哪些进程正在运行,哪些正在休眠。如果进程是休眠的,WCHAN列(“频道”会告诉你这个过程是在等待什么内核事件。

$ ps l
F   UID   PID  PPID PRI  NI    VSZ   RSS WCHAN  STAT TTY        TIME COMMAND
0   500  5942  5928  15   0  12916  1460 wait   Ss   pts/14     0:00 -/bin/bash
0   500 12235  5942  15   0  21004  3572 wait   S+   pts/14     0:01 vim index.php
0   500 12580 12235  15   0   8080  1440 wait   S+   pts/14     0:00 /bin/bash -c (ps l) >/tmp/v727757/1 2>&1
0   500 12581 12580  15   0   4412   824 -      R+   pts/14     0:00 ps l

“等待”等待队列对应的系统调用,因此这些过程将搬到运行状态时,有一个状态改变的子进程。有两种休眠状态:可中断休眠和不间断的休眠。可中断休眠(最常见的情况),而这个过程是一个等待队列中的一部分,当一个信号发送到它那里,它实际上可能也毁被移动到运行状态。如果你看看里面的内核源代码,你会发现,任何内核代码是等待一个事件,必须检查信号等待后schedule()返回,并终止的系统调用。

在上面的ps清单中,统计列显示每个进程的当前状态。同一列中也可能包含一个或多个属性,或标志:

s       This process is a session leader.
+       This process is part of a foreground process group.

这些属性用于作业控制。
