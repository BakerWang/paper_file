# /usr/bin/python 
# -*- coding: utf-8 -*
import requests
import re 
import hashlib
if __name__=='__main__':
   u = 'http://ctf.sobug.com/hack_game/f8495eeedb8e92ee/index.php' 
   s = requests.Session() 
   c = s.get(u) 
   p = c.headers['password'] 
   m = hashlib.md5() 
   m.update(p) 
   r = s.post(u,{'password':m.hexdigest()}) 
   print re.search('key.*<',r.content).group(0)[:-1]

根据错误提示” try again!”在IDA可找到注册验证的关键函数sub_401000

sub401000中可看到当name和pass长度相等时，将pass与offset unk_408030 处数据进行异或计算，随后的字符串比较指令（00401052 repe cmpsb）将计算结果与用户名相比较。相等时，验证通过。OD中00401052下断点。直接使用用户名作为密码，执行到此处。

得到正确的⽤用户名与密码。

CrackMe3

描述说key是弹出来的，代码中只有函数sub_401C40中的401 CAE call ds:MessageBoxA是弹窗。看来触发成功key会在这里弹出。
sub401C40被DialogFunc调用，DialogFunc有对消息的处理。消息ID为0x112 时，将“L”存入byte403070字符数组中，消息ID为0x111时判断该字符数值长度是否为0x30，是则调用sub_401C40。判断字符数组中的字符串是否满足要求。

手工在403070中输入由L和R组成的字符串，长度48。OD在401D1D下断点改 ecx为0x30使长度检测通过。
sub_401C40根据这个字符串计算得到6个字节，并在401C9A处比较这6字节是否与403044处的6字节（“查水表”）相同。

计算方法就是把L当作0，R当作1。这48位二进制转为16进制形式。 将“查水表”B2 E9 CB AE B1 ED转为二进制RLRRLLRLRRRLRLLRRRLLRLRRRLRLRRRLRLRRLLLRRRRLRRLR

得到Key


CrackMe 4

程序结构

程序带有FSG壳，通过跟踪分析，可以发现在0x4001D1处的jmp下断点，就可以把壳的代码运行完成，请不要脱壳，因为程序后面会需要读取自身的数据再释放，脱壳后会造成数据不正确。获取输入的密码将输入的密码加上前缀“HOWMP”，组成新的密码。执行两次MD5计算（大写），生成一个MD5值，与程序中的保存的MD5值进行比较。

后面的操作就是读取自身数据，使用输入的原始密码进行循环xor解密，解密出 PE文件。所释放的PE结构与原始PE相同，第二次释放出来的文件是gif。

解题方法

将第一处MD5的比较进行爆破，跳过验证。通过程序中出现的文件名后缀判断第一次解密所释放的文件为PE，通过PE的文件头与解密前的数据进行异或操作，得出第一次的KEY。

第二次解密方法同上，不同的是第二次文件名是gif文件，使用gif文件头或数据解密，可得第二次的密钥。 KEY

第一次：564987

第二次：w!q&cs

程序结构

整个程序的结构是执行N个int3，然后执行部分正常指令，再执行N个int3，然后执行部分正常指令，如此循环下去。

对用户名进行了三次循环，然后对注册码进行了简单的操作，通过对比两个寄存器的值是否相同来决定是否注册正确。
