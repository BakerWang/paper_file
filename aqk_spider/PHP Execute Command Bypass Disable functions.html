PHP Execute Command Bypass Disable_functions

2014-11-26 16:17:00 阅读：9115次 点赞(0) 收藏 来源： phithon.log

先简单说一下php调用mail()函数的过程。 
看到源码ext/mail.c 

236行： 

1char *sendmail_path = INI_STR("sendmail_path");
2char *sendmail_cmd = NULL;


从INI中获得sendmail_path变量。我们看看php.ini里是怎么说明的： 


1; For Unix only.  You may supply arguments as well (default: "sendmail -t -i").
2;sendmail_path =

注释中可以看到，send_mail默认值为"sendmail -t -i". 

extra_cmd（用户传入的一些额外参数）存在的时候，调用spprintf将sendmail_path和extra_cmd组合成真正执行的命令行sendmail_cmd 。不存在则直接将sendmail_path赋值给sendmail_cmd 。 
如下： 

01if (!sendmail_path) {
02#if (defined PHP_WIN32 || defined NETWARE)
03    /* handle old style win smtp sending */
04    if (TSendMail(INI_STR("SMTP"), &tsm_err, &tsm_errmsg, hdr, subject, to, message, NULL, NULL, NULL TSRMLS_CC) == FAILURE) {
05      if (tsm_errmsg) {
06        php_error_docref(NULL TSRMLS_CC, E_WARNING, "%s", tsm_errmsg);
07        efree(tsm_errmsg);
08      } else {
09        php_error_docref(NULL TSRMLS_CC, E_WARNING, "%s", GetSMErrorText(tsm_err));
10      }
11      MAIL_RET(0);
12    }
13    MAIL_RET(1);
14#else
15    MAIL_RET(0);
16#endif
17  }
18  if (extra_cmd != NULL) {
19    spprintf(&sendmail_cmd, 0, "%s %s", sendmail_path, extra_cmd);
20  } else {
21    sendmail_cmd = sendmail_path;
22  }


之后执行： 

1#ifdef PHP_WIN32
2  sendmail = popen_ex(sendmail_cmd, "wb", NULL, NULL TSRMLS_CC);
3#else
4  /* Since popen() doesn't indicate if the internal fork() doesn't work
5   * (e.g. the shell can't be executed) we explicitly set it to 0 to be
6   * sure we don't catch any older errno value. */
7  errno = 0;
8  sendmail = popen(sendmail_cmd, "w");
9#endif

将sendmail_cmd丢给popen执行。 
如果系统默认sh是bash，popen会派生bash进程。而之前的bash破壳（CVE-2014-6271）漏洞，直接导致我们可以利用mail()函数执行任意命令，绕过disable_functions。 


同样，我们搜索一下php的源码，可以发现，明里调用popen派生进程的php函数还有imap_mail，如果你仅仅通过禁用mail函数来规避这个安全问题，那么imap_mail是可以做替代的。当然，php里还可能有其他地方有调用popen或其他能够派生bash子进程的函数，通过这些地方，都可以通过破壳漏洞执行命令的。

影响版本：php 5.x 已测试，其他版本未测试

修复方法：修复CVE-2014-6271 

给出POC（http://www.exploit-db.com/exploits/35146/）如下：
