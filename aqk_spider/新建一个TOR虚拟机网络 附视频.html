新建一个TOR虚拟机网络【附视频】

2015-08-25 15:41:34 阅读：17558次 点赞(0) 收藏 来源： 360安全播报

虽然本文介绍的方法可以用于实现匿名性,但是这不是我们讨论的重点,本文重点在于创建一个基于or的安全的研究环境。

正如大多数安全研究人员所知道的那样,利用虚拟机分析恶意软件或漏洞的时候,虽然大部分人都喜欢喜欢在离线环境进行逆向分析,但 如果要想监视C&C通信活动或对漏洞利用工具包进行“活体”实验的话,这显然是不可行的。 你使用的是实际的IP地址吗? 你使用第三方软件如proxifier迫使所有流量通过tor进行传输,从而避免软件泄露自己的IP地址吗? 尽管逆向分析时整个系统都是通过VPN进行通信的,但后来是否必须重新连接所有软件呢?

不过没关系,我已经想出了一个解决办法,即创建 一个透明的Tor代理,使用隔离的VM来充当路由器,并令虚拟机通过Tor来传递所有流量。当然,这里都是零配置的,甚至根本感觉不到该代理的存在。 尽管搭建路由器需要花费15分钟左右的时间,但是一旦搭建好了之后,所有的VM根本无需任何配置就可以通过Tor进行通信了。

为了展示这是多么的简单,我们通过一个例子来说明如果通过我的配置(不涉及任何代理设置)来连接一台崭新的VM。

安装路由器

对VM的要求:

内存——至少200MB。

硬盘——至少500MB。

对于路由器软件,我选择的是pfSense,这是一款基于轻量级开源FreeBSD的防火墙和路由器软件,我们可以从其官方网站来下载最新的版本。 虽然这里使用的是VMware工作站,但是本文介绍的方法适用于大部分其他虚拟机软件。

创建一个新的虚拟机,通常会用pfSense的ISO(但还不要打开电源)。

设置虚拟机。对于网络适配器,只要按照平常的因特网连接进行设置即可(通常是“NAT”)。 

单击“LAN segments...”按钮,添加一个新的局域网段。 

点击“Add...”按钮,添加一个新的网络适配器(只需接受默认设置,然后单击结束按钮即可)。

设置新的适配器,让其使用刚才创建的局域网段,其他适配器亦是如此。

现在我们可以给虚拟机上电了。几秒钟后,将如下所示的画面,只要按 1,并等待下一个屏幕即可。

在接下来出现的屏幕中,我们按下“I”键选择安装程序,注意,你必须在10秒之内按下此键。

对于这个安装程序,你只需先单击“Accept these Settings”,然后是“Quick/Easy Install”,最后点击“Ok”即可。

在下一屏中,将提示安装内核,这里选择“Standard kernel”,等它安装完成后,就可以重新引导系统了(注意,VM启动之前必须删除CD,否则容易出现问题)。

如果一切顺利,你将进入欢迎屏幕。在这个屏幕的顶部,会为我们提供相应的广域网和局域网适配器信息。 WAN应该是em0,这是用于连接到互联网的适配器;而局域网应该是em1,这是连接到 LAN 网段的适配器。WAN适配器的IP地址应当是宿主机的VMware适配器所在子网上的一个IP地址,也就是必须在同一个子网中(通常为VMware Network Adapter VMnet8)。 对于LAN IP一般没有什么特殊要求,只要它跟WAN IP 不在同一个子网上面即可。

为了安全起见,这里路由器的Web控制面板默认情况下是不允许通过WAN来访问的,因此,为了访问这个路由器,我们需要给另一台虚拟机(尽可能通过tor进行连接)加电,并将其虚拟网络适配器设置为连接到为路由器设置的局域网段上。

现在,你只需要打开浏览器,并导航至pfSense路由器的局域网的IP(这里为192.168.1.1),就会看到下面的屏幕(这里的登录用户名和口令分别是admin和pfsense)。

你可以浏览安装向导,然后更改登录信息,并转至主配置页面。

请转至“Services > DNS Resolver”,然后取消选中“Enable DNS Resolver”,最后保存设置(我们将使用Tor作为我们的DNS解析器,从而使系统同时能够访问 .onion域)。

安装Tor

为此,你可以直接通过pfSense控制台(按8进入shell)进行安装;或者,进入客户虚拟机配置面板,然后启用SSH,方法是进入“System > Advanced”,然后选中“Enable Secure Shell”复选框,然后单击“Ok”,从而进行安装。

如果你选择使用SSH,那就必须进入客户虚拟机(这里的IP与控制面板的相同,并且登录信息也与管理面板的相同)。

一旦进入shell,我们就可以安装tor了:

pkg install tor
rm -rf /usr/local/etc/tor/torrc
打开“/usr/local/etc/tor/torrc”进行相应的编辑,然后保存以下配置:
DNSPort 53
DNSListenAddress YOUR_PFSENSE_LAN_IP_HERE
VirtualAddrNetworkIPv4 10.192.0.0/11
AutomapHostsOnResolve 1
RunAsDaemon 1
TransPort 9040

现在,将tor设置为开机自启动,并运行它:

touch /usr/local/etc/rc.d/tor.sh
cd /usr/local/etc/rc.d/
echo "/usr/local/bin/tor" >> tor.sh && chmod +x tor.sh
/usr/local/bin/tor

防火墙的配置

现在,我们需要给使用tor的代理建立相应的规则;不幸的是,你能转发所有范围内的端口,因此,最好分别给每种主要的端口(HTTP、HTTPS、IRC等)添加相应的端口转发规则,这样一来,就能够阻止所有与此不匹配的流量。

首先,请切换至“Firewall > Nat > Port Forwarding”,为希望通过tor转发流量的每个TCP端口或端口范围添加一个规则。目前,你可以只添加HTTP和HTTPS端口,然后在将来需要的时候再添加更多的端口。

这里的规则如下所示:

Interface: LAN (Traffic from VMs)
Protocol: TCP
Source: Any (All devices on the LAN)
Destination: not LAN net (Ignore traffic between VMs or the VMs and pfSense router)
Destination port range: from 80 to 80 (HTTP)
Redirect Target IP: 127.0.0.1 (The pfSense router)
Redirect target port: 9040 (The transparent proxy port we set in tor config)
Description: Doesn't matter, put what you want.

对于想使用tor的每个目标端口/端口范围,请重复上述规则。注意,你无法转发的整个端口范围 (1 到 65535)内的流量,但你可以转发其中大部分的端口流量。需要注意的是,tor不支持UDP,也就是说,使用tor就无法转发UDP流量,要想转发 UDP流量,你就必须使用真实的IP地址。

现在,切换至“Firewall > Rules > LAN”,并去掉IPv4和IPv6规则,相应的描述为“default allow LAN IPv4/6 to Any”。

如果删除下面的规则的话,将会阻止所有不符合端口转发规则(无法通过Tor转发)的流量,这意味着我们需要创建一个规则,以允许通过pfSense服务器的 DNS请求。为此,需要在“Firewall > Rules > LAN”中新建一条规则,使其包含如下所示的设置:

Action: Pass (Allow traffic matching this rule)
Interface: LAN (Traffic from VMs)
TCP/IP: Version: IPv4
Protocol: TCP/UDP (DNS can be both)
Source: LAN net (From VMs)
Destination: (type): Single host or alias
Destination (address): The LAN ip of your pfSense router.
Destination port range: from 53 to 53
Description: Allow DNS

如果希望允许通过虚拟机与路由器之间的流量的话,则需要添加如下所示的规则:

Action: Pass (Allow traffic matching this rule)
Interface: LAN (Traffic from VMs)
TCP/IP: Version: IPv4
Protocol: TCP/UDP 
Source: LAN net (From VMs)
Destination: LAN net (To VMs)
Destination port range: from Any to Any
Description: Allow VM Communication

大功告成! 现在,你可以通过dnsleaktest.com来检查你自己的IP地址了,目前它已经无法查到你的DNS了(也就是说,即使你启用了WebRTC,STUN漏洞利用也不灵光了)。
