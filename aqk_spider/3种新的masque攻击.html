3种新的masque攻击

2015-07-01 15:33:47 阅读：5632次 点赞(0) 收藏 来源： 360安全播报

最新发布的iOS8.4中，苹果修复了一些漏洞，包括两种Masque攻击的漏洞（CVE-2015-3722/3725和CVE-2015-3725）。我们称这些利用方式为Manifest Masque和Extension Masque，可被用于破坏应用程序，包括系统应用，并且可以打破数据容器。另外还有一种已经部分修复了的漏洞，但还没有公开细节，名为Plugin Masque，它可以绕过iOS授权机制并且劫持VPN流量。

我们已经公开了5种Masque攻击，如下表所示。

Manifest Masque 攻击利用CVE-2015-3722/3725漏洞去破坏一个iOS上存在的应用程序。被破坏的应用（攻击目标）可以是一个从App Store下载的正常应用，甚至是一个重要的系统应用。这个漏洞影响所有iOS7.x，iOS8.4之前的iOS8.x版本，我们在2014年8月首次将这个漏洞通报给了苹果。

Extension Masque攻击可以打破应用程序数据容器的限制。恶意应用程序随着iOS8内部的应用程序被扩展安装，可以获得目标应用数据容器的访问权限或者防止目标程序访问自己的数据容器。

除了这两个iOS8.4的漏洞补丁，我们还披露了另一个不受信任的代码注入攻击方式的细节，通过替换VPN插件的Plugin Masque攻击。

接下来我们将讨论技术细节，说明这三种Masque攻击。

Manifest Masque：恶意程序替换正品

发布一个iOS应用程序在一个web页面上，其中包含一个超连接重定向到一个托管在https服务器上的XML清单文件。XML清单文件包含有应用程序的元数据，包括它的包标识符，版本和.ipa文件的下载地址，如表1所示，当安装这个应用程序，iOS首先下载这个清单文件，并且为安装过程解析元数据。

<a href="itms-services://?action=downloadmanifest&url=https://example.com/manifest. plist">Install App</a>
 
<plist>
      <array>
          <dict>
             ...
             <key>url</key>
             <string>https://XXXXX.com/another_browser.ipa</string>
            ...
             <key>bundle-identifier</key>
             <string>com.google.chrome.ios</string>
             …
             <key>bundle-version</key>
             <string>1000.0</string>
           </dict>
           <dict>
              … Entries For Another App
           </dict>
       <array>
</plist>

表1.超链接和清单文件的一个例子。

根据苹果官方文档，包标识符字段应该是“你的应用程序的包标识符，在你的Xcode项目中指定”。然而，我们发现iOS并没有验证网页上XML清单文件中的包标识符与应用程序本身的包标识符之间的一致性。如果XML清单文件中的包标识符与另一个真正设备上的应用程序相同，并且清单中的版本高于设备上的版本，真正的应用程序将被拆除成一个虚拟占位符，而应用程序将仍然是安装使用其内置的包ID。虚拟占位符会在受害者重启设备后消失。另外，如表1所示，一个清单文件可以同时包含不同应用程序的元数据，这意味着这一漏洞可以使多个应用程序被拆除，只需要受害者点击。

通过利用这个漏洞，一个应用程序开发者可以安装他/她自己的应用，并且在同一时间拆除掉其他应用（如竞争对手的应用）。这样，攻击者可以在iOS上做拒绝服务攻击或者钓鱼攻击。

演示视频：通过钓鱼攻击安装“恶意Chrome”并且拆除掉正品。


Extension Masque：打破数据容器

苹果在iOS8中推出了APP扩展特征，不同类型的应用程序扩展为开发者提供各种新的方式来扩展iOS8上的应用程序功能。一个应用程序扩展可以执行代码，并且被限制在其数据容器中访问数据。扩展作为应用的一部分是分布式的，可能被攻击者利用作为新的攻击向量。

我们独立发现，iOS8应用程序的扩展可以获得其他应用程序数据容器的访问权限，或者防止其他应用程序访问自己的数据容器，只要扩展和目标应用程序使用同一个包标识符。攻击者可以引诱受害者安装一个内部应用程序，使恶意扩展感染他/她的设备。

这种攻击的效果取决于恶意扩展和目标应用程序的安装顺序。请注意，扩展是不能独立安装的，它必须被分配作为应用程序的一部分。所以之后的内容中，当我们说安装一个扩展，我们指的是安装一个应用程序进行扩展。

如果安装在目标应用程序之前，恶意扩展可以打破数据容器，并获得目标数据容器的完全访问权限，而目标应用程序仍正常工作。

如果恶意扩展安装在目标应用之后，目标应用程序将不能访问它自己的数据容器。这会造成拒绝服务，这种情况下，如果受害者重新安装目标应用程序，其功能将恢复，这就相当于恶意扩展安装在目标应用程序之前的这种情况，恶意扩展将可以打破数据容器获得访问权限。

Plugin Masque:特权执行不受信任的代码

除了iOS扩展，VPN插件是另一种类型，捆绑在.ipa文件中。相比iOS扩展，以在不需特权的情况下嵌入任何iOS8设备，VPN应用程序和VPN插件需要“com.apple.networking.vpn.configuration”权限来提供系统VPN服务。到目前为止只有几个专门的iOS开发者可以分发这些VPN客户端（例如Cisco Anyconnect, Junos Pulse, OpenVPN等等）。安装后，该VPN插件通过特权系统进程加载，没有任何用户界面。

图2显示的Junos Pulse.ipa文件的目录结构，VPN插件（SSLVPNJuniper.vpnplugin）位于Payload目录，随着普通应用（Junos Pulse.app）提供的用户接口来配置VPN。

图2.Junos Pulse App目录结构

我们发现如果一个恶意VPN插件具有和受害者的iOS设备中合法VPN插件相同的包ID，该恶意插件可以成功安装替换，。之后，当受害者发起普通VPN应用程序访问VPN服务，含有不可信代码的恶意VPN插件将被加载到neagent进程并且执行特权操作，如劫持/监测所有VPN流量。代码注入到neagent进程逃避沙箱也是利用了一个用于pangu8越狱工具的关键漏洞。通过利用的VPN插件漏洞，可能对一个iOS8.1.2或者之前版本的设备进行越狱，绅士利用汽其他内核漏洞直接空中越狱。

此漏洞是CVE-2014-4493 ，在iOS8.1.3中被修复。我们在2015年4月的Jailbreak Security Summit 上首次做了展示。

iOS更新现状

2015年1月发布的iOS8.1.3修复了App Masque,和URL Masque，Plugin Masque被部分修复。iOS的版本迭代速度是众所周知非常快的，然而我们的调查发现，正在iOS8.1.3发布五个月后仍有3分之一设备未更新新版本，它们仍存在漏洞。我们鼓励每个人，尤其是企业用户，及时跟进iOS更新。


总结

虽然苹果设备已经修复或半修复原本的Masque攻击，但仍存在其他的攻击面。本文中三种攻击方式的细节可帮助用户了解风险，更好保护自己。同时，我们推荐所有iOS用户保持它们的设备更新。
