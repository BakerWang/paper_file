Bochs hacking指南

2015-07-08 10:06:55 阅读：7638次 点赞(0) 收藏 来源： 360安全播报

这篇指南将指导你如何对Bochs IA-32(x86)模拟处理器进行攻击。本指南是一个研究项目的研究成果，该研究项目是由Michael E.Locasto博士所资助的。在这个研究项目中，为了使用一个禁用了某些CPU指令的漏洞防御系统，我需要对Bochs和Linux内核进行一些修改。如果你想要使用Bochs来测试系统结构的改变，那么这篇指南将会指引你正确的方向。

Bochs与QEMU,VirtualBox以及VMWare等虚拟机软件比较相似，但是它的所有指令都是通过C++的代码实现的。因此，Bochs的运行速度是非常慢的，但是它所能实现的功能却是多种多样的。一旦你熟悉了Bochs的使用，你便可以做很多很炫酷的事情，例如修改现有指令的执行过程，你甚至可以往软件中添加全新的指令。研究Bochs的源代码同样是一个学习x86架构的很好的方式。

这篇指南会带你一步一步编译Bochs，并运行其中的Linux内核。在此之后，我们会向你介绍Bochs的源代码，然后你还会从中学到如何添加和修改CPU指令。

尽管这篇指南也许适用于各类操作系统，但这篇指南是针对Linux操作系统所写的，所以我们假设读者已经有Linux的基础知识了(能够使用命令行，可以编译软件，知道内核是什么等等)。

编译Bochs

如果你的Linux发行版的软件库中包括了Bochs，你可以直接使用它，而不必对源代码进行编译了。但是，本指南将会在后面的章节对Bochs进行修改，所以你应当能够自行编译它。

首先，从SourceForge网站的Bochs页面上 下载最新稳定版的Bochs源代码。在编写本指南的时候，最新的版本为2.6.2，所以本指南的讲解使用的将是这个版本的Bochs。

在源代码下载完成之后，提取它：

$ tar xvf bochs-2.6.2.tar.gz 
$ cd bochs-2.6.2

Bochs为各类操作系统提供了大量的构造配置选项：

$ ls .conf.*
.conf.amigaos     .conf.linux  .conf.macosx   .conf.sparc   .conf.win32-vcpp   .conf.everything  .conf.macos  .conf.nothing  .conf.win32-cygwin

这些是运行了”配置”脚本的脚本文件，里面包含了大量的命令行参数，这些参数可以开启或禁用某些特定的功能。我们将会编写我们自己的脚本，但是事实证明，这些脚本并不能很好的运行。创建一个名为”my_configure.sh”的文件，然后将下列代码写入这个文件：

#!/bin/bash
 
# This is the folder that bochs will install to. Feel free to change this to
# anything you like. By default, we install bochs to a 'my_build" folder in the
# current directory.
install_path=$(realpath ./my_build)
 
# Create the installation folder if it doesn't exist.
mkdir -p "$install_path"
 
# Run Boch's configure script with the features that we need.
./configure \
  --enable-smp \
  --enable-x86-64 \
  --enable-all-optimizations \
  --enable-long-phy-address \
  --enable-configurable-msrs \
  --enable-disasm \
  --enable-fpu \
  --enable-alignment-check \
  --enable-3dnow \
  --enable-svm \
  --enable-vmx=2 \
  --enable-avx \
  --enable-a20-pin \
  --enable-pci \
  --enable-clgd54xx \
  --enable-voodoo \
  --enable-usb \
  --enable-usb-ohci \
  --enable-usb-xhci \
  --enable-cdrom \
  --enable-sb16 \
  --enable-es1370 \
  --enable-show-ips \
  --with-all-libs   \
  --prefix="$install_path"

这个文件将会使Bochs安装至当前目录下的”my_build”文件夹中。其他的参数则启用了Bochs大多数的功能，除了最值得注意的网络功能。无论我进行了多少尝试，我都无法让网络组建工作，所以在本指南中将不予讨论。

为了构建Bochs，运行下列命令：

$ ./my_configure.sh
$ make -j 4
$ make install

这些指令将会编译Bochs，然后将二进制代码放入”./my_build/bin”中。切换到那个文件夹，然后在下一章中，我们将会制作一个磁盘镜像，并为Bochs安装一个Linux微内核。

$ cd ./my_build/bin

创建磁盘镜像

在Bochs虚拟机中安装操作系统，你将需要一个硬盘。Bochs使用的是普通的硬盘镜像。然而，我们无法使用任意大小的文件，所以我们必须创建一个与特定硬盘实体参数差不多的镜像文件(柱面，磁道，磁头和扇区)。

磁盘镜像的大小等于：

磁盘镜像大小(字节)=柱面*磁头*每磁道的扇区*512

为了操作更加简便，我们将会把磁头数设为8，每磁道的扇区数设置为63。然后，我们将通过改变柱面数来选择磁盘大小。如果我们想要一个N兆字节的磁盘，那么我们就可以通过下面的方法计算柱面数：

柱面＝N*1024*1024/512/8/63

我们可以将它放入一个名为”creat_disk_image.sh”的脚本之中使其自动运行：

#!/bin/bash
# Usage: ./create_disk_sectors <megabytes> <path>
# For example, to create a 4GB disk image, run:
#  ./create_disk_sectors 4096 disk.img
 
megs=$1
heads=8
sectors_per_track=63
cylinders=$(expr $megs \* 1024 \* 1024 / 512 / $heads / $sectors_per_track)
sectors=$(expr $cylinders \* $heads \* $sectors_per_track)
dd if=/dev/zero of=$2 bs=512 count=$sectors
 
echo "Disk image file [$2] was created."
echo "Geometry: cylinders=$cylinders, heads=$heads, spt=$sectors_per_track"

现在，如果我们想要创建一个用于安装操作系统的1GB大小的磁盘镜像，我们可以运行：

$ ./create_disk_image.sh 1024 bochs-disk-image.img
2097144+0 records in
2097144+0 records out
1073737728 bytes (1.1 GB) copied, 3.47051 s, 309 MB/s
Disk image file [bochs-disk-image.img] was created.
Geometry: cylinders=4161, heads=8, spt=63

请确保你保存了”几何参数”的输出值，因为在配置Bochs使用磁盘文件时需要用到这些参数，这个过程将在下一章中进行讲解。

安装Linux微内核

既然我们已经对Bochs进行了编译，并生成了一个磁盘镜像，我们就可以准备创建Bochs配置文件并安装一个操作系统了。在这一章中，我们将会安装Linux微内核。

现在立刻开始下载CorePlus-current.iso,因为它需要一定的时间。

创建一个”bochsrc”文件

我们需要做的第一件事就是创建一个”bochsrc”文件。这就是Bochs的配置文件，它会对虚拟系统中所有的硬件部分进行配置。下面是一个bochsrc文件的样例，它将与Linux微内核以及我们在上一章中所创建的磁盘镜像一同工作。

# System configuration.
romimage: file=$BXSHARE/BIOS-bochs-latest
vgaromimage: file=$BXSHARE/VGABIOS-lgpl-latest
cpu: model=corei7_ivy_bridge_3770k, ips=120000000
clock: sync=slowdown
megs: 256
boot: cdrom, disk
 
# Hard disks.
ata0: enabled=1, ioaddr1=0x1f0, ioaddr2=0x3f0, irq=14
ata0-master: type=disk, path="bochs-disk-image.img", cylinders=4161, heads=8, spt=63
 
# CDROM
ata1: enabled=1, ioaddr1=0x170, ioaddr2=0x370, irq=15
ata1-master: type=cdrom, path="CorePlus-current.iso", status=inserted

获取更多有关bochsrc设置的详细信息，请查看bochsrc文档。

引导和安装Linux微内核

当你创建了bochsrc文件之后，请等待Linux微内核下载完成之后，你就可以启动虚拟机了。

$ ./bochs -f ./bochsrc –q

Linux微内核CD将会启动。请选择” Boot Core Plus with default FLWM topside”

     几分钟之后，你应该可以看到Linux微内核的桌面了。

     按住CTRL键的同时，单击鼠标中键(鼠标滚轮),之后你就可以在Bochs中控制鼠标进行操作了。再次进行同样的操作，你就可以释放鼠标光标，并重新回到主机系统了。

现在，你就在Bochs中运行了最新的Linux内核了！你可以选择不继续阅读本指南，这样你就可以开始对系统进行操作了，或者你也可以继续阅读下一章节。

安装Bochs磁盘镜像

我们的下一步，就是将我们所编译的内核安装至虚拟机中。为了做到这一点，我们必须能够访问虚拟机的文件系统。幸运的是，有一些实用工具可以方便我们进行操作。我们可以使用kpartx工具来安装原始的磁盘镜像文件。

用shell脚本安装镜像。将下列代码保存至mount.sh。

#!/bin/bash
kpartx -a -v bochs-disk-image.img
mount -o loop /dev/mapper/loop0p1 /mnt/myimage

然后，将下列卸载脚本代码保存至unmount.sh中。

#!/bin/bash
umount /mnt/myimage
kpartx -d -v disk.img

脚本会将磁盘镜像挂载至/mnt/myimage，所以我们需要创建这个目录。

$ mkdir /mnt/myimage

最后，运行mount.sh脚本来家在磁盘镜像。在这个步骤中，你可能需要root权限。在下一章中，我们将会安装新的内核。

安装新内核

Linux微内核与其他发行版Linux文件系统的工作方式是不同的。如果你查看了加载的镜像文件，你就会发现它的结构是这样的：

# tree /mnt/myimage/
/mnt/myimage/
├── lost+found
└── tce
    ├── boot
    │   ├── core.gz
    │   ├── extlinux
    │   │   ├── extlinux.conf
    │   │   └── ldlinux.sys
    │   └── vmlinuz
    ├── mydata.tgz
    ├── onboot.lst
    ├── ondemand
    ├── optional
    │   ├── aterm.tcz
    │   ├── ...
    │   └── Xvesa.tcz.md5.txt
    └── xwbar.lst

备份旧的文件，当你的内核安装失败时还可以进行恢复。

# cp /mnt/myimage/tce/boot/core.gz /mnt/myimage/tce/boot/core.gz-backup
# cp /mnt/myimage/tce/boot/vmlinuz /mnt/myimage/tce/boot/vmlinuz-backup

要安装新的内核，首先我们得使用新的bzImagge替换vmlinuz。

# cp arch/x86/boot/bzImage /mnt/myimage/tce/boot/vmlinuz

现在，我们需要使用新得内核模块重建core.gz文件。将其复制到你的系统中，然后提取它：

# mkdir core_extract
# cd core_extract
# cp /mnt/myimage/tce/boot/core.gz core.gz
# gunzip core.gz
# cpio -idv < core

将新的内核模块复制到lib/modules中，然后将新固件复制到lib/firmware中。

# mv ../my_modules/lib/firmware/ lib/
# mv ../my_modules/lib/modules/3.12.4-tinycore/ lib/modules/

将所有数据放回cpio文件，然后使用gzip命令将其恢复原样。然后将其重新安装至磁盘镜像中。

# find . | cpio -ov --format='newc' > core
# gzip core
# mv core.gz /mnt/myimage/tce/boot/core.gz

使用上一章中我们制作的unmount.sh脚本来卸载磁盘镜像，然后启动虚拟机。如果所有的步骤你的操作正确，虚拟机就会运行你所编译的内核了。

结论

如果你的操作遵循了这篇操作指南，你应该知道如何在Bochs中运行现代Linux内核了。从现在开始，你想要使用它来做什么事情，完全取决于你自己了。研究和学习几乎是无止境的，当然了，你可以使用你新学习到的技能来做一些有趣的事情！
