Apache Web服务器安全配置小TIPS

2015-03-18 18:42:51 阅读：21066次 点赞(0) 收藏 来源： 360安全播报

Apache是使用最为广泛的开源WEB服务器之一，由于它具有良好的稳定性和丰富的功能模块，使它受到广泛的欢迎，几乎在所有的Linux发行版都有预带二进制包的Apache服务，apache也可以在其他的操作系统运行，如：Unix、Windows，Mac OSX,BSD等。

然而，Linux下的Apache基本安装里，默认情况下不能提供一个完整的防攻击保护，所以，我们必须采取一些安全措施来保护我们的机器。

这篇文章为你提供关于如何保护安装在基于Debian Linux发行版上的ApacheWeb服务的技巧。

在列举、使用安全技巧之前，首先我们需要使用一些系统命令以了解Apache服务的一些信息：软件版本、安装了哪些模块、这些模块是否激活使用、虚拟主机是否开启（配置）

查看Apache版本

获得Apache版本（简短）信息：

Apachectl –v


获得更详细的版本信息：

Apachectl –V


查看Apache的配置信息

查看Apache服务器配置的根目录路径，虚拟主机开启/禁用情况、PID文件和其它有用的信息运行：

apachectl –s


查看激活的Apache模块

查看Apache运行加载的模块，使用命令：

apachectl –M


禁用Apache模块使用命令：

sudo a2dismod [TAB] module_name


APACHE安全TIPS

1.禁用Apache的版本和服务器签名

默认情况下，当访问一个不存在的页面时，报错页面将会泄露服务器的一些信息：apache版本、操作系统名称，这些信息的泄露，使攻击更为简单：攻击者可以利用者些信息找到可以利用的漏洞

如何来把这些信息隐藏起来呢？修改Apache的security.conf配置文件，把ServerSignature选项的on改为off，然后修改ServerTokens选项为Prod，如下图

sudo nano /etc/apache2/conf-enabled/security.conf


security.conf修改后

ServerTokens Prod
ServerSignature Off


修改操作完成之后，重启Apache服务器，使更改生效，在浏览器URL上输入服务器IP和不存在的路径，我们再来看一下它的报错页面：

sudo service apache2 restart


为了测试ServerSignature更改是否生效，我们用命令行来进行检测，例如：wget telnet curl。

使用Telnet测试

telnet localhost 80
HEAD / /HTTP1.1


用curl测试：

curl -I -L 
http://localhost


用wget测试：

wget --server-response --spider http://localhost


可以看到Apache的版本和服务器签名都被隐藏了。

2.禁用目录遍历

Autoindex是Apache的模块，当目录中没有index文件的时候能自动生成和输出目录清单，类似于ls命令。

要禁用Apache的autoindex模块，终端下执行命令：

sudo a2dismod autoindex
sudo service apache2 restart


测试更改是否生效：未找到页面如果跟下面的图片相同，即更改生效。

3.禁用CGI脚本执行、SSI、 以及Follow Symbolic Links

使用下面的配置关闭CGI脚本、SSI、以及Follow Symbolic Links

Options -Includes -ExecCGI –FollowSymLinks


4. 用ssl证书保障Apache的安全

用SSL证书用加密的方式保障信息传输的安全。

企业可以从不同的SSL供应商购买SSL证书。小型企业一般并不愿意购买SSL证书，此时，你仍可以为网站分配一个自签名证书。Apache使用mod_ssl模块来支持SSL证书。

# openssl genrsa -des3 -out example.com.key 1024
# openssl req -new -key example.com.key -out exmaple.csr
# openssl x509 -req -days 365 -in example.com.com.csr -signkey example.com.com.key -out example.com.com.crt


在创建并签署了证书后，你需要在Apache配置中增加这个证书。用vim编辑器打开主配置文件，并增加下面的内容，然后重启服务：

SSLEngine on
SSLCertificateFile /etc/pki/tls/certs/example.com.crt
SSLCertificateKeyFile /etc/pki/tls/certs/example.com.key
SSLCertificateChainFile /etc/pki/tls/certs/sf_bundle.crt
ServerAdmin ravi.saive@example.com
ServerName example.com
DocumentRoot /var/www/html/example/
ErrorLog /var/log/httpd/example.com-error_log
CustomLog /var/log/httpd/example.com-access_log common


此时，打开浏览器，并输入https://example.com，就可以看到自签名的证书。

5.使用Rewrite模块将HTTP跳转到HTTPS

在上一步开启了HTTPS之后，你可以在.htaccess文件中修改配置来重定向

RewriteEngine On
RewriteCond %{HTTPS} on
RewriteRule (.*) http://%{HTTP_HOST}%{REQUEST_URI}


开启rewrite模块

sudo a2enmod rewrite
sudo service apache2 restart


6.使用apache的Access Control List来限定访问

我们可以用httpd.conf文件中的“Allow” 和“Deny”限制对目录的访问。

比如限定一个特定的IP段或者域名访问

<Directory "/var/www/html/">  
Order Deny, Allow
Deny from all
Allow from 192.168.1.0/16 host.domain.tld 10.252.46.165
</Directory>


设置禁止访问的文件，比如后台目录

<Directory "/var/www/html/">     
        Deny from all     
</Directory>


配置httpd.conf限制一些文件类型的访问，如txt的日志

<Files ~ ".txt$"> 
    Order allow,deny 
    Deny from all 
</Files>


7. 启用Apache日志功能

Apache允许你独立记录操作系统的日志。例如，在用户与Web服务器进行交互时所输入的命令信息就非常有用。

为此，你需要包含mod_log_config模块。Apache有三个主要的与日志相关的指令：

TransferLog：创建日志文件

LogFormat ：设置定制格式

CustomLog ：创建并格式化一个日志文件

我们还可以在虚拟主机部分设置这些命令。例如，下面的例子是启用了日志功能的一个网站的虚拟主机配置：

DocumentRoot /var/www/html/example.com/
ServerName www.example.com
DirectoryIndex index.htm index.html index.php
ServerAlias example.com
ErrorDocument 404 /story.php
ErrorLog /var/log/httpd/example.com_error_log
CustomLog /var/log/httpd/example.com_access_log combined


8.设置Apache用户不允许登录

在绝大部分的Debian中，Apache跑在www-data用户下，为了保证系统安全，我们应设置Apache用户不允许登录：

sudo usermod -s /usr/sbin/nologin www-data


上面是一些生产环境中安全配置apache的小tips，希望能对你的系统安全起到帮助。
