使用Meterpreter安全地dump域hash

2015-07-02 13:07:09 阅读：11295次 点赞(0) 收藏 来源： 360安全播报

与一切行业中的其他人一样我们站在了巨人的肩膀上。我们当然不愿意带走别人之前的工作成绩和成就。

从一个含域控制器的域中dump存储的密码hash是一件非常棘手的事情。有很多事情需要考虑有很多方法可以解决这一问题。但是这些方法都有比较明显的问题。我们最近改变了这一切找到了新的方法去解决该问题。在我们讨论新方法之前让我们先看一看那里的历史。

LSASS Injection

比较原始的方式Metasploit dump任何Windows的密码hash是通过LSASS注入。LSASS(Local Security Authority Subsystem Service)是在Windows系统中负责处理身份认证和安全策略的一种服务。Meterpreter可以注入到lsass.exe进程中并且直接从进程中获取密码hash。这是一种比较有效的手段但是非常的危险。注入到 LSASS将会造成进程不稳定并且使系统崩溃。如果在Windows系统中LSASS进程出现异常整个系统将会处于一个非常糟糕的状态并且会通过重启去修复这一问题。如果你在存在域控制器的域中通过这种手段dump域hash你不仅要通过域控制器的身份认证可能整个域中的合法用户也需要通过身份认证。对于针对工作站的渗透测试Metasploit很久以前就放弃了这种方法放弃基于注册表dump Hash的方法。然而事实上这仍然是一种有效的方法去获得域hash。

NTDS.dit

NTDS.dit是一个活动目录Active Directory数据库这是一个可扩展的存储引擎(ESE)数据库。 这是一种基于 Jet数据库引擎的糟糕的数据记录格式。当域控制器正在运行的时候NTDS.dit当然是处于锁定状态。我们需要copy这份文件在不锁定的状态下。几年前有一篇文章讲述如何安全的dump域hash。文章的作者是Tim Tomes这篇文章是Tim Tomes和Mark Baggett共同的研究成果。他们的技术利用Volume Shadow Copy Service(VSS)直接拷贝NTDS.dit文件然后他们使用其他的工具去解析。作为后续研究者的我们在很大程度上是基于Csaba Barta发表的白皮书做研究。

Running Extraction Tools on the Victim在被入侵的计算机中运行提取工具

当我们复制NTDS.dit文件后我们需要去解析它。ntdsxtract等工具就是解析NTDS.dit的有效工具。如果我们在被入侵的电脑中运行类似ntdsxtract的工具它们将会在内存中运行解析用户数据资料并且dump它们然后在用户完全不知道的情况下直接退出。这些工具的问题在于运行它们的时候会非常的占用计算机内存消耗CPU的资源。在渗透测试工作结束后工具仍然会驻留在域控制器的CPU中占用着资源。作为一个渗透测试工程师如果你损坏了域控制器DC的话你的客户会非常不满意的。如果你在做一个比较正义的渗透活动使用这种工具也极大的增加了你被抓住的可能性。使用这种方法去测试的话需要以上这几种可能因素。

Downloading NTDS.dit

人们使用的另一种方法是直接下载拷贝的NTDS.dit文件和部分注册表他们需要去破解存储的hash他们可以在自己的机器中在不联网的状态下进行破解而不用消耗目标计算机的资源。这种方法同样存在很大的问题会消耗更多的计算机资源。NTDS.dit文件大小以GB计算 。在传输大文件的过程中会消耗更多的时间。这可能会引起注意。如果你的文件在传输的过程中丢失部分数据你也没有办法去解决这种问题这当然不是一个理想的状态。

Meterpreter救援!

还记得我们之前讨论过NTDS.dit是一个ESE数据库么ESE是基于Jet数据库引擎的。域控制器必须有某种方式访问NTDS.dit 不读取它作为原始文件么嗯,每一个Windows系统均配有JetAPI。我们可以使用Windows下的JetAPI为我们处理解析所有数据库。我们创建一个Jet数据库引擎实例并将它附到 NTDS 数据库副本中我们可以遍历数据库中的所有内容找到用户数据的具体记录。这种方法允许我们在目标机器中解析数据并且不会将整个文件加载到内存中。

当Meterpreter连接到数据库时,它会在Metasploit客户机创建一个Channel作为通信的解析器。我们的客户在channel种执行读取操作。每一次读取操作被触发,解析器将读取20个用户的账户并且发送回给我们。批处理用户账户有多重目的。我们只有20个账户同时加载到内存中这将使我们的内存占用率非常低它使我们的请求/响应会话有效,对基于HTTP协议的会话传输是非常重要的。最重要的是我们可以实时获取账户信息。如果我们的链接丢失或者会话终止我们不会丢失所有的进程我们仍然可以在存活的进程中获取数据。

用户帐户

NTDS用户数据中包含许多我们需要的账户信息。让我们从每个帐户看一看:

The Account Name (samAccountname) 账户名称(samAccountname)

The account Description 帐户描述

The Date and Time of the last time the account logged in 最后一次账户登录的日期和时间

The Date and Time of the last time the account’s password was changed 账户密码修改的最近日期和时间

The Date of when the current account password expires 账户密码到期时的日期

If the Account is Disabled 帐户是否被禁用

If the Account is locked out 帐户是否被锁定

If the Account’s password has expired 如果账户的密码已经过期

If the Account’s password is set to never expire 如果账户的密码设置为永不过期

The current NT and LM hashes for the account 当前NT和LM用户的hash

The saved history of previous NT and LM hashes (up to 20 depending on AD settings) 当前NT和LM用户的历史hash

这是最后一个值得注意的地方不仅仅是我们dump每一个NTLM用户 hash。我们可以将所有的密码hash转换为之前的密码。如果你破解了这些hash你甚至可以分析出用户密码的设置习惯。一个特定的用户使用密码有一个特定的习惯你会知道他的下一个密码会设置成什么。你在很多用户的历史密码中看到过类似“changeme123”或“summer2015”的密码么之后你可能就会支持IT密码的重置计划。在渗透测试的过程中有很多历史数据值得我们直接利用而且我们不用花费太多的时间因为他们可以直接被使用仅仅使用我们的想象力。

当前和历史的NTLMHash数据将被保存在数据中从Metasploit的 auxiliary/analyze/jtr_crack_fast你可以破解这些hash如果你不想去破解他们只是想使用这些hash作为工具你可以使用Bruteforcers/Psexec mdoules。 Metasploit Pro用户将可以立即使用这些在程序的Credential Reuse部分或者通过Hash Metamodule使用它们。

不久的将来你将获得当你获取到域控制器session的第二个工具。只要使用Metasploit的新模块 atpost/windows/gather/credentials/domain_hashdump。它将为你实现一切功能。

Happy pwning!
