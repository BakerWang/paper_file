在AWS EC2上运行cudaHashCat64

2015-03-09 17:56:33 阅读：22615次 点赞(0) 收藏 来源： 360安全播报

大家好，

我曾经做过一个有趣的项目，并且认为这可以用来帮助那些希望使用硬件进行破解的人们。在其他地方也有一些相关的指引，但它们都过时了并且有的在AMI上根本无法运行。

从另一方面来说，虽然没有办法通过AWS（亚马逊WEB服务）的控制台清晰地控制你的GPU，但是你可以在CloudWatch（亚马逊云监控服务）上自定义指标。 这会用到一些自定义脚本，但我在虚拟化层不是最好的。我才刚开始接触hashcat，所以如果以下测试无法运行我深表抱歉。

步骤：

1. 注册AWS，并且确保你的钱包里有足够的钱来运行g2.2xlarge

你可以在这里找到价格：

http://aws.amazon.com/cn/ec2/pricing/

2. 开启一台使用g2.2xlarge的Amazon Linux AMI (我曾经用过ami-146e2a7c) 然后配置其他任何你想要的实例（storage,tag等等）

3. 运行命令

sudo yum update

4. 使用"lspci"命令查看主机信息: 

lspci

00:00.0 Host bridge: Intel Corporation 440FX - 82441FX PMC [Natoma] (rev 02)

00:01.0 ISA bridge: Intel Corporation 82371SB PIIX3 ISA [Natoma/Triton II]

00:01.1 IDE interface: Intel Corporation 82371SB PIIX3 IDE [Natoma/Triton II]

00:01.3 Bridge: Intel Corporation 82371AB/EB/MB PIIX4 ACPI (rev 01)

00:02.0 VGA compatible controller: Cirrus Logic GD 5446

00:03.0 VGA compatible controller: NVIDIA Corporation GK104GL [GRID K520] (rev a1)

00:1f.0 Unassigned class [ff80]: XenSource, Inc. Xen Platform Device (rev 01)


下面这行就是硬件的信息：

00:03.0 VGA compatible controller: NVIDIA Corporation GK104GL [GRID K520] (rev a1)

5. 从Nvidia上下载驱动：


wget http://us.download.nvidia.com/XFree86/Linux-x86_64/346.35/NVIDIA-Linux-x86_64-346.35.run


6. 修改权限：

chmod +x NVIDIA-Linux-x86_64-346.35.run

7. 注：我本来为了运行它写了一个yum install kernel* ，但是之后试着在另一个实例上运行的时候我发现我不需要它了。


yum install kernel*
reboot


8. 安装开发工具：

yum groupinstall development tools

9. 安装Nvidia驱动：

./NVIDIA-Linux-x86_64-346.35.run

10. 在Amazon Linux上编辑yum.repos.d添加 repos：


nano /etc/yum.repos.d/
Modify /etc/yum.repos.d/epel.repo. Under the section marked [epel], change enabled=0 to enabled=1.

11. 安装p7zip：

yum install p7zip

12: 下载hashcat：

wget http://hashcat.net/files/cudaHashcat-1.33.7z


13: 解压缩：
