三星故意禁用Windows更新

2015-06-25 15:46:06 阅读：10540次 点赞(0) 收藏 来源： 安全客

在我的论坛之家-Sysnative，有一个用户(wavly)一直在帮助处理Windows更新的问题。过程一直进展顺利，只是wavly的Windows更新时不时的会被禁用。最终他发现，在使用了auditpol.exe以及对注册表进行了安全审计之后，禁用Windows更新的程序原来是Disable_Windowsupdate.exe，而且这个程序是三星公司SW更新套件的一部分。

SW更新套件是一个典型的用于更新三星设备的OEM更新软件,同时它还可以更新三星设备上预装的软件。与其他OEM更新软件不同的是，三星的更新软件会禁用Windows的更新功能。

SW更新套件将会安装在：

Windows XP (all Service Packs)
Windows Vista (x86/x64)
Windows 7/SP1 (x86/x64)
Windows 8/8.1 (x86/x64)

请注意，该软件会检查三星程序的运行环境，如果有某个部件没有检测到，那么程序的运行将会出现奇怪的问题，很多功能将会无法正常使用，所以在我们对这个软件进行研究之前，还必须进行很多手动配置。

SW更新套件运行在何种设备上？

三星提示：  SW更新套件可以帮助你的Windows计算机下载并安装最新版本的驱动程序，系统更新以及软件更新。

所以最有可能运行Windows操作系统的设备就是台式电脑和笔记本电脑。

卸载／移除三星SW更新

在安装SW更新套件之后，它将会被列入Windows的添加／移除(或者程序卸载)菜单中，它与你操作系统中其他程序一样，可以被卸载。如果你从Windows的添加／移除(或者程序卸载)菜单中将其卸载并重启系统，它便不会在菜单中出现了。但SW更新服务仍然会处于活跃状态并运行着，而且\ProgramData\Samsung文件夹仍然会存在在硬盘中，文件夹中的”Disable_Windowsupdate.exe”也无法避免的遗留在了你的计算机中。

首先，以传统的方式通过Windows的添加／移除(或者程序卸载)菜单卸载SW软件之后，会删除注册表中的以下表键(以下只列出部分)：

然而，我们通过导航栏快速定位到注册表中该表键所在的地方：

     我们可以看到，这个注册表键仍然在那里。

请记住，正如我之前提到的，这有可能是因为该程序并不是在三星的软件运行环境下卸载的，因此卸载进程可能无法如预期的那样正常地工作。然而，这只是我的一个猜测，也许这种情况也会出现在一个正规的三星软件运行环境之中。

既然我没有正规的三星软件运行环境去对其进行测试，我便无法得出准确的结论，也无法知道通过Windows的添加／移除(或者程序卸载)菜单是否能够完全卸载这个软件。我希望它可以通过这种方法完全地从系统中卸载。

如果你是一名三星用户，现在请你忽略掉如何摆脱这个软件的问题，我想要说的是，请你试试通过Windows的添加／移除(或者程序卸载)菜单这种传统方式来卸载SW更新套件。如果你发现还是会发生同样的问题，那么情况就完全不同了，这个软件就必须要手动卸载了，或者你可以等待一个专用的三星卸载工具，就像联想公司的Superfish一样。

Type (0x00000110):据我所知，这就意味着它是一个Win32程序，它可以由Windows的服务控制器来驱动，而且它遵循Windows的服务控制协议。这种Win32类型的服务会单独以一个进程运行。

Start: (0x00000002): 这就意味着，它的所有服务被设置为自动加载或自动启动。它的加载器是服务控制管理器，就像内核的0x0（boot），以及输入输出子系统的0x1（system）。

ErrorControl: (0x00000001): 从中可以看出，如果驱动器加载或初始化失败，软件会显示警告信息。

所以现在最重要的问题就是，在你修改设置并重启之后，它是如何重置／禁用Windows更新的，而且这些行为并不是SW更新服务完成的。SW更新在这里只是在完成它该完成的任务，即更新三星设备的驱动程序以及软件服务等等。

实际上，导致Windows更新被持续重置／禁用的罪魁祸首就是Disable_Windowsupdate.exe进程所创建的一个计划任务，它会在用户每次登录系统的时候来确保Windows更新被禁用。
