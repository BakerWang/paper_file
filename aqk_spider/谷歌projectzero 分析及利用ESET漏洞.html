谷歌projectzero：分析及利用ESET漏洞

2015-06-24 14:21:12 阅读：16907次 点赞(0) 收藏 来源： 360安全播报

介绍

很多反病毒产品都含有模拟功能，该功能可以在签名被采用之前，让解压包进行模拟运行测试。ESET NOD32使用了一个微过滤器或者kext驱动内核去拦截硬盘上所有的输入输出数据，它会对拦截到的数据进行分析，如果检测到了可执行的代码，那么它便会模拟运行这些代码。

攻击者能够通过Web浏览器，电子邮件，文件共享，网络存储，USB以及其他成千上万的方法来触发系统的I/O操作。无论你在任何时候接收到信息，文件，图片以及其他数据，都有可能是一些流经硬盘的不受信任的数据。因为对于攻击者来说，触发不受信任的代码并虚拟执行它是非常容易的。与此同时，保证模拟器功能强大并且保持其独立性均是至关重要的。

这篇报告讨论的是ESET漏洞远程root权限的开发和利用，并且展示了攻击者是如何入侵ESET的用户主机的。入侵的危险是实实在在的，而且近期的迹象表明，攻击者对反病毒产品的兴趣正在不断加深。

常见问题解答

Q：l  哪些平台受到了影响？

A：因为DAT文件共享于所有的平台和版本之间，所以该产品所有的运行平台都受到了影响。

Q：l  产品的哪个版本受到了影响？

A：所有目前支持的ESET版本都含有漏洞代码。

下面为受影响的版本：

- ESET Smart Security for Windows
- ESET NOD32 Antivirus for Windows
- ESET Cyber Security Pro for OS X
- ESET NOD32 For Linux Desktop
- ESET Endpoint Security for Windows and OS X
- ESET NOD32 Business Edition

Q：l  产品默认配置受到了影响吗？

A：是的。

Q：l  如果我禁用了“实时”扫描，那么我还会受到漏洞影响吗？

A：如果你也禁用了“计划扫描”，那么只有当你从菜单或用户交互界面(GUI)手动去扫描一个文件时，你才有可能被影响。

请注意，如果你禁用了“实时”扫描，ESET将会持续提醒您开启“最大程度保护模式“。

Q：l  分析是否得出了漏洞的利用方法？

A：是的，这篇报告中含有一个有效的远程root权限漏洞利用。

Q：l  产品是否有可用的更新？

A：是的，ESET在2015年6月22日发布了一个扫描引擎的更新。

影响

任何一个运行了ESET并且联网的计算机都有可能被完全地入侵。主机被完全入侵，意味着攻击者可以读取，修改或删除系统中任何一个文件，而不受访问权限的限制；还可以安装任意的恶意软件；访问类似于摄像头，麦克风或扫描仪等硬件；记录系统所有的活动，例如键盘记录或网络流量等等。

入侵不会有丝毫的迹象，因为硬盘的I/O是操作系统运行的一个正常部分。正因为这不需要与用户进行交互，所以这个漏洞对蠕虫病毒来说是个绝佳的宿主。在企业级的ESET产品部署中，所有的商业数据，交易秘密，备份信息以及金融档案等信息都可能被窃取或销毁。

这些事情是有可能发生的，因为扫描进程的权限非常的高。在Windows网络中，远程攻击者入侵并控制ekm.exe进程，得到系统权限均是有可能的。在Mac和Linux系统中，攻击者有可能入侵并控制esets_daemon进程，然后得到root权限。

图片1是一个漏洞利用场景的视频截图：在一台安装了默认配置的ESET NOD32商业版的计算机中，一个普通用户点击了一个链接。一旦用户点击了这个链接，攻击者便可以以root用户的身份执行任意的命令。恶意链接并不是唯一的攻击方法，这只是上百种可能的攻击方法中的其中一种。

所有的版本，平台和产品如果处于默认设置的状态下，都有可能受到攻击，详情请往下看。

图片1:本次攻击使用了一个带有默认ESET配置的Web浏览器，点击观看攻击视频。http://youtu.be/Sk-CuFMXods

技术分析

图片1中所利用的特殊漏洞存在于ESET NOD32签名中。签名需要至少三个部分，并且还需要IMAGE_SECTION_HEADER中IMAGE_SCN_MEM_EXECUTE | IMAGE_SCN_MEM_READ | IMAGE_SCN_MEM_WRITE | IMAGE_SCN_CNT_CODE的特性。在入口点的第一个指令必须是在PUSHF之后的一个对PUSHA的调用命令。

如果符合条件，产品签名会在x86的模拟器中单独运行代码，并持续8万个周期。在每一条指令执行之后，之前的操作码会在栈操作之后进行检查，在40字节的栈缓冲区中检查是否存在PUSH,POP和ESP算法。这种检测机制有可能是为了检测某种形式的入口点混淆。

在这里，你可以看到代码会对PUSH操作进行检测。操作数的值可以从模拟器中获取，并存储于栈中。栈指针同样需要经过检测，以此来保证它没有越界。

 代码进行了一次POP操作，还对指令类进行了测试。

在保证栈指针不越界的时候，通过ESP的算术操作便可以进行绕过，漏洞便在这个地方。下面显示的是与实际代码功能类似的伪代码。

DWORD ShadowStack[10] = {0};
 DWORD ShadowStackPointer = 0;
 for (Cycles = 0; Cycles < 80000; Cycles++) {
    Emulator->Step(&ProgramCounter, &Instruction);
    if (Instruction.Class == PUSH) {
        ShadowStack[ShadowStackPointer++] = Emulator->GetOperandValue();
        if (ShadowStackPointer >= 10)
            Emulator->Shutdown();
    }
    if (Instruction.Class == POP) {
       if (!ShadowStackPointer || Instruction.Operand[1].Type != REGISTER)
            Emulator->Shutdown();
       Registers[Instruction.Operand[1].Register] = ShadowStack[ShadowStackPointer--];
    }
    if (Instruction.Class == ADD && Instruction.Operand[0].Register == REG_ESP) {
        // BUG!
        ShadowStackPointer -= Instruction.Operand[1].Value / 4;
    }
    if (Emulator->Fault) {
        Emulator->Shutdown();
    }
 }   
 Emulator->Shutdown();

 漏洞利用测试

为了协助本次分析，这篇报告提供了一个执行嵌入式脚本的漏洞利用样例，并且可以从此处进行下载。

https://code.google.com/p/google-security-research/issues/detail?id=456

为了测试文章中所介绍的漏洞利用方法，我们首先得禁用“实时”文件系统扫描，来防止意外的入侵。其次，我们还需要苹果公司的Xcode命令行工具包。如果你没有安装这个包，那么当你键入make时，系统会自动提醒你安装它。

$ ls -l
 total 28K
 -rw------- 1 taviso eng 17K Jun 18 12:45 esetemu.asm
 -rw------- 1 taviso eng 605 Jun 18 10:31 Makefile
 -rw------- 1 taviso eng 514 Jun 18 15:58 payload.sh

漏洞利用的过程中会需要payload.sh文件，成功地入侵系统也会需要它。

$ cat payload.sh  #!/bin/sh # # This is the payload code run as root in the context of esets_daemon after # successful exploitation. # osascript -e 'tell application "Finder" to set desktop picture to POSIX file "/usr/share/httpd/icons/bomb.png"'  
/Applications/Calculator.app/Contents/MacOS/Calculator &
echo w00t
uname -a; date; id

有效载荷的示例

电子邮件利用

将漏洞利用以MIME附件的形式发送给用户的Mail.app,Outlook等邮件客户端，用户会在毫不知情的情况下自动许可了利用行为。对于漏洞的利用，收取新邮件的行为就足够了，而且也不需要用户读取这些邮件或者打开邮件中的附件。

Web利用

漏洞利用可以以图片文件的形式上传至受信任的站点，或者上传至攻击者自己架设的站点。

结论

发现，分析并且利用这个漏洞，仅仅花费了我们几天时间。ESET已经告知了我们，他们正在努力提高产品的部署环境，并正在努力修复漏洞。

致谢

这个漏洞是在谷歌的Project Zero计划中，由Tavis Ormandy发现的。
