看国外白帽子如何黑掉Uber网站

2015-06-15 13:01:15 阅读：9257次 点赞(0) 收藏 来源： 360安全播报

概述：Uber通过电子邮件让我在他们的请愿书上签名。于是，我签了，而且签了10万多次。然后， 我把Uber网站黑了。

得有人关注这事儿了。

即使不是技术人员，也可以按照以下指导为几乎每个在线的请愿或竞赛签名10万次以上。人人都可以办到，且不费吹灰之力。不过，网站开发人员也可以轻而易举地修复这个问题。但请愿发起人及请愿网站做不到。为什么？

或许他们想要最大数量的签名/登入——不管是否真的是这样。相比起诚实，他们更青睐于荣耀。

或许作假的人很罕见，因此没有人认为应当造假。

或许他们只是很懒。

不管原因是什么，得有人关注这事儿了。这并非一劳永逸的办法；但如果决策者认为可以信赖这些数字，那么他们就应该作出改变了。而且，如果这些请愿的设计存在重大缺陷，那么我们又怎么能相信它们呢？

下面我们就来看看Uber的缺陷是什么。

洒洒水啊 冇问题 。（没错 小编是广东仔）

6月12日睡醒之后，我打算写一篇关于几家流行请愿跟竞赛网站的文章，因为它们并没有针对虚假登录实施任何保护措施。于是我从谷歌开始。

（是的，我人在泰国。我来自美国旧金山。）

谷歌返回的结果中有一条是关于Uber的请愿。我很好奇它们用的是什么平台？

噢，他们自己搭建了一个平台。Uber.org，是吧？我想着。“这个是个聪明的解决方法么？”答案是否定的。

很奇怪，他们的表单竟然接受“Zipcode（“邮编”的英文单词）”作为邮编。应该只接受数字啊。那表单还接受什么其它的东西？结果如下：

名字： dfs@#*odnsf!)fADFIHF

姓氏：asudf9SFNB(*SBDFSFO@#rnfweof

电子邮箱： –@–.00

邮编：@#(UFEFsdifsdah

结果-> Of course you can’t! Success!!???????（竟然成功了?!）

这个结果为什么不好呢？因为如果一张表单允许人们随便输入比如’?!这样的字符，用户就可以改变网站代码。

我马上注意到表单提交下方是5个最新的表单签名。签名显示了提交者的名字全拼跟姓氏的第一个字母。

这是什么意思？

意思是，我能输入我的名字，然后他们的网站就会把我的代码交给网站的其他访客。

这又是什么意思？

意味着，我能控制他们的网站。而且我确实做到了。

在第一次测试中，我在名字输入框中插入了一个评论提示符<!-。这会让浏览器忽视提示符后面的代码，直到浏览器随后在代码中看到了->。

这么说，我们应该能看到网站有的内容被删除了。来看看下面的视频。

随后，我在网站上放了一个iframe！iframe就是告诉一个网页加载另外一个网页。截图如下：

啊呀，是谁参加这个解析糟糕的请愿网页大聚会了？有人感到被遗忘了——Lyft公司！为什么请愿只给Uber呢？

经过将代码插入Uber表格的“名字”字段后，我能够插入一个iframe将访客带到Lyft.com。既然我们知道可以将代码插入他们的网页，并且运行良好，那么我们知道唯一的限制就是代码本身的限制。

恶意攻击者可以：

l偷偷通过病毒感染Uber的请愿访客。

l将一个看起来合法的骗局放在网页上，例如“无限次免费Uber乘车一年！只花299美元。点这里支付。”

lSQL注入Uber数据库并且接收所有请愿提交的数据。

l可能还可访问Uber网站网络的其他部分。

我没有这样做！Uber可以在数据库中查看这10多万次请愿提交，查看一下

我输入网站的代码。其中没有任何恶意内容。说起这10多万次提交，现在可帮了大忙了。我在网页添加的内容就有30秒钟（除非有5人在我之后签署了这个请愿）的有效期。如果我们能快速创建出签名，就能让这些内容持续有效。

我招募了一个僵尸部队。

为了让签名提交自动化，我跟一个免费项目iMacros进行了密切合作。强烈推荐该项目。iMacros会替我与网页进行交互。举个例子：我不必拿鼠标点击，就可以让软件替我点击。这对于做重复任务来说很有作用。

自动化签名提交就像下面这样。我告诉iMacros：

l打开网站

URL GOTO=http://petition.uber.org/sf/

l随机生成一份电子邮件（因为有人提交已提交过的电子邮件时，他们的服务器一团糟。）

SET email EVAL(“var emailpart = [‘you’,’can’,’do’,’better’]; var string = ”; for(var i = 0; i < 1; i++) {string += emailpart[parseInt(Math.random() * 4)]}; string”)

SET random EVAL(“var randomNumber=Math.floor(Math.random()*10000 + 1); randomNumber”)

lSet first name as: “ツ” (他们允许 ASCII 字符)

TAG POS=1 TYPE=INPUT:TEXT FORM=NAME:NoFormName ATTR=ID:inputFirstName CONTENT=ツ

l姓氏： “1+1=2″

TAG POS=1 TYPE=INPUT:TEXT FORM=NAME:NoFormName ATTR=ID:inputLastName CONTENT=1+1=2

TAG POS=1 TYPE=INPUT:EMAIL FORM=NAME:NoFormName ATTR=ID:inputEmail CONTENT={{email}}{{random}}@-0.0-

l邮编：“Meow”

TAG POS=1 TYPE=INPUT:TEXT FORM=NAME:NoFormName ATTR=ID:inputZip CONTENT=Meow

l信息： “We want people to trust online petitions. So, please do better with yours.”

TAG POS=1 TYPE=TEXTAREA FORM=NAME:NoFormName ATTR=ID:input

TextArea CONTENT=Wewantpeopletotrustonlinepetitions.So,pleasedobetterwithyours.

l提交表单

TAG POS=1 TYPE=BUTTON FORM=NAME:NoFormName ATTR=ID:submit

TAB CLOSE

l重置tab并清理Cookies

TAB OPEN

CLEAR

我在相同的脚本中复制了几次代码，让iMacros重复做这件事。

看看结果吧：（ツ开始出现在“最近签名“的正下方）。

通过这种方法，我15分钟左右便能提交一次。不过问题是网页的加载时间。

随后我意识到网页根本不会关心我是不是重新加载了页面或者清理了cookies。它不过是隐藏了输入框以及提交按钮，但他们依然还停留在页面并且是可用状态。

这意味什么？

意味着，iMacros能够在不重新加载页面的情况下不断重复填写表单。

可以从1分钟提交15次增加到1分钟提交420次签名：（视频后半部分加速）

我更进了一步——多个浏览器同时运行iMacros。

在视频中，我用VirtualBox运行几个Ubuntu Linux虚拟机。

从1分钟提交420个签名增加到1000多个：


计数达到106,000个签名后，我停了下来。当时，我预测合法的签名应该是16,000次左右。而其余的90,000次是我提交的。

然后，我再次重启做最后一次演示。

最终结果

这就是我的网页，位于他们网页的上方

现在你依然可以在他们网站上看到这个恶搞的请愿：http://www.thisshouldbefixed.yolasite.com/

更新：这种情况持续了大概2个小时。在我开始演示时，我注意到Uber有人马上将这个情况汇报了相关人员。一两天前，我也联系了Uber说明了这个情况，告诉他们我发现了一个漏洞代码利用并且准备公开。Uber将网站下线并且将页面先重定向至他们的员工登陆主页。现在他们没有再重定向网页了；然而，它要求尝试加载页面的人取得服务器验证。他们给我发邮件说正在解决这个问题。

更新：所有的Uber.org请愿在24小时之后依然处于下线状态；不过你可以查看缓存版本：

锦上添花

感谢Uber让控制网站变得如此简单。这是一次非常好的、具有教育意义的经历，但请不要再这么干了。所有写脚本的人都急着回家。所有开发了网页的人都是从一个在线辅导中复制粘贴的代码，这些代码声称自己有多么多么地简单易用。我是说真的。

我们来看看

亲爱的Uber，请修复网站。

1.在代码中插入适当的表单逻辑以阻止特殊字符。

2.删除虚假签名（filter by zipcode = “Meow”）

对于以后的请愿来说，执行逻辑以阻止有人在3小时内提交10万次。有人会欺骗你的请愿系统（或其他人的），可以轻易地区别所提交的姓名、邮编、电子邮件等，这样就很难知道哪些提交是假的。

让在线请愿超过应有的范围如在让人们提交签名之前验证政府ID等是不公平的。不过，我相信，如果请愿创建人能多想想，避免网站被恶意攻击，这样就会产生足够的信任。Uber并非唯一一个受人尊敬却会产生如此错误的公司。作为对2015年的提醒，我还会陆续报道此类事件。
