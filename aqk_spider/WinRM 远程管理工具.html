WinRM：远程管理工具

2015-07-23 11:30:40 阅读：23316次 点赞(0) 收藏 来源： 360安全播报

在去年所有发表出来的博文中，我最喜欢的是一篇发表在CrowdStrike网站的文章，这篇文章的名字为《Adversary Tricks and Treats》。在这篇文章中，CrowdStrike对深渊熊猫(Deep Panda)组织的黑客所使用的谍报技术进行了详尽的分析。为了躲避先进的恶意软件检测工具，深渊熊猫组织的黑客利用了本地工具来控制目标主机的系统，并在网络中使用横向传输方式来传输数据。他们有特定的攻击目标，所以他们不会使用恶意软件去攻击其他的对象。

这是一种非常重要的理念。设法找到一个可信的敌人，然后在网络防御过程中去使用新颖的方式来进行实践，这也是我最喜欢的红客团队任务之一。这对于一些喜欢刨根究底的，而且好奇心很重的分析人员来说，是一种非常积极的转变。作为一名红客，我们现在有机会可以将这些安全分析人员培养成为强大的网络防御人员。

暂且抛开这些对未来的设想，对我们来说现在最重要的事情就是思考如何去做。我认为其中一个方法就是学习不同的操作方式。我认为，就进攻方式以及模拟一个正在进行的进攻操作而言，拥有不同的攻击理念是至关重要的。现在我最喜欢的一种方式就是像深渊熊猫的黑客一样，尽可能地控制使用恶意软件的次数。

我敏锐地觉察到，那些技术高超的网络防御人员都会将他们的注意力集中在有价值的数据上，而不会将注意力放在其他的东西上面。域名控制器是无人值守的，一个技术实力雄厚的黑客团队将会像老鹰一样，将域名控制器视为他们的猎物。如此一来，某一固定的工作站就显得不是那么的重要了。

我喜欢在我自己的工作站进行操作，也喜欢使用我自己的恶意软件和本地工具去攻击和控制尽可能多的服务器。

如果你想要使用受信任的身份在远程目标主机上运行命令，那么我们有很多的方法来完成这些操作。我最喜欢的工具有schtasks,sc和wmic,而且我还是一名WinRM粉丝。

WinRM指的是Windows远程管理服务。它会监听5985端口。该端口系统默认是关闭的，但是有些系统管理员为了便于他们对服务器进行远程管理，会将这个端口开启。Windows 远程管理 (WinRM) 是 WS-Management 协议的 Microsoft 实现。该协议是基于简单对象访问协议 (SOAP) 的、防火墙友好的标准协议，使来自不同供应商的硬件和操作系统能够互操作。WS-Management 协议由硬件和软件制造商群体开发，作为一种公共标准，可用于与实现该协议的任何计算机设备远程交换管理数据。

WinRM 旨在改进网络环境中的硬件管理，这种环境包含许多运行各种操作系统的设备。该服务的整体设计侧重于通过实现可互操作的标准协议来监视和管理远程计算机。

当我们启用了WinRM时，你可以使用PowerShell对服务器进行远程查询和控制。当然了，如果你觉得自己很有把握，你也可以自行开启WinRM。下面给大家介绍，如何通过Beacon来开启WinRM服务：

powershell Enable-PSRemoting –Force

然后你就可以看到以下的输出结果：

WinRM要求操作人员的身份必须是受目标系统信任的。你可能需要一个域名使用者的令牌，这个令牌可以证明你是目标主机系统的本地管理员。你可以去偷一个令牌，或者使用Runas工具去生成一个令牌，你也可以使用Mimikatz工具生成一个用于传递密码哈希值的令牌。

为了使用WinRM来控制目标主机，你需要用到下列语句：

powershell InvokeCommand -ComputerName TARGET -ScriptBlock { dir c:\ }

键入上面的语句之后，PowerShell便会运行。使用WinRM，你可以根据你的需要修改脚本代码块中的内容。在命令执行完成之后，PowerShell将会把输出数据返回给你。

能够在远程目标主机中执行命令并且得到返回数据是非常棒的。在大多数的情况下，对于操作和控制一台目标计算机而言，这样的功能已经足够了。其中，我最喜欢做的一件事情就是通过这种方法来运行Mimikatz。PowerSploit的Mimikatz调用命令允许你指定一个计算机名称参数。还有一个有趣的地方：这个参数是一个系统参数，它可以用来运行Mimikatz。对这个选项进行了设置之后，PowerSploit将会通过WinRM，在远程目标主机的系统中运行Mimikatz，并且将输出数据返回给你。

下面是相关操作的语句：

powershell-import /local/path/to/PowerSploit/Exfiltration/Invoke-Mimikatz.ps1
powershell Invoke-Mimikatz -ComputerName TARGET
