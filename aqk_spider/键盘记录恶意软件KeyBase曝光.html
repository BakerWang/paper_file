键盘记录恶意软件KeyBase曝光

2015-06-05 14:08:21 阅读：23187次 点赞(0) 收藏 来源： 安全客

最近几个月里，我们团队一直在跟踪从2015年2月起就非常活跃的键盘记录恶意软件家族中的KeyBase。这个恶意软件拥有很多功能并且可以用50美元直接从作者处购买。它已经被部署在很多行业的攻击中并且可以通过钓鱼邮件进行传递。

总的来说，在过去四个月中，Palo Alto Networks AutoFocus 威胁被标记的覆盖超过大约1500个独特的session的295个样本的情报服务。攻击主要针对高科技、高等教育和零售行业。

恶意软件分发和目标

KeyBase最早于2015年2月被发现。这不久之前，’keybase[.]in’这个域被注册为在线销售KeyBase键盘记录的网店的主页。

域名：KEYBASE.IN

创建时间：2015-02-04  08:27:44 UTC

最新更新时间：2015-04-05 19:20:38 UTC

截至日期：2016-02-04 08:27:44 UTC

这个活动最初是被一个用户于2015年4月7日在hackfoums.net上发帖表扬宣传KeyBase的。在论坛的帖子中，这个恶意软件被吹捧有以下特点：

先进的键盘记录

扫描和运行时完全不会被发现（之后会被删除）

非常友好的web界面

unicode支持

密码恢复

恶意软件概述

KeyBase本身是用C#编写的，使用了.NET Framework。这些因素允许我们反编译底层代码，确定关键功能和键盘记录特点。

KeyBase包括以下功能：

Display a website on startup

Screenshots

Download/Execute

Persistence

Kill Timer

持久性

当它被启用，KeyBase的持久性被使用两种技术来实现——复制恶意代码到启动文件夹或者在启动时设置运行注册表键值来保证自动运行。

当KeyBase把自己复制到启动文件夹，它把自己命名为”important.exe”。这是被作者静态设置的，在以后的版本中，不能被用户更改。键值可以在使用时由用户在注册表中设置，通常是32字节的十六进制值。

HKCU\Software\Microsoft\Windows\CurrentVersion\Run [32 byte key] : [Path to Executable]

键盘记录

KeyBase中的键盘记录功能主要是在一个单独的叫做’KeyHook’的类中完成的。虽然这个类在github上的公共仓库中被分享出来了，这个类是被自定义编写的。虽然是自定义的，但是这个类使用了非常常见的Microsoft Windows SetWindowsHookExA 技术，为了hook受害者的键盘。

作者继续按照预期处理相应的键盘事件。

该类还必须处理Unicode字符，并拥有获得前景窗口的能力。这使得恶意软件不仅能识别哪些键被按下，还可以记录应用程序要求接下来要求按下哪些键。

Command and Control(C2)

通过HTTP和远程服务器进行通信。数据没有以任何方式被加密或者混淆。在初始执行时，KeyBase将执行一个初始登入远程服务器，我们可以从下面的内容中看到。

很多HTTP头都不包括request。这提供了一个简单的技术来标记恶意活动。同样重要的是要注意被包含在request中的使用硬编码的GET变量来监测活动是非常基本的使用方式。而受害者的机器名和当前时间会有所不同，该请求的其余部分保持不变。

Keystrokes

Clipboard

Screenshots

在它与C2服务器通信的过程中，KeyBase将包括使用各种GET参数的原始剪贴板和击键数据记录。这个数据是URI编码，但被明确发送。

最后，KeyBase也将使用一个特定的URI上传截图。路径 ‘/image/upload.php’是恶意软件的硬编码。所有被发送到C2的服务器都被放置在‘/image/Images/’ 路径。上传的数据再次被为加密发送，如下图。

网络面板

网络面板本身不提供任何新颖特征。他使用简单的红灰配色方案，如下图所示。

面板允许攻击者快速浏览感染机器，按键，屏幕截图，剪贴板数据和密码数据。不幸的是，KeyBase作者没有使用分页，这回导致显示大量被攻击者的数据时性能较差。

有趣的发现

在研究过程中，Unit42发现当浏览’/image/Images/’路径时不用提供任何身份验证。一个特定的C2服务与众不同，因为看起来创作者使用了他自己的机器进行测试。这样，截屏被上传到他的服务器，可以被公众看到。在下面的截图中，我们能够清晰的看到‘KeyBase v1.0’文件夹。这个文件夹几乎包含了KeyBase的安装。在查看操作者的桌面时，我也可以看到其他键盘记录信息，比如‘HawkEye Keylogger’ 和 ‘Knight Logger’。还有一个非常流行的名为’AegisCrypter’的加密。最终，我们还可以看到，用户使用盗版，’霍比特人’和’Fury’的副本都出现在了桌面上。

继续检查上传的截图，我们还可以确定用户使用远程桌面登录到了Windows Web Server 2008 R2 的实例。这似乎是攻击者使用‘Turbo-Mailer 2.7.10’的实例。不幸的是，操作者似乎在某个特定时刻忘记了自己的用户名或密码。

对上传截图进一步检查显示，用户登录了他的Facebook账户。用户的名字是‘China Onyeali’，他被观察到正在讨论一些最近的努力。具体来说，我们看到一个.rar文件的链接在rghost[.]net包含以下文件。我们也看到操作者在另一个回话窗口中正在讨论HawkEye keylogger。操作者的Facebook页面中说他住在Mbieri，Nigeria。

更加深入的有趣的发现

另一个有趣的发现是当研究后端的C2代码时发现的。upload.php文件被检查分析，这个文件控制文件上传到服务器。正如我们看到的一样，没有验证上传到服务器的文件类型。

从安全的角度可以提出问题，第三方可以简单的上传PHP脚本到’/image/Images/’目录获得不用验证身份的访问。下面的PHP代码可以被用来阅读KeyBase的’config.php’脚本，其中包含了网页面板上的用户名和密码。
