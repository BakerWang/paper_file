微软发布MS15-078公告：字体驱动含有漏洞影响所有版本

2015-07-21 09:42:41 阅读：19751次 点赞(0) 收藏 来源： 安全客

微软今日发布了MS15-078号安全公告，公告称微软字体驱动程序中含有漏洞，该漏洞可被远程执行代码。

如果用户打开特制文件或不安全的网页，其中包含嵌入式OpenType字体可被远程执行代码。

同时，该漏洞影响“所有受支持的Microsoft Windows”，也就是从Windows Vista、7、8、8.1、10到服务器版的Windows SVR 2008-2012。

相信Windows XP也含有该漏洞，由于XP已经被微软停止支持，所以补丁是没有了，XP用户还是需要自己多注意一些。

目前微软已经向受影响的Windows版本推送补丁用户修复该漏洞，以下是受影响的Windows版本列表：

Windows Vista：Windows Vista Service Pack 2、Windows Vista x64 Edition Service Pack 2

Windows Server 2008：Windows Server 2008  x64/x86 SP1/2 、Windows Server 2008 for Itanium-based Systems Service Pack 2

Windows 7：Windows 7 SP1、Windows 7 SP2

Windows Server 2008 R2：Windows Server 2008 R2 x64/x86 SP1

Windows 8和Windows 8.1：全部

Windows Server 2012 and Windows Server 2012 R2：全部

Windows RT and Windows RT 8.1：全部

Server Core installation option：SVR 2008、SVR 2008 R2、SVR 2012、SVR 2012 R2

漏洞信息

OpenType字体驱动漏洞－CVE-2015-2426

微软的Windows操作系统中，当用户通过错误的方式来使用Windows Adobe字体管理库时，尤其是使用OpenType字体时，将会触发一个远程代码执行漏洞。攻击者如果能够成功地利用这个漏洞，那么他就可以获取到受影响系统的完整控制权，紧接着他就可以在目标主机的系统中安装任意的软件，查看、修改或删除任意的数据，或者在目标主机的系统中创建一个拥有完整控制权限的用户账号。

攻击者如果想要利用这个漏洞，方式是多种多样的。比如说他们可以通过欺骗的手段让用户打开一个经过特殊设计的文本文件，或者通过欺骗用户，让他们去访问一个内嵌了OpenType字体的不受信任的网站等等。此次安全更新通过修正Windows Adobe字体管理库对OpenType字体的处理方式来修复这个漏洞。

当微软公司发布了这个安全公告之后，公司从他们所得到的信息中发现这个漏洞已经被公开的了，但是目前还没有任何迹象可以表明有黑客利用了这个漏洞去攻击客户。我们的分析结果表明，漏洞利用代码在经过特殊设计之后，可以允许攻击者持续地利用这个漏洞。

缓解因素

微软公司目前还没有找到可以缓解此漏洞的方法。

解决方案

下列解决方案对你来说也许会有所帮助：

l  重命名ATMFD.DLL

32位操作系统：

1.   使用管理员身份运行命令行，并输入下列命令代码：

cd "%windir%\system32"
takeown.exe /f atmfd.dll
icacls.exe atmfd.dll /save atmfd.dll.acl
icacls.exe atmfd.dll /grant Administrators:(F)
rename atmfd.dll x-atmfd.dll

2.   重启系统

64位操作系统：

1.   使用管理员身份运行命令行，并输入下列命令代码：

cd "%windir%\system32"
takeown.exe /f atmfd.dll
icacls.exe atmfd.dll /save atmfd.dll.acl
icacls.exe atmfd.dll /grant Administrators:(F)
rename atmfd.dll x-atmfd.dll
cd "%windir%\syswow64"
takeown.exe /f atmfd.dll
icacls.exe atmfd.dll /save atmfd.dll.acl
icacls.exe atmfd.dll /grant Administrators:(F)
rename atmfd.dll x-atmfd.dll

2.   重启系统

Windows 8以及Windows 8之后的操作系统的可选操作－禁用ATMFD：

请注意：如果注册表编辑器使用不当，则会引起严重的问题，很有可能会要求你重装系统。如果你错误地操作或使用注册表编辑器，微软公司不能保证由于你的错误操作所导致的问题得到解决。使用注册表编辑器的风险需由你自己承担。如果你想要了解如何编辑注册表，你可以查看注册表编辑器(Regedit.exe)的帮助文档中的”修改键和值”，或者你也可以查看Regedt32.exe中帮助文档的”添加或删除注册表中的信息”以及”编辑注册表数据”。

方法1(手动编辑系统的注册表)：

1.   以管理员身份运行regedit.exe。

2.   进入注册表编辑器，使用导航工具找到(或创建)下列子键,并且将它们的DWORD值设置为1:

HKLM\Software\Microsoft\Windows NT\CurrentVersion\Windows\DisableATMFD, DWORD = 1

3.   关闭注册表编辑器，并重启系统。

方法2(使用控制部署脚本)：

1.   创建一个文本文件(txt)，并将其命名为ATMFD-disable.reg，然后将下列代码写入文件：

Windows Registry Editor Version 5.00
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Windows]
"DisableATMFD"=dword:00000001

2.   运行regedit.exe。

3.   进入注册表编辑器，点击“文件”菜单，然后点击”导入”。

4.   通过导航工具定位到你在第一步中所创建的ATMFD-disable.reg文件。(请注意：如果你没有找到你所创建的文件，那么请你检查你所创建的文件后缀名是否为.txt，或者你也可以将对话框中的文件类型参数修改为”全部文件”)。

5.   点击”打开”按钮，然后点击”OK”来关闭注册表编辑器。

解决方案所产生的影响

那些依赖于嵌入式字体技术的应用程序将无法正确地显示文字。禁用ATMFD.DLL有可能会引起某些使用了OpenType字体的，特定的应用程序停止运行。微软公司的Windows操作系统在发布的时候并不自带OpenType字体。但是，第三方的应用程序可以安装这种字体，而且它们也会受到这个字体所带来的影响。

如何撤销上述操作？

32位操作系统：

1. 使用管理员身份运行命令行，并输入下列命令代码：

cd "%windir%\system32"
rename x-atmfd.dll atmfd.dll
icacls.exe atmfd.dll /setowner "NT SERVICE\TrustedInstaller"
icacls.exe . /restore atmfd.dll.acl

2.   重启系统

64位操作系统：

1.   使用管理员身份运行命令行，并输入下列命令代码：

cd "%windir%\system32"
rename x-atmfd.dll atmfd.dll
icacls.exe atmfd.dll /setowner "NT SERVICE\TrustedInstaller"
icacls.exe . /restore atmfd.dll.acl
cd "%windir%\syswow64"
rename x-atmfd.dll atmfd.dll
icacls.exe atmfd.dll /setowner "NT SERVICE\TrustedInstaller"
icacls.exe . /restore atmfd.dll.acl

2.   重启系统

Windows 8以及Windows 8之后的操作系统的可选操作－启用ATMFD：

请注意：如果注册表编辑器使用不当，则会引起严重的问题，很有可能会要求你重装系统。如果你错误地操作或使用注册表编辑器，微软公司不能保证由于你的错误操作所导致的问题得到解决。使用注册表编辑器的风险需由你自己承担。如果你想要了解如何编辑注册表，你可以查看注册表编辑器(Regedit.exe)的帮助文档中的”修改键和值”，或者你也可以查看Regedt32.exe中帮助文档的”添加或删除注册表中的信息”以及”编辑注册表数据”。

方法1(手动编辑系统的注册表)：

1.   以管理员身份运行regedit.exe。

HKLM\Software\Microsoft\Windows NT\CurrentVersion\Windows\DisableATMFD, DWORD = 0

3.   关闭注册表编辑器，并重启系统。

方法2(使用控制部署脚本)：

4．创建一个文本文件(txt)，并将其命名为ATMFD-disable.reg，                     然后将下列代码写入文件：

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Windows]

6.进入注册表编辑器，点击“文件”菜单，然后点击”导入”。

7.通过导航工具定位到你在第一步中所创建的ATMFD-enable.reg文件。(请注意：如果你没有找到你所创建的文件，那么请你检查你所创建的文件后缀名是否为.txt，或者你也可以将对话框中的文件类型参数修改为”全部文件”)。
