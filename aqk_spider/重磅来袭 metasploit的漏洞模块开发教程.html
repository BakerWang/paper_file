重磅来袭！metasploit的漏洞模块开发教程！

2015-05-12 08:51:43 阅读：26109次 点赞(0) 收藏 来源： Arthur@360安全播报

怎么写一个Metasploit的POST-开发模块

Metasploit目前包含了大约有150个漏洞利用模块。大部分的漏洞利用模块都是通过Windows，Solaris和思科这些平台进行收集的。同时，Metasploit还可以对这些模块就行升级，维护和管理。

Metasploit可以说是我遇到最强大的漏洞挖掘工具，也是内网渗透工具。下面我来说说怎么编写一个POST-EXPLOITATION模块。在你编写一个模块的时候，你先看看metasploit是否已经有你要编写的模块，以防重复了，做了一堆无用的工作。确定没有之后再进行编写。

这篇文章，会解释一下POST开发模块的基本结构，如何使用post-libraries，如何提供反馈信息给用户，以及如何储存用户通过漏洞模块所获得的数据。

PS:实际上，每个模块都有一个名称，这个名称就是漏洞的编号。大家可以搜索看看在metasploit是否已经有这个漏洞模块。还可以更具自己的metasploit版本信息，在网上查看这个版本安装了哪些漏洞模块。以防自己编写的漏洞模块和现有的漏洞模块重复了。还有一点要解释，就是metasploit会怎么利用这些漏洞模块。

漏洞模块的基本结构

开发metasploit模块，最简单的方法就是在metasploit上找一个功能类似的模块进行修改。举个例子，比如你发现了一个新的SQL注入语句，现在你要做一个SQL注入的metasploit模块。那么你可以在metasploit上找一个现有的SQL注入模块，然后加上你发现的注入语句即可。主要是因为大部分的metasploit模块都是开源的，用户可以随意更改。所有的metasploit模块由三个基本的部分组成，就是初始化，注册选项和运行。

你所开发的模块应该放在下面这个目录里面

msf4/modules/post/

如果没有post目录，那么自己创建一个。Metasploit会从framework加载你创建的模块。

初始化

初始化的意思差不多是当你使用一个漏洞模块，发出任何指令，注册，或者其他高级选项的时候，这个漏洞模块能够反馈出该有的信息。你需要修改的是这个模块的名称，作者名字，还有漏洞模块的描述，比如是什么漏洞种类，适用于什么操作系统之类的。这些描述需要使用到短字符代码。因为一个漏洞模块的主要工作是挖掘漏洞，没必要弄太多数据来描述一个漏洞模块。实际上大家也别想太复杂了。下面就是一个metasploit漏洞模块的例子。这个部分只是一个描述功能。



def initialize(info={})
    super( update_info( info,
        'Name'          => 'Linux Gather System Information Enumeration',
        'Description'   => %q{
            This module gathers basic system information from Linux systems.
            Enumerates users, hashes, services, network config, routing table, installed packages, 
            ,screenshot, and bash_history
        },

比如说sqlmap，一开始这个程序就解释了一些参数和含义之类的。初始化功能的主要地方就是这里，就是描述。

在下面这段代码，你会看到License，这个里面你必须填写MSF_LICENSE，否则metasploit就不会加载你的模块。Version，这个是版本信息，你可以随意填写。



'License'       => MSF_LICENSE,
'Author'        =>
    [
        'Stephen Haywood ',
        'sinn3r',  #Modified the original, and more testing
        'Carlos Perez ', # get_packages and get_services
    ],
'Version'       => '$Revision: 12842 $',
'Platform'      => [ 'linux' ],

下面这串代码有个SessionTypes参数，大概的意思就是教你填写一下会话类型。你可以填写shell，Meterpreter，或者两个都填写。但是我建议你最好两个都填写。因为metasploit有时候非常不稳定，至于为什么，我也不知道，反正你最好填写两个会话类型，以防其中一个失效后，剩下的一个会话类型还可以使用。

'SessionTypes'  => [ "shell", "Meterpreter" ]

注册选项

这个就是当你发出show指令的时候，会列出一些参数。这个大概有这么两部分组成。第一个是参数的名称，第二个是参数的值。第二个参数的值一般都是false或者true。如果有用到多个选项的时候，请用逗号隔开。下面是代码例子。在这个代码后面，是一个参数列表，大家可以更具自己想编写的漏洞提供不同的参数。



register_options(
    [
        OptString.new('RHOSTStrue, 'The target address range or CIDR identifier', nil]),
        OptInt.new('THREADSfalse, 'The number of concurrent threads', 10]),
        OptEnum.new('STARTUPtrue, 'Startup type for the persistent payload.', 'USER', ['USER','SYSTEM','SERVICE']])
    ], self.class)
You can also register advanced options, which are shown when you issue the show advanced command. The valid option types and the arguments are the same. These options should be used for setting parameters that a typical user will not work with but an advanced user may want to modify.
register_advanced_options(
    [
        OptBool.new('SETPASSfalse, 'Reset password', false])
    ], self.class)

注册选项存储在数据存储中的哈希值，并且可以通过调用数据存储区的['option_name']来访问。实际上你也可以直接跳过这一步，因为framework框架会自动帮你进行注册。在framework帮你注册的时候，VERBOSE的值是假。

运行模块

当你使用metasploit的漏洞模块时，你通过发送运行指令和路径让模块运行起来。随后要选择会话类型，或者是输入一些其他高级的指令。不管如何，你必须设置一个运行名称，要不然模块就运行不起来了。下面是代码的例子。实际上很好理解。大部分的安全人员都使用过sqlmap，其中你们输入的参数，比如-u, --thread, --level等等就类似于这个运行模块。而使用过metasploit的朋友对这个就比较熟悉了，最长见的莫过于RHOST和HOST了。就是指定一个目标IP地址。这个也是属于运行模块来处理的。



def run
    print_status("Running module against #{sysinfo['Computer']}")
    arp_scan(datastore['RHOSTS'], datastore['THREADS'])
end


使用POST库

Metasploit的框架里面包含了一个叫做POST开发库。这里面包含了一些开发metasploit漏洞模块常用的方法和代码。这个库在lib/msf/core/post。一般情况下，这个库里面还有Meterpreter 和 shell两个会话选项，以确保你们的metasploit漏洞模块可以被识别和使用。就连metasploit的开发人员也清楚，metasploit的bug还很多，提供这两个会话是必须的。

在使用这个库的时候，你得先导入一些文件，包括你自己设置的漏洞模块。下面是例子，直接copy一下代码更改即可。



require 'msf/core'
require 'rex'
require 'msf/core/post/windows/registry'
require 'msf/core/post/windows/priv'
 
class Metasploit3 < Msf::Post
 
    include Msf::Post::Windows::Registry
    include Msf::Post::Windows::Priv

确保你导入的文件类型为ruby，就是文件后缀为rb。你可以看看lib/msf/core/post/里面的模块名称，或者看看rdoc文件。如果你查看了lib/msf/core/post/windows/registry.rb文件，你顺便也可以看看这个文件的内容，包括isMsf::Post::Windows::Registry.之类的。

提供反馈

储存功能

如果你的漏洞模块的功能是收集一些数据，那么你需要使用一个叫做store_loot的方法来保存数据。这个方法可以把数据保存在一个独一无二的文件夹内，位置在 ~/.msf4/loot内，同时该方法还会添加一些指示信息让你能够尽快的找到那个储存文件。我这里再拿sqlmap举个例子。每当你扫描完一个URL的时候，就会生成一些历史文件存放在某个文件夹内。而文件夹的地址就是在你扫描完后最下方进行提示。这个好理解了吧。下面这个代码就是使用store_loot的例子。

path = store_loot(ltype, ctype, session, data, nil, info)

下面也是store_loot的例子。在这里面，设置储存类型之类的，比如是以txt文本储存，还是excel储存等等。

path = store_loot("host.users.recent", "text/plain", session, tbl.to_s, "recent_users.txt",
    "Recent Users")
 
path = store_loot("host.users.active", "text/plain", session, tbl.to_s, "active_users.txt",
    "Active Users")
 
path = store_loot("chrome.decrypted", "text/plain", session, 
    decrypt_table.to_s, "decrypted_chrome_data.txt", "Decrypted Chrome Data")
 
path = store_loot("host.applications", "text/plain", session, results, "applications.txt",
    "Installed Applications")
