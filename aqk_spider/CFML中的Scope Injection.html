CFML中的Scope Injection

2015-09-01 16:28:39 阅读：10887次 点赞(0) 收藏 来源： L.N.@360安全播报

0x00 前言

早上按例看mickey牛发的每日新知识,其中有一篇关于CFML中的Scope Injection,我不知道怎么翻译这个次比较好,是叫“范围注入”还是叫“变量覆盖”?话不多说,在文中我就写“Scope注入”了。


0x01 什么是CFML


   “CFML的全称是ColdFusion Markup Language,是一种特别适于用来编写互联网应用程序的语言。
        CFML看上去和传统的编程语言并不太一样,更接近HTML,因此和HTML一样非常的易学易用。
        CFML可以在ColdFusion应用程序服务器上运行,也可以在其它一些应用程序服务器运行。”
                                                                                                                                      -----百度百科
       简而言之,CFML就是一种开发语言,所用于web开发。


0x02 SCOPE注入

假设这是一段权限控制代码,通过session.isAdmin判断用户是不是管理员:

<cfif session.isAdmin>
    Do something only an admin can do...
</cfif>

如果isAdmin没有被定义在session中,也就是说isAdmin没有被初始化,那么我们像如下来发送一个GET请求会发生什么呢?


/admin/something.cfm?session.isAdmin=1

执行流程:

1.首先检测session.isAdmin是否被定义在session域中

2.如果没有被定义在session中,程序将到别的域中搜索,比如url域

3.如果在url域中有url.session.isAdmin,并且为true

4.上面代码的权限控制就被绕过了

通过上面的描述,有没有感觉有点像php中的register_globals。


0x03 实际测试

测试代码是Joseph Lamoree写的

https://github.com/ecivis/miniapp

测试环境:windows7+Adobe ColdFusion 9(Ps:只找到了这个)

测试代码主要片段:

<cfsilent>
<cfset pageScope = structNew()/>
<cfset pageScope.message = "You are a regular user."/>
<!--- Make sure the user is not isAdmin --->
<cfif structKeyExists(session, "isAdmin")>
<cfset structDelete(session, "isAdmin")/>
</cfif>
<!--- See if scope cascade finds this in the url scope --->
<cftry>
<cfif session.isAdmin eq 1>
<cfset pageScope.message = "You are an administrator due to scope shenanigans."/>
</cfif>
<cfcatch>
<!---
The lookup for session.isAdmin was invalid.
This is the behavior we would expect from a reasonable from a CFML engine.
 --->
</cfcatch>
</cftry>
</cfsilent>


发送请求:

http://127.0.0.1:8500/scope-injection.cfm?session.isAdmin=1


0x04 避免方案

1.变量初始化再使用

如上文提到代码,在Application.cfc中加入初始化:

<cffunction name="onSessionStart">
    <cfset session.isAdmin = false>
</cffunction>


当然了测试的时候不要忘记删除上边代码中的:

<!--- Make sure the user is not isAdmin --->
<cfif structKeyExists(session, "isAdmin")>
<cfset structDelete(session, "isAdmin")/>
</cfif>

2.检测权限时候使用StructKeyExists检测变量是否存在

<cfif StructKeyExists(session, "isAdmin") AND session.isAdmin>
  do admin stuff
</cfif>
